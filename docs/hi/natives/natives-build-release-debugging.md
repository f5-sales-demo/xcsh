# Natives बिल्ड, रिलीज़, और डिबगिंग रनबुक

यह रनबुक बताती है कि `@f5xc-salesdemos/pi-natives` बिल्ड पाइपलाइन `.node` एडऑन कैसे उत्पन्न करती है, कंपाइल किए गए डिस्ट्रीब्यूशन उन्हें कैसे लोड करते हैं, और लोडर/बिल्ड विफलताओं को कैसे डिबग करें।

यह `docs/natives-architecture.md` के आर्किटेक्चर शब्दों का पालन करती है:

- **बिल्ड-टाइम आर्टिफैक्ट उत्पादन** (`scripts/build-native.ts`)
- **एम्बेडेड एडऑन मैनिफेस्ट जनरेशन** (`scripts/embed-native.ts`)
- **रनटाइम एडऑन लोडिंग + वैलिडेशन गेट** (`src/native.ts`)

## कार्यान्वयन फाइलें

- `packages/natives/scripts/build-native.ts`
- `packages/natives/scripts/embed-native.ts`
- `packages/natives/package.json`
- `packages/natives/src/native.ts`
- `crates/pi-natives/Cargo.toml`

## बिल्ड पाइपलाइन अवलोकन

### 1) बिल्ड एंट्रीपॉइंट

`packages/natives/package.json` स्क्रिप्ट्स:

- `bun scripts/build-native.ts` (`build`) → रिलीज़ बिल्ड
- `bun scripts/build-native.ts --dev` (`dev:native`) → डिबग/डेव प्रोफाइल बिल्ड (समान आउटपुट नामकरण)
- `bun scripts/embed-native.ts` (`embed:native`) → बिल्ट फाइलों से `src/embedded-addon.ts` जनरेट करें

### 2) Rust आर्टिफैक्ट बिल्ड

`build-native.ts` `crates/pi-natives` में Cargo चलाता है:

- बेस कमांड: `cargo build`
- रिलीज़ मोड `--release` जोड़ता है जब तक `--dev` पास न किया जाए
- क्रॉस टारगेट `--target <CROSS_TARGET>` जोड़ता है

`crates/pi-natives/Cargo.toml` `crate-type = ["cdylib"]` घोषित करता है, इसलिए Cargo एक शेयर्ड लाइब्रेरी (`.so`/`.dylib`/`.dll`) उत्सर्जित करता है जिसे फिर `.node` एडऑन फाइलनाम में कॉपी/रीनेम किया जाता है।

### 3) आर्टिफैक्ट खोज और इंस्टॉल

Cargo पूरा होने के बाद, `build-native.ts` क्रम में उम्मीदवार आउटपुट डायरेक्टरी स्कैन करता है:

1. `${CARGO_TARGET_DIR}` (यदि सेट हो)
2. `<repo>/target`
3. `crates/pi-natives/target`

प्रत्येक रूट के लिए यह प्रोफाइल डायरेक्टरी जांचता है:

- क्रॉस बिल्ड: `<root>/<crossTarget>/<profile>` फिर `<root>/<profile>`
- नेटिव बिल्ड: `<root>/<profile>`

फिर यह इनमें से एक की तलाश करता है:

- `libpi_natives.so`
- `libpi_natives.dylib`
- `pi_natives.dll`
- `libpi_natives.dll`

मिलने पर, यह temp-file + rename सिमेंटिक्स के साथ `packages/natives/native/` में एटॉमिक रूप से इंस्टॉल करता है (Windows फॉलबैक लॉक्ड DLL रिप्लेसमेंट विफलताओं को स्पष्ट रूप से संभालता है)।

## टारगेट/वेरिएंट मॉडल और नामकरण परंपराएं

## प्लेटफॉर्म टैग

बिल्ड और रनटाइम दोनों प्लेटफॉर्म टैग का उपयोग करते हैं:

`<platform>-<arch>` (उदाहरण: `darwin-arm64`, `linux-x64`)

## वेरिएंट मॉडल (केवल x64)

x64 CPU वेरिएंट का समर्थन करता है:

- `modern` (AVX2-सक्षम पथ)
- `baseline` (फॉलबैक)

नॉन-x64 एकल डिफ़ॉल्ट आर्टिफैक्ट का उपयोग करता है (कोई वेरिएंट प्रत्यय नहीं)।

### आउटपुट फाइलनाम

रिलीज़ बिल्ड:

- x64: `pi_natives.<platform>-<arch>-modern.node` या `...-baseline.node`
- नॉन-x64: `pi_natives.<platform>-<arch>.node`

डेव बिल्ड (`--dev`):

- डिबग प्रोफाइल फ्लैग का उपयोग करता है लेकिन मानक प्लेटफॉर्म-टैग्ड आउटपुट नामकरण रखता है

`native.ts` में रनटाइम लोडर उम्मीदवार क्रम:

- रिलीज़ उम्मीदवार
- कंपाइल्ड मोड पैकेज-लोकल फाइलों से पहले एक्सट्रैक्टेड/कैश उम्मीदवार जोड़ता है

## एनवायरनमेंट फ्लैग और बिल्ड विकल्प

## रनटाइम फ्लैग

- `PI_DEV` (लोडर व्यवहार): लोडर डायग्नोस्टिक्स सक्षम करें
- `PI_NATIVE_VARIANT` (लोडर व्यवहार, केवल x64): रनटाइम पर `modern` या `baseline` चयन को बलपूर्वक करें
- `PI_COMPILED` (लोडर व्यवहार): कंपाइल्ड-बाइनरी उम्मीदवार/एक्सट्रैक्शन व्यवहार सक्षम करें

## बिल्ड-टाइम फ्लैग/विकल्प

- `--dev` (स्क्रिप्ट आर्ग): डिबग प्रोफाइल बिल्ड
- `CROSS_TARGET`: Cargo `--target` को पास किया जाता है
- `TARGET_PLATFORM`: आउटपुट प्लेटफॉर्म टैग नामकरण ओवरराइड
- `TARGET_ARCH`: आउटपुट आर्क नामकरण ओवरराइड
- `TARGET_VARIANT` (केवल x64): आउटपुट फाइलनाम और RUSTFLAGS नीति के लिए `modern` या `baseline` बलपूर्वक करें
- `CARGO_TARGET_DIR`: Cargo आउटपुट खोजते समय अतिरिक्त रूट
- `RUSTFLAGS`:
  - यदि अनसेट है और क्रॉस-कंपाइलिंग नहीं हो रही, तो स्क्रिप्ट सेट करती है:
    - modern: `-C target-cpu=x86-64-v3`
    - baseline: `-C target-cpu=x86-64-v2`
    - नॉन-x64 / कोई वेरिएंट नहीं: `-C target-cpu=native`
  - यदि पहले से सेट है, तो स्क्रिप्ट ओवरराइड नहीं करती

## बिल्ड स्टेट/लाइफसाइकल ट्रांज़िशन

### बिल्ड लाइफसाइकल (`build-native.ts`)

1. **Init**: आर्ग/एनवी पार्स करें (`--dev`, टारगेट ओवरराइड, क्रॉस फ्लैग)
2. **वेरिएंट रिज़ॉल्व**:
   - नॉन-x64 → कोई वेरिएंट नहीं
   - x64 + `TARGET_VARIANT` → स्पष्ट वेरिएंट
   - `TARGET_VARIANT` के बिना x64 क्रॉस-बिल्ड → हार्ड एरर
   - ओवरराइड के बिना x64 लोकल बिल्ड → होस्ट AVX2 का पता लगाएं
3. **कंपाइल**: रिज़ॉल्व किए गए प्रोफाइल/टारगेट के साथ Cargo चलाएं
4. **आर्टिफैक्ट ढूंढें**: टारगेट रूट/प्रोफाइल डायरेक्टरी/लाइब्रेरी नाम स्कैन करें
5. **इंस्टॉल**: `packages/natives/native` में कॉपी + एटॉमिक रीनेम
6. **पूर्ण**: लोडर उम्मीदवारों के लिए एडऑन तैयार

विफलता स्पष्ट एरर टेक्स्ट के साथ किसी भी चरण में बाहर निकलती है (अमान्य वेरिएंट, विफल cargo बिल्ड, गायब आउटपुट लाइब्रेरी, इंस्टॉल/रीनेम विफलता)।

### एम्बेड लाइफसाइकल (`embed-native.ts`)

1. **Init**: `TARGET_PLATFORM`/`TARGET_ARCH` या होस्ट वैल्यू से प्लेटफॉर्म टैग की गणना करें
2. **उम्मीदवार सेट**:
   - x64 दोनों `modern` और `baseline` की अपेक्षा करता है
   - नॉन-x64 एक डिफ़ॉल्ट फाइल की अपेक्षा करता है
3. **उपलब्धता सत्यापित करें** `packages/natives/native` में
4. **मैनिफेस्ट जनरेट करें** (`src/embedded-addon.ts`) Bun `file` इम्पोर्ट और पैकेज वर्शन के साथ
5. **रनटाइम एक्सट्रैक्शन तैयार** कंपाइल्ड मोड के लिए

`--reset` वैलिडेशन को बायपास करता है और एक नल मैनिफेस्ट स्टब लिखता है (`embeddedAddon = null`)।

## डेव वर्कफ़्लो बनाम शिप्ड/कंपाइल्ड व्यवहार

## लोकल डेवलपमेंट वर्कफ़्लो

सामान्य लोकल लूप:

1. एडऑन बिल्ड करें:
   - रिलीज़: `bun --cwd=packages/natives run build`
   - डिबग प्रोफाइल: `bun --cwd=packages/natives run dev:native`
2. लोडर डायग्नोस्टिक्स टेस्ट करते समय `PI_DEV=1` सेट करें
3. `native.ts` में लोडर पैकेज-लोकल `native/` (और executable-dir फॉलबैक) उम्मीदवारों को रिज़ॉल्व करता है
4. `validateNative` रैपर बाइंडिंग का उपयोग करने से पहले एक्सपोर्ट कम्पैटिबिलिटी लागू करता है

## शिप्ड/कंपाइल्ड बाइनरी वर्कफ़्लो

कंपाइल्ड मोड में (`PI_COMPILED` या Bun एम्बेडेड मार्कर):

1. लोडर वर्शन्ड कैश डायरेक्टरी की गणना करता है: `<getNativesDir()>/<packageVersion>` (ऑपरेशनली `~/.xcsh/natives/<version>`)
2. यदि एम्बेडेड मैनिफेस्ट वर्तमान प्लेटफॉर्म+वर्शन से मेल खाता है, तो लोडर चयनित एम्बेडेड फाइल को उस वर्शन्ड डायरेक्टरी में एक्सट्रैक्ट कर सकता है
3. रनटाइम उम्मीदवार क्रम में शामिल हैं:
   - वर्शन्ड कैश डायरेक्टरी
   - लेगेसी कंपाइल्ड-बाइनरी डायरेक्टरी (Windows पर `%LOCALAPPDATA%/xcsh`, अन्यत्र `~/.local/bin`)
   - पैकेज/executable डायरेक्टरी
4. पहले सफलतापूर्वक लोड हुए एडऑन को अभी भी `validateNative` पास करना होगा

यही कारण है कि पैकेजिंग + रनटाइम लोडर अपेक्षाओं का संरेखित होना आवश्यक है: फाइलनाम, प्लेटफॉर्म टैग, और एक्सपोर्टेड सिंबल उससे मेल खाने चाहिए जो `native.ts` प्रोब और वैलिडेट करता है।

## JS API ↔ Rust एक्सपोर्ट मैपिंग (वैलिडेशन गेट सबसेट)

`native.ts` के लिए आवश्यक है कि ये JS-विज़िबल एक्सपोर्ट लोड किए गए एडऑन पर मौजूद हों। ये `crates/pi-natives/src` में Rust N-API एक्सपोर्ट से मैप होते हैं:

| `validateNative` द्वारा आवश्यक JS नाम | Rust एक्सपोर्ट डिक्लेरेशन | Rust सोर्स फाइल |
| --- | --- | --- |
| `glob` | `#[napi] pub fn glob(...)` | `crates/pi-natives/src/glob.rs` |
| `grep` | `#[napi] pub fn grep(...)` | `crates/pi-natives/src/grep.rs` |
| `search` | `#[napi] pub fn search(...)` | `crates/pi-natives/src/grep.rs` |
| `highlightCode` | `#[napi] pub fn highlight_code(...)` | `crates/pi-natives/src/highlight.rs` |
| `getSystemInfo` | `#[napi] pub fn get_system_info(...)` | `crates/pi-natives/src/system_info.rs` |
| `getWorkProfile` | `#[napi] pub fn get_work_profile(...)` (कैमल-केस्ड एक्सपोर्ट) | `crates/pi-natives/src/prof.rs` |
| `invalidateFsScanCache` | `#[napi] pub fn invalidate_fs_scan_cache(...)` | `crates/pi-natives/src/fs_cache.rs` |

यदि कोई आवश्यक सिंबल गायब है, तो लोडर रीबिल्ड हिंट के साथ तुरंत विफल हो जाता है।

## विफलता व्यवहार और डायग्नोस्टिक्स

## बिल्ड-टाइम विफलताएं

- अमान्य वेरिएंट कॉन्फ़िगरेशन:
  - नॉन-x64 पर `TARGET_VARIANT` सेट → तत्काल एरर
  - स्पष्ट `TARGET_VARIANT` के बिना x64 क्रॉस-बिल्ड → तत्काल एरर
- Cargo बिल्ड विफलता:
  - स्क्रिप्ट नॉन-ज़ीरो एक्ज़िट और stderr दिखाती है
- आर्टिफैक्ट नहीं मिला:
  - स्क्रिप्ट हर जांची गई प्रोफाइल डायरेक्टरी प्रिंट करती है
- इंस्टॉल विफलता:
  - स्पष्ट संदेश; Windows में लॉक्ड-फाइल हिंट शामिल है

## रनटाइम लोडर विफलताएं (`native.ts`)

- असमर्थित प्लेटफॉर्म टैग:
  - समर्थित प्लेटफॉर्म सूची के साथ थ्रो करता है
- कोई उम्मीदवार लोड नहीं हो सका:
  - पूर्ण उम्मीदवार एरर सूची और मोड-विशिष्ट सुधार हिंट के साथ थ्रो करता है
- गायब एक्सपोर्ट:
  - सटीक गायब सिंबल नामों और रीबिल्ड कमांड के साथ थ्रो करता है
- एम्बेडेड एक्सट्रैक्शन समस्याएं:
  - एक्सट्रैक्शन mkdir/write एरर रिकॉर्ड होते हैं और अंतिम डायग्नोस्टिक्स में शामिल किए जाते हैं

## समस्या निवारण मैट्रिक्स

| लक्षण | संभावित कारण | सत्यापन | समाधान |
| --- | --- | --- | --- |
| `Native addon missing exports ... Missing: <name>` | पुरानी `.node` बाइनरी, Rust एक्सपोर्ट नाम बेमेल, या गलत बाइनरी लोड हुई | लोड किया गया पथ देखने के लिए `PI_DEV=1` के साथ चलाएं; उस फाइल के लिए एक्सपोर्ट सूची जांचें | `build` रीबिल्ड करें; सुनिश्चित करें कि Rust `#[napi]` एक्सपोर्ट नाम (या आवश्यकतानुसार स्पष्ट एलियास) JS की से मेल खाता है; पुरानी कैश्ड/वर्शन्ड फाइलें हटाएं |
| x64 मशीन modern की अपेक्षा होने पर baseline लोड करती है | `PI_NATIVE_VARIANT=baseline`, कोई AVX2 नहीं पाया गया, या केवल baseline फाइल मौजूद है | `PI_NATIVE_VARIANT` जांचें; `native/` में `-modern` फाइल देखें | modern वेरिएंट बिल्ड करें (`TARGET_VARIANT=modern ... build`) और सुनिश्चित करें कि फाइल शिप की गई है |
| क्रॉस-बिल्ड अनुपयोगी/गलत-लेबल वाली बाइनरी उत्पन्न करता है | `CROSS_TARGET` और `TARGET_PLATFORM`/`TARGET_ARCH` के बीच बेमेल, या x64 के लिए `TARGET_VARIANT` गायब | एनवी ट्यूपल और आउटपुट फाइलनाम की पुष्टि करें | सुसंगत एनवी वैल्यू और स्पष्ट x64 `TARGET_VARIANT` के साथ पुनः चलाएं |
| अपग्रेड के बाद कंपाइल्ड बाइनरी विफल होती है | पुरानी एक्सट्रैक्टेड कैश (`~/.xcsh/natives/<old-or-mismatched-version>`) या एम्बेडेड मैनिफेस्ट बेमेल | वर्शन्ड natives डायरेक्टरी और लोडर एरर सूची जांचें | पैकेज वर्शन के लिए वर्शन्ड natives कैश हटाएं और पुनः चलाएं; पैकेजिंग के दौरान एम्बेडेड मैनिफेस्ट पुनः जनरेट करें |
| लोडर कई पथ प्रोब करता है और कोई काम नहीं करता | प्लेटफॉर्म बेमेल या पैकेज `native/` में गायब रिलीज़ आर्टिफैक्ट | `platformTag` बनाम वास्तविक फाइलनाम जांचें | सुनिश्चित करें कि बिल्ट फाइलनाम बिल्कुल `pi_natives.<platform>-<arch>(-variant).node` परंपरा से मेल खाता है और पैकेज में `native/` शामिल है |
| `embed:native` "Incomplete native addons" के साथ विफल होता है | एम्बेडिंग से पहले आवश्यक वेरिएंट फाइलें नहीं बिल्ट हुईं | एरर टेक्स्ट में अपेक्षित बनाम पाई गई सूची जांचें | पहले आवश्यक फाइलें बिल्ड करें (x64: modern+baseline दोनों; नॉन-x64: डिफ़ॉल्ट), फिर `embed:native` पुनः चलाएं |

## ऑपरेशनल कमांड

```bash
# वर्तमान होस्ट के लिए रिलीज़ आर्टिफैक्ट
bun --cwd=packages/natives run build

# डिबग प्रोफाइल आर्टिफैक्ट बिल्ड
bun --cwd=packages/natives run dev:native

# स्पष्ट x64 वेरिएंट बिल्ड करें
TARGET_VARIANT=modern bun --cwd=packages/natives run build
TARGET_VARIANT=baseline bun --cwd=packages/natives run build

# बिल्ट नेटिव फाइलों से एम्बेडेड एडऑन मैनिफेस्ट जनरेट करें
bun --cwd=packages/natives run embed:native

# एम्बेडेड मैनिफेस्ट को नल स्टब पर रीसेट करें
bun --cwd=packages/natives run embed:native -- --reset
```
