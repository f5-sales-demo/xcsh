# `/tree` कमांड संदर्भ

`/tree` इंटरैक्टिव **Session Tree** नेविगेटर खोलता है। यह आपको वर्तमान सत्र फ़ाइल में किसी भी प्रविष्टि पर जाने और उस बिंदु से जारी रखने की अनुमति देता है।

यह एक इन-फ़ाइल लीफ मूव है, नया सत्र एक्सपोर्ट नहीं।

## `/tree` क्या करता है

- वर्तमान सत्र प्रविष्टियों से एक ट्री बनाता है (`SessionManager.getTree()`)
- कीबोर्ड नेविगेशन, फ़िल्टर और सर्च के साथ `TreeSelectorComponent` खोलता है
- चयन पर, `AgentSession.navigateTree(targetId, { summarize, customInstructions })` को कॉल करता है
- नए लीफ पथ से दृश्यमान चैट को पुनर्निर्मित करता है
- user/custom संदेश चुनते समय वैकल्पिक रूप से एडिटर टेक्स्ट प्रीफ़िल करता है

प्राथमिक कार्यान्वयन:

- `src/modes/controllers/input-controller.ts` (`/tree`, कीबाइंडिंग वायरिंग, डबल-एस्केप व्यवहार)
- `src/modes/controllers/selector-controller.ts` (ट्री UI लॉन्च + सारांश प्रॉम्प्ट फ़्लो)
- `src/modes/components/tree-selector.ts` (नेविगेशन, फ़िल्टर, सर्च, लेबल, रेंडरिंग)
- `src/session/agent-session.ts` (`navigateTree` लीफ स्विचिंग + वैकल्पिक सारांश)
- `src/session/session-manager.ts` (`getTree`, `branch`, `branchWithSummary`, `resetLeaf`, लेबल पर्सिस्टेंस)

## इसे कैसे खोलें

निम्नलिखित में से कोई भी वही सेलेक्टर खोलता है:

- `/tree`
- कॉन्फ़िगर की गई कीबाइंडिंग एक्शन `tree`
- खाली एडिटर पर डबल-एस्केप जब `doubleEscapeAction = "tree"` (डिफ़ॉल्ट)
- `/branch` जब `doubleEscapeAction = "tree"` (केवल-user ब्रांच पिकर के बजाय ट्री सेलेक्टर पर रूट करता है)

## ट्री UI मॉडल

ट्री सत्र प्रविष्टि पैरेंट पॉइंटर्स (`id` / `parentId`) से रेंडर किया जाता है।

- चिल्ड्रन को टाइमस्टैंप आरोही क्रम में सॉर्ट किया जाता है (पुराने पहले, नए नीचे)
- सक्रिय शाखा (रूट से वर्तमान लीफ तक का पथ) बुलेट से चिह्नित होती है
- लेबल (यदि मौजूद हों) नोड टेक्स्ट से पहले `[label]` के रूप में रेंडर होते हैं
- यदि एकाधिक रूट मौजूद हैं (अनाथ/टूटी पैरेंट चेन), तो वे एक वर्चुअल ब्रांचिंग रूट के अंतर्गत दिखाए जाते हैं

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

सेलेक्टर वर्तमान चयन के चारों ओर रीसेंटर करता है और अधिकतम दिखाता है:

- `max(5, floor(terminalHeight / 2))` पंक्तियाँ

## ट्री सेलेक्टर के अंदर कीबाइंडिंग

- `Up` / `Down`: चयन को स्थानांतरित करें (रैप करता है)
- `Left` / `Right`: पेज अप / पेज डाउन
- `Enter`: नोड चुनें
- `Esc`: सक्रिय होने पर सर्च साफ़ करें; अन्यथा सेलेक्टर बंद करें
- `Ctrl+C`: सेलेक्टर बंद करें
- `Type`: सर्च क्वेरी में जोड़ें
- `Backspace`: सर्च अक्षर हटाएं
- `Shift+L`: चयनित प्रविष्टि पर लेबल संपादित/साफ़ करें
- `Ctrl+O`: फ़िल्टर आगे साइकल करें
- `Shift+Ctrl+O`: फ़िल्टर पीछे साइकल करें
- `Alt+D/T/U/L/A`: सीधे विशिष्ट फ़िल्टर मोड पर जाएं

## फ़िल्टर और सर्च सिमैंटिक्स

फ़िल्टर मोड (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

अधिकांश वार्तालाप नोड्स दिखाता है, लेकिन बुककीपिंग प्रविष्टि प्रकार छुपाता है:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

`default` के समान, साथ ही `toolResult` संदेश छुपाता है।

### `user-only`

केवल `message` प्रविष्टियाँ जहाँ role `user` है।

### `labeled-only`

केवल वे प्रविष्टियाँ जो वर्तमान में किसी लेबल से हल होती हैं।

### `all`

सत्र ट्री में सब कुछ, बुककीपिंग/कस्टम प्रविष्टियों सहित।

### केवल-टूल असिस्टेंट नोड व्यवहार

असिस्टेंट संदेश जिनमें **केवल टूल कॉल** होते हैं (कोई टेक्स्ट नहीं) सभी फ़िल्टर किए गए व्यू में डिफ़ॉल्ट रूप से छिपे होते हैं जब तक कि:

- संदेश error/aborted हो (`stopReason` `stop`/`toolUse` नहीं), या
- यह वर्तमान लीफ हो (हमेशा दृश्यमान रखा जाता है)

### सर्च व्यवहार

- क्वेरी को स्पेस द्वारा टोकनाइज़ किया जाता है
- मैचिंग केस-इनसेंसिटिव है
- सभी टोकन मैच होने चाहिए (AND सिमैंटिक्स)
- सर्च योग्य टेक्स्ट में लेबल, role, और प्रकार-विशिष्ट सामग्री शामिल है (संदेश टेक्स्ट, शाखा सारांश टेक्स्ट, कस्टम प्रकार, टूल कमांड स्निपेट्स, आदि)

## चयन परिणाम (महत्वपूर्ण)

`navigateTree` चयनित प्रविष्टि प्रकार से नया लीफ व्यवहार गणना करता है:

### `user` संदेश का चयन

- नया लीफ चयनित प्रविष्टि का `parentId` बन जाता है
- यदि पैरेंट `null` है (रूट user संदेश), तो लीफ रूट पर रीसेट हो जाता है (`resetLeaf()`)
- चयनित संदेश टेक्स्ट संपादन/पुनः सबमिट के लिए एडिटर में कॉपी किया जाता है

### `custom_message` का चयन

- user संदेशों के समान लीफ नियम (`parentId`)
- टेक्स्ट सामग्री निकाली जाती है और एडिटर में कॉपी की जाती है

### गैर-user नोड का चयन (assistant/tool/summary/compaction/custom bookkeeping/आदि)

- नया लीफ चयनित नोड id बन जाता है
- एडिटर प्रीफ़िल नहीं होता

### वर्तमान लीफ का चयन

- नो-ऑप; सेलेक्टर "Already at this point" के साथ बंद हो जाता है

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## स्विच-पर-सारांश फ़्लो

सारांश प्रॉम्प्ट `branchSummary.enabled` द्वारा नियंत्रित होता है (डिफ़ॉल्ट: `false`)।

सक्षम होने पर, नोड चुनने के बाद UI पूछता है:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

फ़्लो विवरण:

- सारांश प्रॉम्प्ट में एस्केप ट्री सेलेक्टर को फिर से खोलता है
- कस्टम प्रॉम्प्ट रद्दीकरण सारांश चॉइस लूप पर वापस लौटता है
- सारांशीकरण के दौरान, UI लोडर दिखाता है और `Esc` को `abortBranchSummary()` से बाइंड करता है
- यदि सारांशीकरण निरस्त होता है, तो ट्री सेलेक्टर फिर से खुलता है और कोई मूव लागू नहीं होता

`navigateTree` आंतरिक कार्यप्रणाली:

- पुराने लीफ से कॉमन एंसेस्टर तक छोड़ी गई शाखा प्रविष्टियाँ एकत्र करता है
- `session_before_tree` एमिट करता है (एक्सटेंशन रद्द कर सकते हैं या सारांश इंजेक्ट कर सकते हैं)
- डिफ़ॉल्ट सारांशकर्ता का उपयोग केवल तभी करता है जब अनुरोध किया गया हो और आवश्यक हो
- मूव लागू करता है:
  - `branchWithSummary(...)` जब सारांश मौजूद हो
  - `branch(newLeafId)` बिना सारांश के गैर-रूट मूव के लिए
  - `resetLeaf()` बिना सारांश के रूट मूव के लिए
- एजेंट वार्तालाप को पुनर्निर्मित सत्र संदर्भ से बदलता है
- `session_tree` एमिट करता है

नोट: यदि उपयोगकर्ता सारांश का अनुरोध करता है लेकिन सारांशित करने के लिए कुछ नहीं है, तो नेविगेशन सारांश प्रविष्टि बनाए बिना आगे बढ़ता है।

## लेबल

ट्री UI में लेबल संपादन `appendLabelChange(targetId, label)` को कॉल करता है।

- गैर-रिक्त लेबल हल किए गए लेबल को सेट/अपडेट करता है
- रिक्त लेबल इसे साफ़ करता है
- लेबल एपेंड-ओनली `label` प्रविष्टियों के रूप में संग्रहीत होते हैं
- ट्री नोड्स हल किए गए लेबल स्थिति दिखाते हैं, कच्चा लेबल-प्रविष्टि इतिहास नहीं

## `/tree` बनाम संबंधित ऑपरेशन

| ऑपरेशन | दायरा | परिणाम |
|---|---|---|
| `/tree` | वर्तमान सत्र फ़ाइल | लीफ को चयनित बिंदु पर ले जाता है (वही फ़ाइल) |
| `/branch` | आमतौर पर वर्तमान सत्र फ़ाइल -> नई सत्र फ़ाइल | डिफ़ॉल्ट रूप से चयनित **user** संदेश से नई सत्र फ़ाइल में शाखा बनाता है; यदि `doubleEscapeAction = "tree"`, तो `/branch` इसके बजाय ट्री नेविगेशन UI खोलता है |
| `/fork` | संपूर्ण वर्तमान सत्र | सत्र को नई पर्सिस्टेड सत्र फ़ाइल में डुप्लिकेट करता है |
| `/resume` | सत्र सूची | किसी अन्य सत्र फ़ाइल पर स्विच करता है |

मुख्य अंतर: `/tree` एक सत्र फ़ाइल के अंदर नेविगेशन/रिपोज़िशनिंग टूल है। `/branch`, `/fork`, और `/resume` सभी सत्र-फ़ाइल संदर्भ बदलते हैं।

## ऑपरेटर वर्कफ़्लो

### वर्तमान शाखा खोए बिना पहले के user प्रॉम्प्ट से फिर से चलाएं

1. `/tree`
2. पहले के user संदेश को सर्च/चुनें
3. `No summary` चुनें (या आवश्यकता हो तो सारांशित करें)
4. एडिटर में प्रीफ़िल किए गए टेक्स्ट को संपादित करें
5. सबमिट करें

प्रभाव: वही सत्र फ़ाइल में चयनित बिंदु से नई शाखा बढ़ती है।

### संदर्भ ब्रेडक्रंब के साथ वर्तमान शाखा छोड़ें

1. `branchSummary.enabled` सक्षम करें
2. `/tree` और लक्ष्य नोड चुनें
3. `Summarize` चुनें (या कस्टम प्रॉम्प्ट)

प्रभाव: जारी रखने से पहले लक्ष्य स्थिति पर एक `branch_summary` प्रविष्टि जोड़ी जाती है।

### छिपी बुककीपिंग प्रविष्टियों की जांच करें

1. `/tree`
2. `Alt+A` दबाएं (सभी)
3. `model`, `thinking`, `custom`, या लेबल के लिए सर्च करें

प्रभाव: केवल वार्तालाप नोड्स ही नहीं, पूर्ण आंतरिक टाइमलाइन का निरीक्षण करें।

### बाद में जंप के लिए पिवट पॉइंट बुकमार्क करें

1. `/tree`
2. प्रविष्टि पर जाएं
3. `Shift+L` और लेबल सेट करें
4. बाद में जल्दी से जंप करने के लिए `Alt+L` (`labeled-only`) का उपयोग करें

प्रभाव: टिकाऊ शाखा लैंडमार्क्स के बीच तेज़ नेविगेशन।
