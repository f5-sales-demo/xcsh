# `/tree` कमांड संदर्भ

`/tree` इंटरैक्टिव **Session Tree** नेविगेटर खोलता है। यह आपको वर्तमान सत्र फ़ाइल में किसी भी प्रविष्टि पर जाने और उस बिंदु से जारी रखने की अनुमति देता है।

यह एक इन-फ़ाइल लीफ मूव है, नया सत्र एक्सपोर्ट नहीं।

## `/tree` क्या करता है

- वर्तमान सत्र प्रविष्टियों से ट्री बनाता है (`SessionManager.getTree()`)
- कीबोर्ड नेविगेशन, फ़िल्टर और सर्च के साथ `TreeSelectorComponent` खोलता है
- चयन पर, `AgentSession.navigateTree(targetId, { summarize, customInstructions })` कॉल करता है
- नए लीफ पथ से दृश्यमान चैट पुनर्निर्मित करता है
- user/custom संदेश चयन करते समय वैकल्पिक रूप से एडिटर टेक्स्ट प्रीफ़िल करता है

प्राथमिक कार्यान्वयन:

- `src/modes/controllers/input-controller.ts` (`/tree`, कीबाइंडिंग वायरिंग, डबल-एस्केप व्यवहार)
- `src/modes/controllers/selector-controller.ts` (ट्री UI लॉन्च + सारांश प्रॉम्प्ट फ़्लो)
- `src/modes/components/tree-selector.ts` (नेविगेशन, फ़िल्टर, सर्च, लेबल, रेंडरिंग)
- `src/session/agent-session.ts` (`navigateTree` लीफ स्विचिंग + वैकल्पिक सारांश)
- `src/session/session-manager.ts` (`getTree`, `branch`, `branchWithSummary`, `resetLeaf`, लेबल पर्सिस्टेंस)

## इसे कैसे खोलें

निम्नलिखित में से कोई भी समान सिलेक्टर खोलता है:

- `/tree`
- कॉन्फ़िगर किया गया कीबाइंडिंग एक्शन `tree`
- खाली एडिटर पर डबल-एस्केप जब `doubleEscapeAction = "tree"` (डिफ़ॉल्ट)
- `/branch` जब `doubleEscapeAction = "tree"` (user-only ब्रांच पिकर के बजाय ट्री सिलेक्टर पर रूट करता है)

## ट्री UI मॉडल

ट्री सत्र प्रविष्टि पैरेंट पॉइंटर्स (`id` / `parentId`) से रेंडर किया जाता है।

- चिल्ड्रेन टाइमस्टैम्प के अनुसार आरोही क्रम में सॉर्ट किए जाते हैं (पुराने पहले, नए नीचे)
- सक्रिय शाखा (रूट से वर्तमान लीफ तक का पथ) बुलेट के साथ चिह्नित होती है
- लेबल (यदि मौजूद हैं) नोड टेक्स्ट से पहले `[label]` के रूप में रेंडर होते हैं
- यदि एकाधिक रूट मौजूद हैं (अनाथ/टूटी पैरेंट चेन), तो उन्हें वर्चुअल ब्रांचिंग रूट के अंतर्गत दिखाया जाता है

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

सिलेक्टर वर्तमान चयन के आसपास पुनः केंद्रित होता है और अधिकतम दिखाता है:

- `max(5, floor(terminalHeight / 2))` पंक्तियाँ

## ट्री सिलेक्टर के अंदर कीबाइंडिंग

- `Up` / `Down`: चयन स्थानांतरित करें (रैप होता है)
- `Left` / `Right`: पेज अप / पेज डाउन
- `Enter`: नोड चुनें
- `Esc`: सक्रिय होने पर सर्च साफ़ करें; अन्यथा सिलेक्टर बंद करें
- `Ctrl+C`: सिलेक्टर बंद करें
- `Type`: सर्च क्वेरी में जोड़ें
- `Backspace`: सर्च कैरेक्टर हटाएँ
- `Shift+L`: चयनित प्रविष्टि पर लेबल संपादित/साफ़ करें
- `Ctrl+O`: फ़िल्टर आगे साइकल करें
- `Shift+Ctrl+O`: फ़िल्टर पीछे साइकल करें
- `Alt+D/T/U/L/A`: सीधे विशिष्ट फ़िल्टर मोड पर जाएँ

## फ़िल्टर और सर्च सिमैंटिक्स

फ़िल्टर मोड (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

अधिकांश वार्तालाप नोड दिखाता है, लेकिन बुककीपिंग प्रविष्टि प्रकार छिपाता है:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

`default` के समान, साथ ही `toolResult` संदेश छिपाता है।

### `user-only`

केवल `message` प्रविष्टियाँ जहाँ रोल `user` है।

### `labeled-only`

केवल वे प्रविष्टियाँ जो वर्तमान में किसी लेबल से हल होती हैं।

### `all`

सत्र ट्री में सब कुछ, बुककीपिंग/कस्टम प्रविष्टियों सहित।

### केवल-टूल असिस्टेंट नोड व्यवहार

असिस्टेंट संदेश जिनमें **केवल टूल कॉल** हैं (कोई टेक्स्ट नहीं) सभी फ़िल्टर किए गए व्यू में डिफ़ॉल्ट रूप से छिपे होते हैं जब तक कि:

- संदेश error/aborted हो (`stopReason` `stop`/`toolUse` नहीं है), या
- यह वर्तमान लीफ हो (हमेशा दृश्यमान रखा जाता है)

### सर्च व्यवहार

- क्वेरी रिक्त स्थानों द्वारा टोकनाइज़ की जाती है
- मैचिंग केस-इनसेंसिटिव है
- सभी टोकन मैच होने चाहिए (AND सिमैंटिक्स)
- खोजने योग्य टेक्स्ट में लेबल, रोल, और प्रकार-विशिष्ट सामग्री (संदेश टेक्स्ट, ब्रांच सारांश टेक्स्ट, कस्टम प्रकार, टूल कमांड स्निपेट आदि) शामिल हैं

## चयन परिणाम (महत्वपूर्ण)

`navigateTree` चयनित प्रविष्टि प्रकार से नए लीफ व्यवहार की गणना करता है:

### `user` संदेश का चयन

- नया लीफ चयनित प्रविष्टि का `parentId` बनता है
- यदि पैरेंट `null` है (रूट user संदेश), तो लीफ रूट पर रीसेट होता है (`resetLeaf()`)
- चयनित संदेश टेक्स्ट संपादन/पुनः सबमिट के लिए एडिटर में कॉपी किया जाता है

### `custom_message` का चयन

- user संदेशों के समान लीफ नियम (`parentId`)
- टेक्स्ट सामग्री निकाली जाती है और एडिटर में कॉपी की जाती है

### गैर-user नोड का चयन (assistant/tool/summary/compaction/custom bookkeeping/आदि)

- नया लीफ चयनित नोड id बनता है
- एडिटर प्रीफ़िल नहीं होता

### वर्तमान लीफ का चयन

- कोई कार्रवाई नहीं; सिलेक्टर "Already at this point" के साथ बंद होता है

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## स्विच-पर-सारांश फ़्लो

सारांश प्रॉम्प्ट `branchSummary.enabled` (डिफ़ॉल्ट: `false`) द्वारा नियंत्रित होता है।

सक्षम होने पर, नोड चुनने के बाद UI पूछता है:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

फ़्लो विवरण:

- सारांश प्रॉम्प्ट में एस्केप ट्री सिलेक्टर पुनः खोलता है
- कस्टम प्रॉम्प्ट रद्दीकरण सारांश चयन लूप पर लौटता है
- सारांशीकरण के दौरान, UI लोडर दिखाता है और `Esc` को `abortBranchSummary()` से बाइंड करता है
- यदि सारांशीकरण रद्द होता है, तो ट्री सिलेक्टर पुनः खुलता है और कोई मूव लागू नहीं होता

`navigateTree` आंतरिक कार्यप्रणाली:

- पुराने लीफ से सामान्य पूर्वज तक छोड़ी गई शाखा प्रविष्टियाँ एकत्र करता है
- `session_before_tree` उत्सर्जित करता है (एक्सटेंशन रद्द कर सकते हैं या सारांश इंजेक्ट कर सकते हैं)
- डिफ़ॉल्ट सारांशकर्ता का उपयोग केवल तभी करता है जब अनुरोध किया गया हो और आवश्यक हो
- इसके साथ मूव लागू करता है:
  - `branchWithSummary(...)` जब सारांश मौजूद हो
  - `branch(newLeafId)` बिना सारांश के गैर-रूट मूव के लिए
  - `resetLeaf()` बिना सारांश के रूट मूव के लिए
- एजेंट वार्तालाप को पुनर्निर्मित सत्र संदर्भ से बदलता है
- `session_tree` उत्सर्जित करता है

नोट: यदि उपयोगकर्ता सारांश का अनुरोध करता है लेकिन सारांशित करने के लिए कुछ नहीं है, तो नेविगेशन सारांश प्रविष्टि बनाए बिना आगे बढ़ता है।

## लेबल

ट्री UI में लेबल संपादन `appendLabelChange(targetId, label)` कॉल करता है।

- गैर-रिक्त लेबल हल किए गए लेबल को सेट/अपडेट करता है
- रिक्त लेबल इसे साफ़ करता है
- लेबल append-only `label` प्रविष्टियों के रूप में संग्रहीत होते हैं
- ट्री नोड हल किए गए लेबल स्थिति प्रदर्शित करते हैं, कच्चा लेबल-प्रविष्टि इतिहास नहीं

## `/tree` बनाम संबंधित ऑपरेशन

| ऑपरेशन | दायरा | परिणाम |
|---|---|---|
| `/tree` | वर्तमान सत्र फ़ाइल | चयनित बिंदु पर लीफ स्थानांतरित करता है (उसी फ़ाइल में) |
| `/branch` | आमतौर पर वर्तमान सत्र फ़ाइल -> नई सत्र फ़ाइल | डिफ़ॉल्ट रूप से चयनित **user** संदेश से नई सत्र फ़ाइल में ब्रांच करता है; यदि `doubleEscapeAction = "tree"`, तो `/branch` इसके बजाय ट्री नेविगेशन UI खोलता है |
| `/fork` | संपूर्ण वर्तमान सत्र | सत्र को नई स्थायी सत्र फ़ाइल में डुप्लिकेट करता है |
| `/resume` | सत्र सूची | दूसरी सत्र फ़ाइल पर स्विच करता है |

मुख्य अंतर: `/tree` एक सत्र फ़ाइल के अंदर नेविगेशन/पुनःस्थापन उपकरण है। `/branch`, `/fork`, और `/resume` सभी सत्र-फ़ाइल संदर्भ बदलते हैं।

## ऑपरेटर वर्कफ़्लो

### वर्तमान शाखा खोए बिना पहले के user प्रॉम्प्ट से पुनः चलाएँ

1. `/tree`
2. पहले के user संदेश को खोजें/चुनें
3. `No summary` चुनें (या आवश्यक होने पर सारांशित करें)
4. एडिटर में प्रीफ़िल किए गए टेक्स्ट को संपादित करें
5. सबमिट करें

प्रभाव: उसी सत्र फ़ाइल में चयनित बिंदु से नई शाखा बढ़ती है।

### संदर्भ ब्रेडक्रंब के साथ वर्तमान शाखा छोड़ें

1. `branchSummary.enabled` सक्षम करें
2. `/tree` और लक्ष्य नोड चुनें
3. `Summarize` चुनें (या कस्टम प्रॉम्प्ट)

प्रभाव: जारी रखने से पहले लक्ष्य स्थिति पर एक `branch_summary` प्रविष्टि जोड़ी जाती है।

### छिपी बुककीपिंग प्रविष्टियों की जाँच करें

1. `/tree`
2. `Alt+A` (all) दबाएँ
3. `model`, `thinking`, `custom`, या लेबल खोजें

प्रभाव: केवल वार्तालाप नोड ही नहीं, पूर्ण आंतरिक टाइमलाइन का निरीक्षण करें।

### बाद में त्वरित जंप के लिए पिवट बिंदुओं को बुकमार्क करें

1. `/tree`
2. प्रविष्टि पर जाएँ
3. `Shift+L` और लेबल सेट करें
4. बाद में त्वरित जंप के लिए `Alt+L` (`labeled-only`) का उपयोग करें

प्रभाव: स्थायी शाखा लैंडमार्क के बीच तेज़ नेविगेशन।
