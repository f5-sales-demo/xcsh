xcsh एक AI-संचालित डेवलपमेंट CLI है जिसमें एक TypeScript कोडिंग एजेंट और एक
Rust नेटिव लेयर (`pi-natives`) है। यह ओपन-सोर्स
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) लाइन को एक
सुदृढ़ रनटाइम, ट्री नेविगेशन और कॉम्पैक्शन के साथ लंबे समय तक चलने वाले सेशन,
एक Python IPython टूल, पूर्ण MCP सपोर्ट, एक स्किल्स सिस्टम, और Linux, macOS,
तथा Windows को लक्षित करने वाली प्लेटफ़ॉर्म पैकेजिंग के साथ विस्तारित करता है।

## कहाँ से शुरू करें

- **[F5 XC कॉन्टेक्स्ट](/runtime-tools/context-command)** — F5 Distributed Cloud
  टेनेंट से कनेक्ट करें। कॉन्टेक्स्ट बनाएँ, उनके बीच स्विच करें, नेमस्पेस और क्रेडेंशियल प्रबंधित करें।
- **कॉन्फ़िगरेशन** — xcsh कॉन्फ़िगरेशन को कैसे खोजता है, रिज़ॉल्व करता है, और लेयर करता है।
- **रनटाइम और टूल्स** — bash / notebook / resolve टूल रनटाइम और
  स्लैश-कमांड सरफ़ेस।
- **सेशन** — अपेंड-ओनली एंट्री लॉग, ट्री नेविगेशन, कॉम्पैक्शन, और
  स्वायत्त मेमोरी सिस्टम।
- **नेटिव्स (Rust)** — `pi-natives` N-API ऐडऑन की आर्किटेक्चर जो
  shell / PTY / मीडिया / सर्च को संचालित करती है।
- **MCP** — कॉन्फ़िगरेशन, प्रोटोकॉल इंटर्नल्स, रनटाइम लाइफ़साइकल, और
  सर्वर तथा टूल्स कैसे लिखें।
- **एक्सटेंशन, स्किल्स और प्लगइन्स** — ऑथरिंग, लोडिंग, मैचिंग नियम,
  मार्केटप्लेस, और प्लगइन इंस्टॉलर।
- **प्रोवाइडर और मॉडल** — मॉडल कॉन्फ़िगरेशन, स्ट्रीमिंग इंटर्नल्स, और
  Python / IPython रनटाइम।
- **TUI** — थीमिंग, `/tree` कमांड, और एक्सटेंशन तथा कस्टम टूल्स के लिए
  इंटीग्रेशन हुक्स।

## यह दस्तावेज़ सेट कैसे व्यवस्थित है

साइडबार में प्रत्येक शीर्ष-स्तरीय समूह एजेंट के एक सबसिस्टम से मैप होता है। एक
समूह के भीतर, पेज "अवलोकन" से "इंटर्नल्स" तक जाते हैं ताकि जब आपके पास
अपने कार्य के लिए पर्याप्त संदर्भ हो तो आप पढ़ना बंद कर सकें।
