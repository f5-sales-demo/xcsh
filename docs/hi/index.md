xcsh एक AI-संचालित विकास CLI है जिसमें एक TypeScript कोडिंग एजेंट और एक
Rust नेटिव लेयर (`pi-natives`) है। यह ओपन-सोर्स
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) लाइन को एक
सुदृढ़ रनटाइम, ट्री नेविगेशन और कॉम्पैक्शन के साथ दीर्घकालिक सत्र,
एक Python IPython टूल, पूर्ण MCP समर्थन, एक स्किल्स सिस्टम, और Linux,
macOS तथा Windows को लक्षित करने वाली प्लेटफ़ॉर्म पैकेजिंग के साथ विस्तारित करता है।

## कहाँ से शुरू करें

- **[F5 XC कॉन्टेक्स्ट](/runtime-tools/context-command)** — F5 Distributed Cloud
  टेनेंट से कनेक्ट करें। कॉन्टेक्स्ट बनाएँ, उनके बीच स्विच करें, नेमस्पेस और क्रेडेंशियल प्रबंधित करें।
- **कॉन्फ़िगरेशन** — xcsh कॉन्फ़िगरेशन को कैसे खोजता, हल करता और लेयर करता है।
- **रनटाइम और टूल्स** — bash / notebook / resolve टूल रनटाइम और
  स्लैश-कमांड सरफ़ेस।
- **सत्र** — केवल-जोड़ने वाला एंट्री लॉग, ट्री नेविगेशन, कॉम्पैक्शन, और
  स्वायत्त मेमोरी सिस्टम।
- **नेटिव्स (Rust)** — `pi-natives` N-API ऐडऑन की आर्किटेक्चर जो
  shell / PTY / मीडिया / सर्च को संचालित करती है।
- **MCP** — कॉन्फ़िगरेशन, प्रोटोकॉल इंटर्नल्स, रनटाइम जीवनचक्र, और
  सर्वर तथा टूल कैसे लिखें।
- **एक्सटेंशन, स्किल्स और प्लगइन** — लेखन, लोडिंग, मैचिंग नियम,
  मार्केटप्लेस, और प्लगइन इंस्टॉलर।
- **प्रोवाइडर और मॉडल** — मॉडल कॉन्फ़िगरेशन, स्ट्रीमिंग इंटर्नल्स, और
  Python / IPython रनटाइम।
- **TUI** — थीमिंग, `/tree` कमांड, और एक्सटेंशन तथा कस्टम टूल्स के लिए
  इंटीग्रेशन हुक।

## यह डॉक्यूमेंट सेट कैसे व्यवस्थित है

साइडबार में प्रत्येक शीर्ष-स्तरीय समूह एजेंट के एक उपसिस्टम से मैप होता है।
एक समूह के भीतर, पृष्ठ "अवलोकन" से "इंटर्नल्स" तक चलते हैं ताकि आप
अपने कार्य के लिए पर्याप्त संदर्भ मिलने पर पढ़ना बंद कर सकें।
