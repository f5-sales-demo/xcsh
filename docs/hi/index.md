xcsh एक AI-संचालित डेवलपमेंट CLI है जिसमें एक TypeScript कोडिंग एजेंट और एक
Rust नेटिव लेयर (`pi-natives`) है। यह ओपन-सोर्स
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) लाइन को एक
सुदृढ़ रनटाइम, ट्री नेविगेशन और कम्पैक्शन के साथ दीर्घकालिक सत्र,
एक Python IPython टूल, पूर्ण MCP समर्थन, एक स्किल्स सिस्टम, और Linux,
macOS, तथा Windows को लक्षित करने वाली प्लेटफ़ॉर्म पैकेजिंग के साथ विस्तारित करता है।

## कहाँ से शुरू करें

- **[F5 XC Contexts](/runtime-tools/context-command)** — F5 Distributed Cloud
  टेनेंट्स से कनेक्ट करें। कॉन्टेक्स्ट बनाएँ, उनके बीच स्विच करें, नेमस्पेस और क्रेडेंशियल्स प्रबंधित करें।
- **कॉन्फ़िगरेशन** — xcsh कॉन्फ़िगरेशन को कैसे खोजता है, रिज़ॉल्व करता है और लेयर करता है।
- **रनटाइम और टूल्स** — bash / notebook / resolve टूल रनटाइम और
  स्लैश-कमांड सतह।
- **सत्र (Sessions)** — append-only एंट्री लॉग, ट्री नेविगेशन, कम्पैक्शन, और
  स्वायत्त मेमोरी सिस्टम।
- **नेटिव्स (Rust)** — `pi-natives` N-API ऐडऑन की आर्किटेक्चर जो
  shell / PTY / media / search को संचालित करती है।
- **MCP** — कॉन्फ़िगरेशन, प्रोटोकॉल आंतरिक विवरण, रनटाइम जीवनचक्र, और
  सर्वर तथा टूल्स कैसे लिखें।
- **एक्सटेंशन, स्किल्स और प्लगइन्स** — लेखन, लोडिंग, मैचिंग नियम,
  मार्केटप्लेस, और प्लगइन इंस्टॉलर।
- **प्रोवाइडर्स और मॉडल्स** — मॉडल कॉन्फ़िगरेशन, स्ट्रीमिंग आंतरिक विवरण, और
  Python / IPython रनटाइम।
- **TUI** — थीमिंग, `/tree` कमांड, और एक्सटेंशन तथा कस्टम टूल्स के लिए
  इंटीग्रेशन हुक्स।

## यह दस्तावेज़ सेट कैसे संगठित है

साइडबार में प्रत्येक शीर्ष-स्तरीय समूह एजेंट के एक उपतंत्र से मैप करता है। एक
समूह के भीतर, पृष्ठ "अवलोकन" से "आंतरिक विवरण" तक क्रमबद्ध हैं ताकि आप
जब आपके कार्य के लिए पर्याप्त संदर्भ मिल जाए तो पढ़ना बंद कर सकें।
