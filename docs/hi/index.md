xcsh एक AI-संचालित डेवलपमेंट CLI है जिसमें एक TypeScript कोडिंग एजेंट और एक
Rust नेटिव लेयर (`pi-natives`) शामिल है। यह ओपन-सोर्स
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) लाइन को एक
हार्डन्ड रनटाइम, ट्री नेविगेशन और कॉम्पैक्शन के साथ लंबे समय तक चलने वाले सेशन,
एक Python IPython उपकरण, पूर्ण MCP सहायता, एक स्किल्स सिस्टम, और Linux, macOS
तथा Windows को लक्षित प्लेटफ़ॉर्म पैकेजिंग के साथ विस्तारित करता है।

## शुरू कहाँ से करें

- **[F5 XC कंटेक्स्ट](/runtime-tools/context-command)** — F5 Distributed Cloud
  टेनेंट से कनेक्ट करें। कंटेक्स्ट बनाएँ, उनके बीच स्विच करें, नेमस्पेस और क्रेडेंशियल प्रबंधित करें।
- **कॉन्फ़िगरेशन** — xcsh किस प्रकार कॉन्फ़िगरेशन को खोजता, हल करता और लेयर करता है।
- **रनटाइम और उपकरण** — bash / नोटबुक / रिज़ॉल्व उपकरण रनटाइम और
  स्लैश-कमांड इंटरफ़ेस।
- **सेशन** — अपेंड-ओनली एंट्री लॉग, ट्री नेविगेशन, कॉम्पैक्शन, और
  ऑटोनॉमस मेमोरी सिस्टम।
- **नेटिव्स (Rust)** — `pi-natives` N-API एडऑन की आर्किटेक्चर जो
  शेल / PTY / मीडिया / सर्च को संचालित करती है।
- **MCP** — कॉन्फ़िगरेशन, प्रोटोकॉल इंटर्नल्स, रनटाइम लाइफ़साइकिल, और सर्वर तथा उपकरण कैसे बनाएँ।
- **एक्सटेंशन, स्किल्स और प्लगइन** — ऑथरिंग, लोडिंग, मिलान नियम, मार्केटप्लेस और प्लगइन इंस्टॉलर।
- **प्रोवाइडर और मॉडल** — मॉडल कॉन्फ़िगरेशन, स्ट्रीमिंग इंटर्नल्स, और Python / IPython रनटाइम।
- **TUI** — थीमिंग, `/tree` कमांड, और एक्सटेंशन तथा कस्टम उपकरण के लिए इंटीग्रेशन हुक।

## यह दस्तावेज़ सेट कैसे व्यवस्थित है

साइडबार में प्रत्येक शीर्ष-स्तरीय समूह एजेंट के किसी उप-सिस्टम से संबंधित है। किसी
समूह के भीतर, पृष्ठ "अवलोकन" से "इंटर्नल्स" तक क्रम में होते हैं, ताकि आप जब
किसी कार्य के लिए पर्याप्त संदर्भ प्राप्त हो जाए तो पढ़ना बंद कर सकें।
