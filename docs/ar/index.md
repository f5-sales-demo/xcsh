xcsh هي واجهة سطر أوامر للتطوير مدعومة بالذكاء الاصطناعي مع وكيل برمجة TypeScript وطبقة
أصلية بلغة Rust (`pi-natives`). تمتد من المشروع مفتوح المصدر
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) مع بيئة تشغيل
مُعززة، وجلسات طويلة الأمد مع تنقل شجري وضغط،
وأداة Python IPython، ودعم كامل لـ MCP، ونظام مهارات، وتحزيم
للمنصات يستهدف Linux وmacOS وWindows.

## من أين تبدأ

- **[سياقات F5 XC](/runtime-tools/context-command)** — الاتصال بمستأجري F5 Distributed Cloud.
  إنشاء السياقات، والتبديل بينها، وإدارة مساحات الأسماء وبيانات الاعتماد.
- **التكوين** — كيف يكتشف xcsh التكوين ويحلله ويطبقه في طبقات.
- **بيئة التشغيل والأدوات** — بيئات تشغيل bash / notebook / resolve tool وواجهة
  أوامر الشرطة المائلة.
- **الجلسات** — سجل الإدخالات بالإلحاق فقط، والتنقل الشجري، والضغط، ونظام
  الذاكرة المستقل.
- **الطبقة الأصلية (Rust)** — بنية إضافة N-API الخاصة بـ `pi-natives` التي
  تُشغّل shell / PTY / media / search.
- **MCP** — التكوين، وتفاصيل البروتوكول الداخلية، ودورة حياة التشغيل، وكيفية
  تأليف الخوادم والأدوات.
- **الإضافات والمهارات والملحقات** — التأليف، والتحميل، وقواعد المطابقة، ومتجر
  التطبيقات، ومُثبّت الملحقات.
- **المزودون والنماذج** — تكوين النماذج، وتفاصيل البث الداخلية، وبيئة تشغيل
  Python / IPython.
- **واجهة المستخدم النصية** — التنسيق المرئي، وأمر `/tree`، وخطافات التكامل مع
  الإضافات والأدوات المخصصة.
