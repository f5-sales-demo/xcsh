xcsh هو واجهة سطر أوامر للتطوير مدعومة بالذكاء الاصطناعي مع وكيل برمجة TypeScript
وطبقة أصلية بلغة Rust (`pi-natives`). يُوسّع مشروع المصدر المفتوح
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) بإضافة بيئة تشغيل
محصّنة، وجلسات طويلة الأمد مع التنقل الشجري والضغط، وأداة Python IPython،
ودعم كامل لـ MCP، ونظام مهارات، وتغليف منصّات يستهدف Linux وmacOS وWindows.

## من أين تبدأ

- **[سياقات F5 XC](/runtime-tools/context-command)** — الاتصال بمستأجري F5 Distributed Cloud.
  إنشاء السياقات، والتبديل بينها، وإدارة مساحات الأسماء وبيانات الاعتماد.
- **التكوين** — كيف يكتشف xcsh التكوين ويحلّله ويُطبّقه بطبقات.
- **بيئة التشغيل والأدوات** — بيئات تشغيل bash / notebook / resolve وواجهة
  أوامر الشرطة المائلة.
- **الجلسات** — سجل الإدخالات المُلحَق فقط، والتنقل الشجري، والضغط، ونظام
  الذاكرة المستقل.
- **الطبقة الأصلية (Rust)** — بنية إضافة N-API `pi-natives` التي
  تُشغّل shell / PTY / الوسائط / البحث.
- **MCP** — التكوين، والتفاصيل الداخلية للبروتوكول، ودورة حياة التشغيل، وكيفية
  تأليف الخوادم والأدوات.
- **الإضافات والمهارات والملحقات** — التأليف، والتحميل، وقواعد المطابقة، والسوق،
  ومُثبّت الملحقات.
- **المزوّدون والنماذج** — تكوين النماذج، والتفاصيل الداخلية للبث، وبيئة تشغيل
  Python / IPython.
- **TUI** — السمات، وأمر `/tree`، وخطافات التكامل مع
  الإضافات والأدوات المخصصة.

## كيف تم تنظيم مجموعة التوثيق هذه

كل مجموعة رئيسية في الشريط الجانبي تتوافق مع نظام فرعي من الوكيل. داخل
كل مجموعة، تتدرج الصفحات من "نظرة عامة" إلى "التفاصيل الداخلية" بحيث يمكنك
التوقف عن القراءة عندما يكون لديك سياق كافٍ للمهمة التي بين يديك.
