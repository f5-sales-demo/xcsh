xcsh هي واجهة سطر أوامر للتطوير مدعومة بالذكاء الاصطناعي، تتضمن وكيل ترميز TypeScript وطبقة Rust أصلية (`pi-natives`). وهي تمتد على خط المصدر المفتوح
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) بإضافة بيئة تشغيل محصّنة، وجلسات طويلة الأمد مع التنقل في الشجرة والضغط، وأداة Python IPython، ودعم كامل لـ MCP، ونظام مهارات، وحزم للمنصة تستهدف Linux وmacOS وWindows.

## من أين تبدأ

- **[سياقات F5 XC](/runtime-tools/context-command)** — الاتصال بمستأجري F5 Distributed Cloud.
  إنشاء السياقات، والتبديل بينها، وإدارة مساحات الأسماء وبيانات الاعتماد.
- **الإعداد** — كيفية اكتشاف xcsh للإعداد وحله وتطبيق طبقاته.
- **بيئة التشغيل والأدوات** — بيئات تشغيل أدوات bash / notebook / resolve وواجهة أوامر الشرطة المائلة.
- **الجلسات** — سجل الإدخال للإلحاق فقط، والتنقل في الشجرة، والضغط، ونظام الذاكرة المستقل.
- **العناصر الأصلية (Rust)** — معمارية الوحدة الإضافية N-API الخاصة بـ `pi-natives` التي تشغّل shell / PTY / الوسائط / البحث.
- **MCP** — الإعداد، وبروتوكول الاتصال الداخلي، ودورة حياة بيئة التشغيل، وكيفية تأليف الخوادم والأدوات.
- **الامتدادات والمهارات والإضافات** — التأليف، والتحميل، وقواعد المطابقة، والسوق، ومثبّت الإضافات.
- **الموفرون والنماذج** — إعداد النماذج، والبث الداخلي، وبيئة تشغيل Python / IPython.
- **واجهة المستخدم النصية (TUI)** — التصميم، وأمر `/tree`، وخطاطيف التكامل للامتدادات والأدوات المخصصة.

## كيف يُنظَّم هذا المجموعة من التوثيق

كل مجموعة رئيسية في الشريط الجانبي تُعيَّن لنظام فرعي من الوكيل. وضمن
كل مجموعة، تسير الصفحات من "نظرة عامة" إلى "التفاصيل الداخلية" حتى تتمكن من التوقف عن القراءة
حين يصبح لديك سياق كافٍ للمهمة التي تعمل عليها.
