xcsh هي واجهة سطر أوامر للتطوير مدعومة بالذكاء الاصطناعي مع وكيل برمجة TypeScript وطبقة أصلية بلغة Rust (`pi-natives`). تُوسّع خط المصدر المفتوح
[`badlogic/pi-mono`](https://github.com/badlogic/pi-mono) بوقت تشغيل مُعزّز، وجلسات طويلة الأمد مع التنقل الشجري والضغط، وأداة Python IPython، ودعم كامل لـ MCP، ونظام مهارات، وتحزيم للمنصات يستهدف Linux وmacOS وWindows.

## من أين تبدأ

- **[سياقات F5 XC](/runtime-tools/context-command)** — الاتصال بمستأجري F5 Distributed Cloud.
  إنشاء السياقات، والتبديل بينها، وإدارة مساحات الأسماء وبيانات الاعتماد.
- **التكوين** — كيف يكتشف xcsh التكوين ويحلّله ويُطبّقه بطبقات.
- **وقت التشغيل والأدوات** — أوقات تشغيل bash / notebook / resolve وواجهة
  أوامر الشرطة المائلة.
- **الجلسات** — سجل الإدخالات المُلحقة فقط، والتنقل الشجري، والضغط، ونظام
  الذاكرة المستقل.
- **الطبقة الأصلية (Rust)** — بنية إضافة N-API الخاصة بـ `pi-natives` التي
  تُشغّل shell / PTY / الوسائط / البحث.
- **MCP** — التكوين، والتفاصيل الداخلية للبروتوكول، ودورة حياة وقت التشغيل، وكيفية
  تأليف الخوادم والأدوات.
- **الإضافات والمهارات والمكونات الإضافية** — التأليف، والتحميل، وقواعد المطابقة، والسوق،
  ومُثبّت المكونات الإضافية.
- **المزوّدون والنماذج** — تكوين النماذج، والتفاصيل الداخلية للبث، ووقت تشغيل
  Python / IPython.
- **واجهة المستخدم النصية (TUI)** — السمات، وأمر `/tree`، وخطافات التكامل
  للإضافات والأدوات المخصصة.
