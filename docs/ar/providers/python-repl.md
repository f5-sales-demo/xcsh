# أداة Python وبيئة تشغيل IPython

يصف هذا المستند مجموعة تنفيذ Python الحالية في `packages/coding-agent`.
يغطي سلوك الأداة، ودورة حياة النواة/البوابة، ومعالجة البيئة، ودلالات التنفيذ، وعرض المخرجات، وأوضاع الفشل التشغيلية.

## النطاق والملفات الرئيسية

- واجهة الأداة: `src/tools/python.ts`
- تنسيق النواة لكل جلسة/استدعاء: `src/ipy/executor.ts`
- بروتوكول النواة + تكامل البوابة: `src/ipy/kernel.ts`
- منسق البوابة المحلية المشتركة: `src/ipy/gateway-coordinator.ts`
- عارض الوضع التفاعلي لعمليات تشغيل Python التي يطلقها المستخدم: `src/modes/components/python-execution.ts`
- تصفية بيئة التشغيل وتحديد مسار Python: `src/ipy/runtime.ts`

## ما هي أداة Python

أداة `python` تنفذ خلية Python واحدة أو أكثر من خلال نواة مدعومة بـ Jupyter Kernel Gateway (وليس عبر تشغيل `python -c` مباشرة لكل خلية).

معاملات الأداة:

```ts
{
  cells: Array<{ code: string; title?: string }>;
  timeout?: number; // seconds, clamped to 1..600, default 30
  cwd?: string;
  reset?: boolean; // reset kernel before first cell only
}
```

الأداة ذات `concurrency = "exclusive"` لكل جلسة، لذا لا تتداخل الاستدعاءات.

## دورة حياة البوابة

### الأوضاع

هناك مساران للبوابة:

1. **بوابة خارجية** (عند تعيين `PI_PYTHON_GATEWAY_URL`)
   - تستخدم عنوان URL المُعدّ مباشرة.
   - مصادقة اختيارية باستخدام `PI_PYTHON_GATEWAY_TOKEN`.
   - لا يتم إنشاء أو إدارة عملية بوابة محلية.

2. **بوابة محلية مشتركة** (المسار الافتراضي)
   - تستخدم عملية مشتركة واحدة منسقة تحت `~/.xcsh/agent/python-gateway`.
   - ملف البيانات الوصفية: `gateway.json`
   - ملف القفل: `gateway.lock`
   - أمر التشغيل:
     - `python -m kernel_gateway`
     - مرتبطة بـ `127.0.0.1:<allocated-port>`
     - فحص صحة بدء التشغيل: `GET /api/kernelspecs`

### تنسيق البوابة المحلية المشتركة

`acquireSharedGateway()`:

- يأخذ قفل ملف (`gateway.lock`) مع نبضات قلب.
- يعيد استخدام `gateway.json` إذا كان معرّف العملية (PID) نشطاً وفحص الصحة ناجحاً.
- ينظف المعلومات/معرّفات العمليات القديمة عند الحاجة.
- يبدأ بوابة جديدة عند عدم وجود بوابة سليمة.

`releaseSharedGateway()` حالياً لا يفعل شيئاً (إيقاف النواة لا يوقف البوابة المشتركة).

`shutdownSharedGateway()` ينهي العملية المشتركة صراحة ويمسح بيانات البوابة الوصفية.

### قيد مهم

`python.sharedGateway=false` يُرفض عند بدء النواة:

- خطأ: `Shared Python gateway required; local gateways are disabled`
- لا يوجد وضع بوابة محلية غير مشتركة لكل عملية.

## دورة حياة النواة

كل عملية تنفيذ تستخدم نواة مُنشأة عبر `POST /api/kernels` على البوابة المحددة.

تسلسل بدء النواة:

1. فحص التوفر (`checkPythonKernelAvailability`)
2. إنشاء النواة (`/api/kernels`)
3. فتح اتصال websocket (`/api/kernels/:id/channels`)
4. تهيئة بيئة النواة (`cwd`، متغيرات البيئة، `sys.path`)
5. تنفيذ `PYTHON_PRELUDE`
6. تحميل وحدات الامتداد من:
   - المستخدم: `~/.xcsh/agent/modules/*.py`
   - المشروع: `<cwd>/.xcsh/modules/*.py` (تتجاوز وحدة المستخدم ذات الاسم نفسه)

إيقاف النواة:

- حذف النواة البعيدة عبر `DELETE /api/kernels/:id`
- إغلاق اتصال websocket
- استدعاء خطاف تحرير البوابة المشتركة (لا يفعل شيئاً حالياً)

## دلالات استمرارية الجلسة

`python.kernelMode` يتحكم في إعادة استخدام النواة:

- `session` (الافتراضي)
  - يعيد استخدام جلسات النواة المفتاحية بهوية الجلسة + دليل العمل.
  - التنفيذ متسلسل لكل جلسة عبر طابور.
  - يتم إخراج الجلسات الخاملة بعد 5 دقائق.
  - 4 جلسات كحد أقصى؛ يتم إخراج الأقدم عند التجاوز.
  - فحوصات نبض القلب تكشف النوى الميتة.
  - يُسمح بإعادة التشغيل التلقائي مرة واحدة؛ الانهيار المتكرر يؤدي إلى فشل حاد.

- `per-call`
  - ينشئ نواة جديدة لكل طلب تنفيذ.
  - يوقف النواة بعد الطلب.
  - لا توجد استمرارية حالة بين الاستدعاءات.

### سلوك الخلايا المتعددة في استدعاء أداة واحد

تعمل الخلايا بالتتابع في نفس مثيل النواة لذلك الاستدعاء.

إذا فشلت خلية وسيطة:

- تبقى حالة الخلايا السابقة في الذاكرة.
- تُعيد الأداة خطأ محدداً يشير إلى الخلية التي فشلت.
- لا يتم تنفيذ الخلايا اللاحقة.

`reset=true` ينطبق فقط على تنفيذ الخلية الأولى في ذلك الاستدعاء.

## تصفية البيئة وتحديد بيئة التشغيل

يتم تصفية البيئة قبل إطلاق بيئة تشغيل البوابة/النواة:

- القائمة المسموحة تتضمن المتغيرات الأساسية مثل `PATH`، `HOME`، متغيرات اللغة المحلية، `VIRTUAL_ENV`، `PYTHONPATH`، إلخ.
- بادئات مسموحة: `LC_`، `XDG_`، `PI_`
- القائمة المحظورة تزيل مفاتيح API الشائعة (OpenAI/Anthropic/Gemini/إلخ.)

ترتيب اختيار بيئة التشغيل:

1. بيئة افتراضية نشطة/موجودة (`VIRTUAL_ENV`، ثم `<cwd>/.venv`، `<cwd>/venv`)
2. بيئة افتراضية مُدارة في `~/.xcsh/python-env`
3. `python` أو `python3` في PATH

عند اختيار بيئة افتراضية، يتم إضافة مسار bin/Scripts الخاص بها في بداية `PATH`.

تهيئة بيئة النواة داخل Python أيضاً:

- `os.chdir(cwd)`
- حقن خريطة البيئة المقدمة في `os.environ`
- التأكد من وجود دليل العمل في `sys.path`

## توفر الأداة واختيار الوضع

`python.toolMode` (الافتراضي `both`) + تجاوز اختياري `PI_PY` يتحكمان في العرض:

- `ipy-only`
- `bash-only`
- `both`

القيم المقبولة لـ `PI_PY`:

- `0` / `bash` -> `bash-only`
- `1` / `py` -> `ipy-only`
- `mix` / `both` -> `both`

إذا فشل الفحص المسبق لـ Python، يتراجع إنشاء الأداة إلى وضع bash فقط لتلك الجلسة.

## تدفق التنفيذ والإلغاء/المهلة الزمنية

### المهلة الزمنية على مستوى الأداة

المهلة الزمنية لأداة `python` بالثواني، الافتراضي 30، محصورة بين `1..600`.

تجمع الأداة بين:

- إشارة إلغاء المُستدعي
- إشارة إلغاء المهلة الزمنية

باستخدام `AbortSignal.any(...)`.

### إلغاء تنفيذ النواة

عند الإلغاء/انتهاء المهلة:

- يُعلَّم التنفيذ كمُلغى.
- يتم محاولة مقاطعة النواة عبر REST (`POST /interrupt`) وقناة التحكم `interrupt_request`.
- تتضمن النتيجة `cancelled=true`.
- مسار المهلة الزمنية يُعلّق على المخرجات `Command timed out after <n> seconds`.

### سلوك stdin

الإدخال القياسي التفاعلي غير مدعوم.

إذا أصدرت النواة `input_request`:

- تسجل الأداة `stdinRequested=true`
- تُصدر نصاً توضيحياً
- ترسل `input_reply` فارغاً
- يُعامل التنفيذ كفشل في طبقة المنفذ

## التقاط المخرجات وعرضها

### فئات المخرجات المُلتقطة

من رسائل النواة:

- `stream` -> أجزاء نص عادي
- `display_data`/`execute_result` -> معالجة عرض غنية
- `error` -> نص تتبع الأخطاء
- MIME مخصص `application/x-xcsh-status` -> أحداث حالة مُهيكلة

أولوية عرض MIME:

1. `text/markdown`
2. `text/plain`
3. `text/html` (يُحوَّل إلى markdown أساسي)

بالإضافة إلى ذلك تُلتقط كمخرجات مُهيكلة:

- `application/json` -> بيانات شجرة JSON
- `image/png` -> حمولات الصور
- `application/x-xcsh-status` -> أحداث الحالة

### التخزين والاقتطاع

يتم بث المخرجات عبر `OutputSink` وقد تُحفظ في مخزن القطع الأثرية.

يمكن أن تتضمن نتائج الأداة بيانات وصفية للاقتطاع و `artifact://<id>` لاسترداد المخرجات الكاملة.

### سلوك العارض

- عارض الأداة (`python.ts`):
  - يعرض كتل خلايا الكود مع حالة لكل خلية
  - المعاينة المطوية تعرض 10 أسطر افتراضياً
  - يدعم الوضع الموسع للمخرجات الكاملة وتفاصيل الحالة الأغنى
- العارض التفاعلي (`python-execution.ts`):
  - يُستخدم لتنفيذ Python الذي يطلقه المستخدم في واجهة TUI
  - المعاينة المطوية تعرض 20 سطراً افتراضياً
  - يحد الأسطر الفردية الطويلة جداً إلى 4000 حرف لسلامة العرض
  - يعرض إشعارات الإلغاء/الخطأ/الاقتطاع

## دعم البوابة الخارجية

قم بتعيين:

```bash
export PI_PYTHON_GATEWAY_URL="http://127.0.0.1:8888"
# Optional:
export PI_PYTHON_GATEWAY_TOKEN="..."
```

الاختلافات في السلوك عن البوابة المحلية المشتركة:

- لا توجد ملفات قفل/معلومات بوابة محلية
- لا يتم إنشاء/إنهاء عملية محلية
- فحوصات الصحة وعمليات CRUD للنواة تعمل ضد نقطة النهاية الخارجية
- أخطاء المصادقة تُعرض مع إرشادات صريحة للرمز المميز

## استكشاف الأخطاء التشغيلية (أوضاع الفشل الحالية)

- **أداة Python غير متاحة**
  - تحقق من `python.toolMode` / `PI_PY`.
  - إذا فشل الفحص المسبق، تتراجع بيئة التشغيل إلى وضع bash فقط.

- **أخطاء توفر النواة**
  - الوضع المحلي يتطلب أن يكون كل من `kernel_gateway` و `ipykernel` قابلين للاستيراد في بيئة تشغيل Python المحددة.
  - قم بالتثبيت عبر:

    ```bash
    python -m pip install jupyter_kernel_gateway ipykernel
    ```

- **`python.sharedGateway=false` يسبب فشل بدء التشغيل**
  - هذا متوقع مع التنفيذ الحالي.

- **أخطاء مصادقة/إمكانية الوصول للبوابة الخارجية**
  - 401/403 -> قم بتعيين `PI_PYTHON_GATEWAY_TOKEN`.
  - مهلة زمنية/غير قابل للوصول -> تحقق من عنوان URL/الشبكة وصحة البوابة.

- **التنفيذ يتوقف ثم تنتهي المهلة الزمنية**
  - قم بزيادة `timeout` للأداة (الحد الأقصى 600 ثانية) إذا كان حمل العمل مشروعاً.
  - للكود المتوقف، يؤدي الإلغاء إلى مقاطعة النواة لكن قد يحتاج كود المستخدم إلى إعادة هيكلة.

- **طلبات stdin/input في كود Python**
  - `input()` غير مدعوم تفاعلياً في مسار التشغيل هذا؛ قم بتمرير البيانات برمجياً.

- **استنفاد الموارد (`EMFILE` / عدد كبير جداً من الملفات المفتوحة)**
  - مدير الجلسات يطلق استرداد البوابة المشتركة (إنهاء الجلسة + إعادة تشغيل البوابة المشتركة).

- **أخطاء دليل العمل**
  - تتحقق الأداة من وجود `cwd` وأنه دليل قبل التنفيذ.

## متغيرات البيئة ذات الصلة

- `PI_PY` — تجاوز عرض الأداة (ربط `bash-only`/`ipy-only`/`both` أعلاه)
- `PI_PYTHON_GATEWAY_URL` — استخدام بوابة خارجية
- `PI_PYTHON_GATEWAY_TOKEN` — رمز مصادقة اختياري للبوابة الخارجية
- `PI_PYTHON_SKIP_CHECK=1` — تجاوز فحوصات Python المسبقة/التحضيرية
- `PI_PYTHON_IPC_TRACE=1` — تسجيل تتبعات إرسال/استقبال IPC للنواة
- `PI_DEBUG_STARTUP=1` — إصدار علامات تصحيح مراحل بدء التشغيل
