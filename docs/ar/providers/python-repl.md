# أداة Python وبيئة تشغيل IPython

يصف هذا المستند مكدس تنفيذ Python الحالي في `packages/coding-agent`.
يغطي سلوك الأداة، ودورة حياة النواة/البوابة، ومعالجة البيئة، ودلالات التنفيذ، وعرض المخرجات، وأنماط الفشل التشغيلية.

## النطاق والملفات الرئيسية

- سطح الأداة: `src/tools/python.ts`
- تنسيق النواة لكل جلسة/استدعاء: `src/ipy/executor.ts`
- بروتوكول النواة + تكامل البوابة: `src/ipy/kernel.ts`
- منسق البوابة المحلية المشتركة: `src/ipy/gateway-coordinator.ts`
- عارض الوضع التفاعلي لعمليات تشغيل Python التي يطلقها المستخدم: `src/modes/components/python-execution.ts`
- تصفية بيئة التشغيل وتحديد مسار Python: `src/ipy/runtime.ts`

## ما هي أداة Python

أداة `python` تنفذ خلية Python واحدة أو أكثر من خلال نواة مدعومة ببوابة Jupyter Kernel Gateway (وليس عن طريق تشغيل `python -c` مباشرة لكل خلية).

معاملات الأداة:

```ts
{
  cells: Array<{ code: string; title?: string }>;
  timeout?: number; // seconds, clamped to 1..600, default 30
  cwd?: string;
  reset?: boolean; // reset kernel before first cell only
}
```

الأداة تعمل بوضع `concurrency = "exclusive"` للجلسة، لذا لا تتداخل الاستدعاءات.

## دورة حياة البوابة

### الأوضاع

هناك مساران للبوابة:

1. **بوابة خارجية** (عند تعيين `PI_PYTHON_GATEWAY_URL`)
   - تستخدم عنوان URL المُعدّ مباشرة.
   - مصادقة اختيارية باستخدام `PI_PYTHON_GATEWAY_TOKEN`.
   - لا يتم إنشاء أو إدارة عملية بوابة محلية.

2. **بوابة محلية مشتركة** (المسار الافتراضي)
   - تستخدم عملية مشتركة واحدة تُنسّق تحت `~/.xcsh/agent/python-gateway`.
   - ملف البيانات الوصفية: `gateway.json`
   - ملف القفل: `gateway.lock`
   - أمر التشغيل:
     - `python -m kernel_gateway`
     - مرتبط بـ `127.0.0.1:<allocated-port>`
     - فحص صحة بدء التشغيل: `GET /api/kernelspecs`

### تنسيق البوابة المحلية المشتركة

`acquireSharedGateway()`:

- تأخذ قفل ملف (`gateway.lock`) مع نبضات حياة.
- تعيد استخدام `gateway.json` إذا كان معرّف العملية حيًا وفحص الصحة ناجحًا.
- تنظف المعلومات/معرّفات العمليات القديمة عند الحاجة.
- تبدأ بوابة جديدة عند عدم وجود بوابة سليمة.

`releaseSharedGateway()` هي حاليًا عملية فارغة (إيقاف النواة لا يُوقف البوابة المشتركة).

`shutdownSharedGateway()` تُنهي صراحةً العملية المشتركة وتمسح البيانات الوصفية للبوابة.

### قيد مهم

`python.sharedGateway=false` يُرفض عند بدء تشغيل النواة:

- خطأ: `Shared Python gateway required; local gateways are disabled`
- لا يوجد وضع بوابة محلية غير مشتركة لكل عملية.

## دورة حياة النواة

كل عملية تنفيذ تستخدم نواة تُنشأ عبر `POST /api/kernels` على البوابة المحددة.

تسلسل بدء تشغيل النواة:

1. فحص التوفر (`checkPythonKernelAvailability`)
2. إنشاء النواة (`/api/kernels`)
3. فتح websocket (`/api/kernels/:id/channels`)
4. تهيئة بيئة النواة (`cwd`، متغيرات البيئة، `sys.path`)
5. تنفيذ `PYTHON_PRELUDE`
6. تحميل وحدات الامتداد من:
   - المستخدم: `~/.xcsh/agent/modules/*.py`
   - المشروع: `<cwd>/.xcsh/modules/*.py` (تتجاوز وحدة المستخدم ذات الاسم نفسه)

إيقاف النواة:

- حذف النواة البعيدة عبر `DELETE /api/kernels/:id`
- إغلاق websocket
- استدعاء خطاف تحرير البوابة المشتركة (عملية فارغة حاليًا)

## دلالات استمرارية الجلسة

`python.kernelMode` يتحكم في إعادة استخدام النواة:

- `session` (الافتراضي)
  - يعيد استخدام جلسات النواة المُفهرسة بهوية الجلسة + مجلد العمل.
  - التنفيذ يتم بشكل تسلسلي لكل جلسة عبر طابور.
  - الجلسات الخاملة تُطرد بعد 5 دقائق.
  - بحد أقصى 4 جلسات؛ الأقدم تُطرد عند التجاوز.
  - فحوصات النبض تكشف الأنوية الميتة.
  - يُسمح بإعادة التشغيل التلقائي مرة واحدة؛ التعطل المتكرر يؤدي إلى فشل نهائي.

- `per-call`
  - تُنشئ نواة جديدة لكل طلب تنفيذ.
  - تُوقف النواة بعد الطلب.
  - لا توجد استمرارية حالة بين الاستدعاءات.

### سلوك الخلايا المتعددة في استدعاء أداة واحد

تعمل الخلايا بشكل تسلسلي في نفس مثيل النواة لذلك الاستدعاء.

إذا فشلت خلية وسيطة:

- تبقى حالة الخلايا السابقة في الذاكرة.
- تُعيد الأداة خطأً مُحددًا يشير إلى الخلية التي فشلت.
- لا تُنفذ الخلايا اللاحقة.

`reset=true` ينطبق فقط على تنفيذ الخلية الأولى في ذلك الاستدعاء.

## تصفية البيئة وتحديد بيئة التشغيل

تُصفّى البيئة قبل تشغيل البوابة/النواة:

- قائمة السماح تتضمن متغيرات أساسية مثل `PATH`، `HOME`، متغيرات الإعدادات المحلية، `VIRTUAL_ENV`، `PYTHONPATH`، إلخ.
- بادئات السماح: `LC_`، `XDG_`، `PI_`
- قائمة الحظر تُزيل مفاتيح API الشائعة (OpenAI/Anthropic/Gemini/إلخ.)

ترتيب تحديد بيئة التشغيل:

1. بيئة افتراضية نشطة/موجودة (`VIRTUAL_ENV`، ثم `<cwd>/.venv`، `<cwd>/venv`)
2. بيئة افتراضية مُدارة في `~/.xcsh/python-env`
3. `python` أو `python3` في PATH

عند تحديد بيئة افتراضية، يُضاف مسار bin/Scripts الخاص بها في بداية `PATH`.

تهيئة بيئة النواة داخل Python أيضًا:

- `os.chdir(cwd)`
- تحقن خريطة البيئة المقدمة في `os.environ`
- تضمن وجود مجلد العمل في `sys.path`

## توفر الأداة واختيار الوضع

`python.toolMode` (الافتراضي `both`) + تجاوز اختياري عبر `PI_PY` يتحكم في العرض:

- `ipy-only`
- `bash-only`
- `both`

القيم المقبولة لـ `PI_PY`:

- `0` / `bash` -> `bash-only`
- `1` / `py` -> `ipy-only`
- `mix` / `both` -> `both`

إذا فشل الفحص المسبق لـ Python، يتراجع إنشاء الأداة إلى وضع bash فقط لتلك الجلسة.

## تدفق التنفيذ والإلغاء/انتهاء المهلة

### مهلة مستوى الأداة

مهلة أداة `python` بالثواني، الافتراضي 30، محصورة بين `1..600`.

تجمع الأداة بين:

- إشارة إلغاء المُستدعي
- إشارة إلغاء المهلة الزمنية

باستخدام `AbortSignal.any(...)`.

### إلغاء تنفيذ النواة

عند الإلغاء/انتهاء المهلة:

- يُعلّم التنفيذ كملغى.
- يُحاول مقاطعة النواة عبر REST (`POST /interrupt`) وقناة التحكم `interrupt_request`.
- تتضمن النتيجة `cancelled=true`.
- مسار انتهاء المهلة يُضيف تعليقًا على المخرجات بـ `Command timed out after <n> seconds`.

### سلوك stdin

الإدخال التفاعلي عبر stdin غير مدعوم.

إذا أصدرت النواة `input_request`:

- تسجل الأداة `stdinRequested=true`
- تُصدر نصًا توضيحيًا
- ترسل `input_reply` فارغًا
- يُعامل التنفيذ كفشل في طبقة المُنفّذ

## التقاط المخرجات وعرضها

### فئات المخرجات الملتقطة

من رسائل النواة:

- `stream` -> أجزاء نصية عادية
- `display_data`/`execute_result` -> معالجة عرض غنية
- `error` -> نص تتبع المكدس
- نوع MIME مخصص `application/x-xcsh-status` -> أحداث حالة مُهيكلة

أسبقية MIME للعرض:

1. `text/markdown`
2. `text/plain`
3. `text/html` (يُحوّل إلى markdown أساسي)

تُلتقط أيضًا كمخرجات مُهيكلة:

- `application/json` -> بيانات شجرة JSON
- `image/png` -> حمولات صور
- `application/x-xcsh-status` -> أحداث حالة

### التخزين والاقتطاع

تُبث المخرجات عبر `OutputSink` وقد تُحفظ في مخزن القطع الأثرية.

يمكن أن تتضمن نتائج الأداة بيانات وصفية للاقتطاع و `artifact://<id>` لاسترداد المخرجات الكاملة.

### سلوك العارض

- عارض الأداة (`python.ts`):
  - يعرض كتل خلايا الكود مع حالة لكل خلية
  - المعاينة المطوية تعرض افتراضيًا 10 أسطر
  - يدعم الوضع الموسع لعرض المخرجات الكاملة وتفاصيل الحالة الأغنى
- العارض التفاعلي (`python-execution.ts`):
  - يُستخدم لتنفيذ Python الذي يطلقه المستخدم في واجهة TUI
  - المعاينة المطوية تعرض افتراضيًا 20 سطرًا
  - يحد الأسطر الفردية الطويلة جدًا إلى 4000 حرف لسلامة العرض
  - يعرض إشعارات الإلغاء/الخطأ/الاقتطاع

## دعم البوابة الخارجية

عيّن:

```bash
export PI_PYTHON_GATEWAY_URL="http://127.0.0.1:8888"
# Optional:
export PI_PYTHON_GATEWAY_TOKEN="..."
```

اختلافات السلوك عن البوابة المحلية المشتركة:

- لا توجد ملفات قفل/معلومات بوابة محلية
- لا يتم إنشاء/إنهاء عملية محلية
- فحوصات الصحة وعمليات CRUD للنواة تعمل ضد نقطة النهاية الخارجية
- إخفاقات المصادقة تُعرض مع إرشادات صريحة حول الرمز المميز

## استكشاف الأخطاء التشغيلية وإصلاحها (أنماط الفشل الحالية)

- **أداة Python غير متوفرة**
  - تحقق من `python.toolMode` / `PI_PY`.
  - إذا فشل الفحص المسبق، تتراجع بيئة التشغيل إلى وضع bash فقط.

- **أخطاء توفر النواة**
  - الوضع المحلي يتطلب أن يكون كل من `kernel_gateway` و `ipykernel` قابلين للاستيراد في بيئة تشغيل Python المُحددة.
  - ثبّتهما بواسطة:

    ```bash
    python -m pip install jupyter_kernel_gateway ipykernel
    ```

- **`python.sharedGateway=false` يسبب فشل بدء التشغيل**
  - هذا متوقع مع التنفيذ الحالي.

- **إخفاقات مصادقة/وصول البوابة الخارجية**
  - 401/403 -> عيّن `PI_PYTHON_GATEWAY_TOKEN`.
  - انتهاء المهلة/عدم إمكانية الوصول -> تحقق من عنوان URL/الشبكة وصحة البوابة.

- **التنفيذ يتوقف ثم تنتهي مهلته**
  - زِد `timeout` للأداة (الحد الأقصى 600 ثانية) إذا كان حمل العمل مشروعًا.
  - للكود المتوقف، يؤدي الإلغاء إلى مقاطعة النواة لكن كود المستخدم قد يحتاج إلى إعادة هيكلة.

- **مطالبات stdin/input في كود Python**
  - `input()` غير مدعوم تفاعليًا في مسار التشغيل هذا؛ مرر البيانات برمجيًا.

- **استنفاد الموارد (`EMFILE` / كثرة الملفات المفتوحة)**
  - مدير الجلسات يُفعّل استرداد البوابة المشتركة (إيقاف الجلسات + إعادة تشغيل البوابة المشتركة).

- **أخطاء مجلد العمل**
  - تتحقق الأداة من وجود `cwd` وأنه مجلد قبل التنفيذ.

## متغيرات البيئة ذات الصلة

- `PI_PY` — تجاوز عرض الأداة (تعيين `bash-only`/`ipy-only`/`both` كما ذُكر أعلاه)
- `PI_PYTHON_GATEWAY_URL` — استخدام بوابة خارجية
- `PI_PYTHON_GATEWAY_TOKEN` — رمز مصادقة البوابة الخارجية الاختياري
- `PI_PYTHON_SKIP_CHECK=1` — تجاوز فحوصات Python المسبقة/الإحمائية
- `PI_PYTHON_IPC_TRACE=1` — تسجيل تتبعات إرسال/استقبال IPC للنواة
- `PI_DEBUG_STARTUP=1` — إصدار علامات تصحيح مراحل بدء التشغيل
