# أداة Python وبيئة تشغيل IPython

يصف هذا المستند مكدس تنفيذ Python الحالي في `packages/coding-agent`.
يغطي سلوك الأداة، ودورة حياة النواة/البوابة، ومعالجة البيئة، ودلالات التنفيذ، وعرض المخرجات، وأوضاع الفشل التشغيلية.

## النطاق والملفات الرئيسية

- واجهة الأداة: `src/tools/python.ts`
- تنسيق النواة لكل جلسة/استدعاء: `src/ipy/executor.ts`
- بروتوكول النواة + تكامل البوابة: `src/ipy/kernel.ts`
- منسق البوابة المحلية المشتركة: `src/ipy/gateway-coordinator.ts`
- عارض الوضع التفاعلي لتشغيل Python المُفعّل من المستخدم: `src/modes/components/python-execution.ts`
- تصفية بيئة التشغيل وتحديد مسار Python: `src/ipy/runtime.ts`

## ما هي أداة Python

أداة `python` تنفذ خلية واحدة أو أكثر من خلايا Python من خلال نواة مدعومة بـ Jupyter Kernel Gateway (وليس عبر تشغيل `python -c` مباشرة لكل خلية).

معاملات الأداة:

```ts
{
  cells: Array<{ code: string; title?: string }>;
  timeout?: number; // seconds, clamped to 1..600, default 30
  cwd?: string;
  reset?: boolean; // reset kernel before first cell only
}
```

الأداة هي `concurrency = "exclusive"` للجلسة، لذا لا تتداخل الاستدعاءات.

## دورة حياة البوابة

### الأوضاع

هناك مساران للبوابة:

1. **بوابة خارجية** (عند تعيين `PI_PYTHON_GATEWAY_URL`)
   - تستخدم عنوان URL المُعدّ مباشرة.
   - مصادقة اختيارية باستخدام `PI_PYTHON_GATEWAY_TOKEN`.
   - لا يتم إنشاء أو إدارة عملية بوابة محلية.

2. **بوابة محلية مشتركة** (المسار الافتراضي)
   - تستخدم عملية مشتركة واحدة منسقة تحت `~/.xcsh/agent/python-gateway`.
   - ملف البيانات الوصفية: `gateway.json`
   - ملف القفل: `gateway.lock`
   - أمر التشغيل:
     - `python -m kernel_gateway`
     - مرتبطة بـ `127.0.0.1:<allocated-port>`
     - فحص صحة بدء التشغيل: `GET /api/kernelspecs`

### تنسيق البوابة المحلية المشتركة

`acquireSharedGateway()`:

- يأخذ قفل ملف (`gateway.lock`) مع نبضات قلب.
- يعيد استخدام `gateway.json` إذا كان معرف العملية (PID) حياً وفحص الصحة ناجحاً.
- ينظف المعلومات/معرفات العمليات القديمة عند الحاجة.
- يبدأ بوابة جديدة عندما لا توجد بوابة سليمة.

`releaseSharedGateway()` هي حالياً عملية فارغة (no-op) (إيقاف النواة لا يُنهي البوابة المشتركة).

`shutdownSharedGateway()` تُنهي صراحةً العملية المشتركة وتمسح البيانات الوصفية للبوابة.

### قيد مهم

`python.sharedGateway=false` يُرفض عند بدء تشغيل النواة:

- خطأ: `Shared Python gateway required; local gateways are disabled`
- لا يوجد وضع بوابة محلية غير مشتركة لكل عملية.

## دورة حياة النواة

كل عملية تنفيذ تستخدم نواة مُنشأة عبر `POST /api/kernels` على البوابة المحددة.

تسلسل بدء تشغيل النواة:

1. فحص التوفر (`checkPythonKernelAvailability`)
2. إنشاء النواة (`/api/kernels`)
3. فتح اتصال websocket (`/api/kernels/:id/channels`)
4. تهيئة بيئة النواة (`cwd`، متغيرات البيئة، `sys.path`)
5. تنفيذ `PYTHON_PRELUDE`
6. تحميل وحدات الامتداد من:
   - المستخدم: `~/.xcsh/agent/modules/*.py`
   - المشروع: `<cwd>/.xcsh/modules/*.py` (تتجاوز وحدة المستخدم ذات الاسم نفسه)

إيقاف النواة:

- حذف النواة البعيدة عبر `DELETE /api/kernels/:id`
- إغلاق اتصال websocket
- استدعاء خطاف تحرير البوابة المشتركة (عملية فارغة حالياً)

## دلالات استمرار الجلسة

`python.kernelMode` يتحكم في إعادة استخدام النواة:

- `session` (الافتراضي)
  - يعيد استخدام جلسات النواة المُفهرسة بهوية الجلسة + مجلد العمل (cwd).
  - التنفيذ مُتسلسل لكل جلسة عبر قائمة انتظار.
  - يتم إخلاء الجلسات الخاملة بعد 5 دقائق.
  - بحد أقصى 4 جلسات؛ يتم إخلاء الأقدم عند التجاوز.
  - فحوصات نبضات القلب تكتشف الأنوية الميتة.
  - يُسمح بإعادة التشغيل التلقائي مرة واحدة؛ الانهيار المتكرر => فشل حاد.

- `per-call`
  - تُنشئ نواة جديدة لكل طلب تنفيذ.
  - تُوقف النواة بعد الطلب.
  - لا يوجد استمرار للحالة بين الاستدعاءات.

### سلوك الخلايا المتعددة في استدعاء أداة واحد

تُنفذ الخلايا بالتسلسل في نفس مثيل النواة لذلك الاستدعاء.

إذا فشلت خلية وسيطة:

- تبقى حالة الخلايا السابقة في الذاكرة.
- تُرجع الأداة خطأً مُوجهاً يشير إلى الخلية التي فشلت.
- لا يتم تنفيذ الخلايا اللاحقة.

`reset=true` ينطبق فقط على تنفيذ الخلية الأولى في ذلك الاستدعاء.

## تصفية البيئة وتحديد بيئة التشغيل

يتم تصفية البيئة قبل تشغيل البوابة/نواة بيئة التشغيل:

- قائمة السماح تتضمن المتغيرات الأساسية مثل `PATH`، `HOME`، متغيرات اللغة المحلية، `VIRTUAL_ENV`، `PYTHONPATH`، إلخ.
- بادئات السماح: `LC_`، `XDG_`، `PI_`
- قائمة الحظر تزيل مفاتيح API الشائعة (OpenAI/Anthropic/Gemini/إلخ.)

ترتيب اختيار بيئة التشغيل:

1. بيئة افتراضية نشطة/موجودة (`VIRTUAL_ENV`، ثم `<cwd>/.venv`، `<cwd>/venv`)
2. بيئة افتراضية مُدارة في `~/.xcsh/python-env`
3. `python` أو `python3` في PATH

عند اختيار بيئة افتراضية، يُضاف مسار bin/Scripts الخاص بها في بداية `PATH`.

تهيئة بيئة النواة داخل Python أيضاً:

- `os.chdir(cwd)`
- تحقن خريطة البيئة المُقدمة في `os.environ`
- تضمن وجود مجلد العمل في `sys.path`

## توفر الأداة واختيار الوضع

`python.toolMode` (الافتراضي `both`) + تجاوز `PI_PY` الاختياري يتحكمان في الإتاحة:

- `ipy-only`
- `bash-only`
- `both`

القيم المقبولة لـ `PI_PY`:

- `0` / `bash` -> `bash-only`
- `1` / `py` -> `ipy-only`
- `mix` / `both` -> `both`

إذا فشل الفحص الأولي لـ Python، يتراجع إنشاء الأداة إلى وضع bash فقط لتلك الجلسة.

## تدفق التنفيذ والإلغاء/المهلة الزمنية

### المهلة الزمنية على مستوى الأداة

مهلة أداة `python` بالثواني، الافتراضي 30، محصورة بين `1..600`.

تجمع الأداة بين:

- إشارة إلغاء المُستدعي
- إشارة إلغاء المهلة الزمنية

باستخدام `AbortSignal.any(...)`.

### إلغاء تنفيذ النواة

عند الإلغاء/المهلة الزمنية:

- يُعلّم التنفيذ كملغي.
- تُحاول مقاطعة النواة عبر REST (`POST /interrupt`) وقناة التحكم `interrupt_request`.
- تتضمن النتيجة `cancelled=true`.
- مسار المهلة الزمنية يُضيف تعليقاً للمخرجات: `Command timed out after <n> seconds`.

### سلوك stdin

الإدخال القياسي التفاعلي غير مدعوم.

إذا أصدرت النواة `input_request`:

- تسجل الأداة `stdinRequested=true`
- تُصدر نصاً توضيحياً
- ترسل `input_reply` فارغاً
- يُعامل التنفيذ كفشل في طبقة المنفذ

## التقاط المخرجات والعرض

### فئات المخرجات الملتقطة

من رسائل النواة:

- `stream` -> أجزاء نص عادي
- `display_data`/`execute_result` -> معالجة عرض غني
- `error` -> نص تتبع الأخطاء
- MIME مخصص `application/x-xcsh-status` -> أحداث حالة منظمة

أولوية MIME للعرض:

1. `text/markdown`
2. `text/plain`
3. `text/html` (يُحوّل إلى markdown أساسي)

بالإضافة إلى ذلك تُلتقط كمخرجات منظمة:

- `application/json` -> بيانات شجرة JSON
- `image/png` -> حمولات الصور
- `application/x-xcsh-status` -> أحداث الحالة

### التخزين والاقتطاع

يتم بث المخرجات عبر `OutputSink` وقد تُحفظ في مخزن القطع الأثرية.

يمكن أن تتضمن نتائج الأداة بيانات وصفية للاقتطاع و `artifact://<id>` لاستعادة المخرجات الكاملة.

### سلوك العارض

- عارض الأداة (`python.ts`):
  - يعرض كتل خلايا الشيفرة مع حالة كل خلية
  - المعاينة المطوية تعرض افتراضياً 10 أسطر
  - يدعم الوضع الموسع لعرض المخرجات الكاملة وتفاصيل الحالة الأغنى
- العارض التفاعلي (`python-execution.ts`):
  - يُستخدم لتنفيذ Python المُفعّل من المستخدم في واجهة المستخدم النصية (TUI)
  - المعاينة المطوية تعرض افتراضياً 20 سطراً
  - يحصر الأسطر الفردية الطويلة جداً عند 4000 حرف لسلامة العرض
  - يعرض إشعارات الإلغاء/الخطأ/الاقتطاع

## دعم البوابة الخارجية

عيّن:

```bash
export PI_PYTHON_GATEWAY_URL="http://127.0.0.1:8888"
# Optional:
export PI_PYTHON_GATEWAY_TOKEN="..."
```

الاختلافات في السلوك عن البوابة المحلية المشتركة:

- لا توجد ملفات قفل/معلومات للبوابة المحلية
- لا يتم إنشاء/إنهاء عملية محلية
- تعمل فحوصات الصحة وعمليات CRUD للنواة مقابل نقطة النهاية الخارجية
- تظهر أخطاء المصادقة مع إرشادات صريحة بشأن الرمز المميز

## استكشاف الأخطاء التشغيلية وإصلاحها (أوضاع الفشل الحالية)

- **أداة Python غير متاحة**
  - تحقق من `python.toolMode` / `PI_PY`.
  - إذا فشل الفحص الأولي، تتراجع بيئة التشغيل إلى وضع bash فقط.

- **أخطاء توفر النواة**
  - الوضع المحلي يتطلب أن يكون كل من `kernel_gateway` و `ipykernel` قابلين للاستيراد في بيئة تشغيل Python المحددة.
  - ثبّتهما باستخدام:

    ```bash
    python -m pip install jupyter_kernel_gateway ipykernel
    ```

- **`python.sharedGateway=false` يسبب فشل بدء التشغيل**
  - هذا متوقع مع التنفيذ الحالي.

- **أخطاء مصادقة/وصول البوابة الخارجية**
  - 401/403 -> عيّن `PI_PYTHON_GATEWAY_TOKEN`.
  - مهلة زمنية/غير قابلة للوصول -> تحقق من عنوان URL/الشبكة وصحة البوابة.

- **التنفيذ يتوقف ثم تنتهي المهلة الزمنية**
  - زد مهلة الأداة `timeout` (الحد الأقصى 600 ثانية) إذا كان حمل العمل مشروعاً.
  - للشيفرة المتوقفة، يُفعّل الإلغاء مقاطعة النواة لكن قد تحتاج شيفرة المستخدم إلى إعادة هيكلة.

- **طلبات stdin/input في شيفرة Python**
  - `input()` غير مدعومة تفاعلياً في مسار بيئة التشغيل هذا؛ مرر البيانات برمجياً.

- **استنفاد الموارد (`EMFILE` / كثرة الملفات المفتوحة)**
  - يُفعّل مدير الجلسات استعادة البوابة المشتركة (إنهاء الجلسة + إعادة تشغيل البوابة المشتركة).

- **أخطاء مجلد العمل**
  - تتحقق الأداة من وجود `cwd` وأنه مجلد قبل التنفيذ.

## متغيرات البيئة ذات الصلة

- `PI_PY` — تجاوز إتاحة الأداة (تعيين `bash-only`/`ipy-only`/`both` أعلاه)
- `PI_PYTHON_GATEWAY_URL` — استخدام بوابة خارجية
- `PI_PYTHON_GATEWAY_TOKEN` — رمز مصادقة البوابة الخارجية الاختياري
- `PI_PYTHON_SKIP_CHECK=1` — تجاوز فحوصات Python الأولية/التسخينية
- `PI_PYTHON_IPC_TRACE=1` — تسجيل تتبعات إرسال/استقبال IPC للنواة
- `PI_DEBUG_STARTUP=1` — إصدار علامات تصحيح مراحل بدء التشغيل
