# مرجع أمر `/tree`

`/tree` يفتح متصفح **شجرة الجلسة** التفاعلي. يتيح لك الانتقال إلى أي إدخال في ملف الجلسة الحالية والمتابعة من تلك النقطة.

هذا انتقال ورقي داخل الملف، وليس تصديرًا لجلسة جديدة.

## ما يفعله `/tree`

- يبني شجرة من إدخالات الجلسة الحالية (`SessionManager.getTree()`)
- يفتح `TreeSelectorComponent` مع التنقل بلوحة المفاتيح والفلاتر والبحث
- عند الاختيار، يستدعي `AgentSession.navigateTree(targetId, { summarize, customInstructions })`
- يعيد بناء المحادثة المرئية من مسار الورقة الجديد
- يملأ نص المحرر اختياريًا عند تحديد رسالة مستخدم/مخصصة

التنفيذ الرئيسي:

- `src/modes/controllers/input-controller.ts` (`/tree`، ربط اختصارات لوحة المفاتيح، سلوك الضغط المزدوج على Escape)
- `src/modes/controllers/selector-controller.ts` (تشغيل واجهة الشجرة + تدفق طلب التلخيص)
- `src/modes/components/tree-selector.ts` (التنقل، الفلاتر، البحث، التسميات، العرض)
- `src/session/agent-session.ts` (`navigateTree` تبديل الورقة + التلخيص الاختياري)
- `src/session/session-manager.ts` (`getTree`، `branch`، `branchWithSummary`، `resetLeaf`، استمرارية التسميات)

## كيفية فتحه

أي مما يلي يفتح نفس المحدد:

- `/tree`
- إجراء اختصار لوحة المفاتيح المكوَّن `tree`
- الضغط المزدوج على Escape في محرر فارغ عندما يكون `doubleEscapeAction = "tree"` (الافتراضي)
- `/branch` عندما يكون `doubleEscapeAction = "tree"` (يوجه إلى محدد الشجرة بدلاً من منتقي الفروع للمستخدم فقط)

## نموذج واجهة الشجرة

يتم عرض الشجرة من مؤشرات العناصر الأصلية لإدخالات الجلسة (`id` / `parentId`).

- يتم ترتيب العناصر الفرعية حسب الطابع الزمني تصاعديًا (الأقدم أولاً، والأحدث أدناه)
- يتم تمييز الفرع النشط (المسار من الجذر إلى الورقة الحالية) بنقطة
- تُعرض التسميات (إن وُجدت) كـ `[label]` قبل نص العقدة
- إذا وُجدت جذور متعددة (سلاسل أصلية يتيمة/مكسورة)، تُعرض تحت جذر تفريعي افتراضي

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

يعيد المحدد التمركز حول التحديد الحالي ويعرض حتى:

- `max(5, floor(terminalHeight / 2))` صفوف

## اختصارات لوحة المفاتيح داخل محدد الشجرة

- `Up` / `Down`: تحريك التحديد (يلتف)
- `Left` / `Right`: صفحة لأعلى / صفحة لأسفل
- `Enter`: تحديد العقدة
- `Esc`: مسح البحث إن كان نشطًا؛ وإلا إغلاق المحدد
- `Ctrl+C`: إغلاق المحدد
- `Type`: الإلحاق باستعلام البحث
- `Backspace`: حذف حرف من البحث
- `Shift+L`: تعديل/مسح التسمية على الإدخال المحدد
- `Ctrl+O`: التنقل الدوري بين الفلاتر للأمام
- `Shift+Ctrl+O`: التنقل الدوري بين الفلاتر للخلف
- `Alt+D/T/U/L/A`: الانتقال مباشرة إلى وضع فلتر محدد

## دلالات الفلاتر والبحث

أوضاع الفلتر (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

يعرض معظم العقد التحادثية، لكنه يخفي أنواع الإدخالات الإدارية:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

مثل `default`، بالإضافة إلى إخفاء رسائل `toolResult`.

### `user-only`

فقط إدخالات `message` حيث الدور هو `user`.

### `labeled-only`

فقط الإدخالات التي تحتوي حاليًا على تسمية محلولة.

### `all`

كل شيء في شجرة الجلسة، بما في ذلك الإدخالات الإدارية/المخصصة.

### سلوك عقد المساعد التي تحتوي على أدوات فقط

رسائل المساعد التي تحتوي **فقط على استدعاءات أدوات** (بدون نص) تكون مخفية افتراضيًا في جميع العروض المفلترة ما لم:

- تكون الرسالة خطأ/متوقفة (`stopReason` ليس `stop`/`toolUse`)، أو
- تكون الورقة الحالية (تظل مرئية دائمًا)

### سلوك البحث

- يتم تقسيم الاستعلام إلى رموز بالمسافات
- المطابقة غير حساسة لحالة الأحرف
- يجب أن تتطابق جميع الرموز (دلالات AND)
- يتضمن النص القابل للبحث التسمية والدور والمحتوى الخاص بالنوع (نص الرسالة، نص ملخص الفرع، النوع المخصص، مقتطفات أوامر الأدوات، إلخ.)

## نتائج التحديد (مهم)

يحسب `navigateTree` سلوك الورقة الجديد من نوع الإدخال المحدد:

### تحديد رسالة `user`

- تصبح الورقة الجديدة `parentId` الخاص بالإدخال المحدد
- إذا كان الأصل `null` (رسالة مستخدم جذرية)، تتم إعادة تعيين الورقة إلى الجذر (`resetLeaf()`)
- يتم نسخ نص الرسالة المحددة إلى المحرر للتعديل/إعادة الإرسال

### تحديد `custom_message`

- نفس قاعدة الورقة كرسائل المستخدم (`parentId`)
- يتم استخراج محتوى النص ونسخه إلى المحرر

### تحديد عقدة غير مستخدم (مساعد/أداة/ملخص/ضغط/إدارية مخصصة/إلخ.)

- تصبح الورقة الجديدة معرّف العقدة المحددة
- لا يتم ملء المحرر مسبقًا

### تحديد الورقة الحالية

- لا يحدث شيء؛ يُغلق المحدد مع رسالة "بالفعل عند هذه النقطة"

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## تدفق التلخيص عند التبديل

يتم التحكم في طلب التلخيص بواسطة `branchSummary.enabled` (الافتراضي: `false`).

عند التفعيل، بعد اختيار عقدة تسأل الواجهة:

- `بدون تلخيص`
- `تلخيص`
- `تلخيص بطلب مخصص`

تفاصيل التدفق:

- الضغط على Escape في طلب التلخيص يعيد فتح محدد الشجرة
- إلغاء الطلب المخصص يعود إلى حلقة اختيار التلخيص
- أثناء التلخيص، تعرض الواجهة مؤشر تحميل وتربط `Esc` بـ `abortBranchSummary()`
- إذا تم إلغاء التلخيص، يُعاد فتح محدد الشجرة ولا يتم تطبيق أي انتقال

تفاصيل `navigateTree` الداخلية:

- يجمع إدخالات الفرع المهجور من الورقة القديمة إلى السلف المشترك
- يُصدر `session_before_tree` (يمكن للإضافات الإلغاء أو حقن تلخيص)
- يستخدم الملخص الافتراضي فقط إذا طُلب وكان مطلوبًا
- يطبق الانتقال بـ:
  - `branchWithSummary(...)` عند وجود تلخيص
  - `branch(newLeafId)` للانتقال غير الجذري بدون تلخيص
  - `resetLeaf()` للانتقال الجذري بدون تلخيص
- يستبدل محادثة الوكيل بسياق الجلسة المُعاد بناؤه
- يُصدر `session_tree`

ملاحظة: إذا طلب المستخدم تلخيصًا ولكن لا يوجد شيء لتلخيصه، يستمر التنقل دون إنشاء إدخال تلخيص.

## التسميات

تعديلات التسميات في واجهة الشجرة تستدعي `appendLabelChange(targetId, label)`.

- التسمية غير الفارغة تعيّن/تحدّث التسمية المحلولة
- التسمية الفارغة تمسحها
- يتم تخزين التسميات كإدخالات `label` إلحاقية فقط
- تعرض عقد الشجرة حالة التسمية المحلولة، وليس سجل إدخالات التسميات الخام

## `/tree` مقابل العمليات المجاورة

| العملية | النطاق | النتيجة |
|---|---|---|
| `/tree` | ملف الجلسة الحالية | ينقل الورقة إلى النقطة المحددة (نفس الملف) |
| `/branch` | عادةً ملف الجلسة الحالية -> ملف جلسة جديد | بشكل افتراضي يتفرع من رسالة **المستخدم** المحددة إلى ملف جلسة جديد؛ إذا كان `doubleEscapeAction = "tree"`، يفتح `/branch` واجهة التنقل الشجري بدلاً من ذلك |
| `/fork` | الجلسة الحالية بالكامل | يكرر الجلسة إلى ملف جلسة جديد مستمر |
| `/resume` | قائمة الجلسات | يتحول إلى ملف جلسة آخر |

التمييز الرئيسي: `/tree` أداة تنقل/إعادة تموضع داخل ملف جلسة واحد. `/branch` و `/fork` و `/resume` جميعها تغيّر سياق ملف الجلسة.

## سير عمل المشغّل

### إعادة التشغيل من طلب مستخدم سابق دون فقدان الفرع الحالي

1. `/tree`
2. البحث/تحديد رسالة المستخدم السابقة
3. اختيار `بدون تلخيص` (أو التلخيص إذا لزم الأمر)
4. تعديل النص المملوء مسبقًا في المحرر
5. الإرسال

التأثير: ينمو فرع جديد من النقطة المحددة ضمن نفس ملف الجلسة.

### مغادرة الفرع الحالي مع علامة سياق مرجعية

1. تفعيل `branchSummary.enabled`
2. `/tree` وتحديد العقدة المستهدفة
3. اختيار `تلخيص` (أو طلب مخصص)

التأثير: يتم إلحاق إدخال `branch_summary` عند الموضع المستهدف قبل المتابعة.

### فحص الإدخالات الإدارية المخفية

1. `/tree`
2. الضغط على `Alt+A` (الكل)
3. البحث عن `model` أو `thinking` أو `custom` أو التسميات

التأثير: فحص الخط الزمني الداخلي الكامل، وليس فقط العقد التحادثية.

### وضع إشارات مرجعية لنقاط المحور للانتقال لاحقًا

1. `/tree`
2. الانتقال إلى الإدخال
3. `Shift+L` وتعيين تسمية
4. لاحقًا استخدام `Alt+L` (`المسمّاة فقط`) للانتقال بسرعة

التأثير: تنقل سريع بين المعالم المرجعية الدائمة للفروع.
