# مرجع الأمر `/tree`

يفتح `/tree` متصفح **شجرة الجلسة** التفاعلي. يتيح لك الانتقال إلى أي إدخال في ملف الجلسة الحالي والمتابعة من تلك النقطة.

هذه عملية انتقال داخل الملف، وليست تصديرًا لجلسة جديدة.

## ما الذي يفعله `/tree`

- يبني شجرة من إدخالات الجلسة الحالية (`SessionManager.getTree()`)
- يفتح `TreeSelectorComponent` مع التنقل بلوحة المفاتيح والمرشحات والبحث
- عند التحديد، يستدعي `AgentSession.navigateTree(targetId, { summarize, customInstructions })`
- يُعيد بناء المحادثة المرئية من مسار الورقة الجديد
- يُعبئ نص المحرر مسبقًا اختياريًا عند تحديد رسالة مستخدم/مخصصة

التنفيذ الأساسي:

- `src/modes/controllers/input-controller.ts` (الأمر `/tree`، ربط الاختصارات، سلوك الإفلات المزدوج)
- `src/modes/controllers/selector-controller.ts` (إطلاق واجهة شجرة + تدفق طلب الملخص)
- `src/modes/components/tree-selector.ts` (التنقل، المرشحات، البحث، التسميات، العرض)
- `src/session/agent-session.ts` (تبديل ورقة `navigateTree` + الملخص الاختياري)
- `src/session/session-manager.ts` (`getTree`، `branch`، `branchWithSummary`، `resetLeaf`، استمرارية التسميات)

## كيفية فتحه

أي مما يلي يفتح نفس المحدد:

- `/tree`
- إجراء الاختصار المُهيَّأ `tree`
- الإفلات المزدوج على المحرر الفارغ عندما يكون `doubleEscapeAction = "tree"` (الإعداد الافتراضي)
- `/branch` عندما يكون `doubleEscapeAction = "tree"` (يوجّه إلى محدد الشجرة بدلًا من منتقي الفرع للمستخدم فقط)

## نموذج واجهة الشجرة

تُعرض الشجرة من مؤشرات الأصل لإدخالات الجلسة (`id` / `parentId`).

- يُرتَّب العناصر الفرعية حسب الطابع الزمني تصاعديًا (الأقدم أولًا، الأحدث أسفل)
- تُميَّز الفرع النشط (المسار من الجذر إلى الورقة الحالية) بنقطة
- تُعرض التسميات (إذا وُجدت) بتنسيق `[label]` قبل نص العقدة
- إذا كانت هناك جذور متعددة (سلاسل أصل مكسورة/يتيمة)، تُعرض تحت جذر تفريع افتراضي

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

يُعيد المحدد التمركز حول التحديد الحالي ويعرض ما يصل إلى:

- `max(5, floor(terminalHeight / 2))` صفًا

## اختصارات لوحة المفاتيح داخل محدد الشجرة

- `Up` / `Down`: تحريك التحديد (يلتف)
- `Left` / `Right`: الصفحة للأعلى / الصفحة للأسفل
- `Enter`: تحديد العقدة
- `Esc`: مسح البحث إذا كان نشطًا؛ وإلا إغلاق المحدد
- `Ctrl+C`: إغلاق المحدد
- `Type`: الإضافة إلى استعلام البحث
- `Backspace`: حذف حرف من البحث
- `Shift+L`: تحرير/مسح تسمية الإدخال المحدد
- `Ctrl+O`: تدوير المرشح للأمام
- `Shift+Ctrl+O`: تدوير المرشح للخلف
- `Alt+D/T/U/L/A`: الانتقال مباشرةً إلى وضع مرشح محدد

## المرشحات ودلالات البحث

أوضاع المرشح (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

يعرض معظم عقد المحادثة، لكنه يُخفي أنواع إدخالات الحجز:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

مثل `default`، بالإضافة إلى إخفاء رسائل `toolResult`.

### `user-only`

إدخالات `message` فقط حيث يكون الدور `user`.

### `labeled-only`

الإدخالات التي تحل إلى تسمية حاليًا فقط.

### `all`

كل شيء في شجرة الجلسة، بما في ذلك إدخالات الحجز/المخصصة.

### سلوك عقدة المساعد للأدوات فقط

رسائل المساعد التي تحتوي **على استدعاءات أدوات فقط** (بدون نص) مخفية افتراضيًا في جميع العروض المُرشَّحة ما لم:

- تكن الرسالة خطأً/مُجهَضة (`stopReason` ليس `stop`/`toolUse`)، أو
- كانت هي الورقة الحالية (تظل مرئية دائمًا)

### سلوك البحث

- يُجزَّأ الاستعلام بالمسافات
- المطابقة غير حساسة لحالة الأحرف
- يجب أن تتطابق جميع الرموز (دلالات AND)
- النص القابل للبحث يشمل التسمية والدور والمحتوى الخاص بالنوع (نص الرسالة، نص ملخص الفرع، النوع المخصص، مقتطفات أمر الأداة، وما إلى ذلك)

## نتائج التحديد (مهم)

تحسب `navigateTree` سلوك الورقة الجديدة من نوع الإدخال المحدد:

### تحديد رسالة `user`

- تصبح الورقة الجديدة هي `parentId` الخاصة بالإدخال المحدد
- إذا كان الأصل `null` (رسالة مستخدم جذرية)، تُعاد الورقة إلى الجذر (`resetLeaf()`)
- يُنسخ نص الرسالة المحددة إلى المحرر للتحرير/إعادة الإرسال

### تحديد `custom_message`

- نفس قاعدة الورقة كرسائل المستخدم (`parentId`)
- يُستخرج محتوى النص ويُنسخ إلى المحرر

### تحديد عقدة غير مستخدم (مساعد/أداة/ملخص/ضغط/حجز مخصص/إلخ)

- تصبح الورقة الجديدة هي معرّف العقدة المحددة
- لا يُعبأ المحرر مسبقًا

### تحديد الورقة الحالية

- لا تأثير؛ يُغلق المحدد مع رسالة "Already at this point"

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## تدفق الملخص عند التبديل

يتحكم في طلب الملخص `branchSummary.enabled` (الإعداد الافتراضي: `false`).

عند التفعيل، بعد اختيار عقدة، تسأل الواجهة:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

تفاصيل التدفق:

- الإفلات في طلب الملخص يُعيد فتح محدد الشجرة
- إلغاء الطلب المخصص يعود إلى حلقة اختيار الملخص
- أثناء التلخيص، تعرض الواجهة مؤشر تحميل وتربط `Esc` بـ `abortBranchSummary()`
- إذا أُجهض التلخيص، يُعاد فتح محدد الشجرة ولا يُطبَّق أي انتقال

داخليات `navigateTree`:

- تجمع إدخالات الفرع المهجور من الورقة القديمة إلى السلف المشترك
- يُصدر `session_before_tree` (يمكن للامتدادات الإلغاء أو حقن ملخص)
- يستخدم المُلخِّص الافتراضي فقط عند الطلب والحاجة
- يطبّق الانتقال باستخدام:
  - `branchWithSummary(...)` عند وجود ملخص
  - `branch(newLeafId)` لانتقال غير جذري بدون ملخص
  - `resetLeaf()` لانتقال جذري بدون ملخص
- يستبدل محادثة الوكيل بسياق الجلسة المُعاد بناؤه
- يُصدر `session_tree`

ملاحظة: إذا طلب المستخدم ملخصًا ولكن لا يوجد شيء لتلخيصه، يتم الانتقال دون إنشاء إدخال ملخص.

## التسميات

تستدعي تعديلات التسمية في واجهة الشجرة `appendLabelChange(targetId, label)`.

- التسمية غير الفارغة تضبط/تحدث التسمية المحلولة
- التسمية الفارغة تمسحها
- تُخزَّن التسميات كإدخالات `label` للإلحاق فقط
- تعرض عقد الشجرة حالة التسمية المحلولة، وليس تاريخ إدخالات التسمية الخام

## `/tree` مقابل العمليات المجاورة

| العملية | النطاق | النتيجة |
|---|---|---|
| `/tree` | ملف الجلسة الحالي | ينقل الورقة إلى النقطة المحددة (نفس الملف) |
| `/branch` | عادةً ملف الجلسة الحالي -> ملف جلسة جديد | يتفرع افتراضيًا من رسالة **المستخدم** المحددة إلى ملف جلسة جديد؛ إذا كان `doubleEscapeAction = "tree"`، يفتح `/branch` واجهة التنقل في الشجرة بدلًا من ذلك |
| `/fork` | الجلسة الحالية بالكامل | يُكرر الجلسة إلى ملف جلسة جديد مستمر |
| `/resume` | قائمة الجلسات | ينتقل إلى ملف جلسة آخر |

التمييز الرئيسي: `/tree` هو أداة تنقل/إعادة تموضع داخل ملف جلسة واحد. أما `/branch` و`/fork` و`/resume` فكلها تُغيّر سياق ملف الجلسة.

## سير عمل المشغّل

### إعادة التشغيل من طلب مستخدم سابق دون فقدان الفرع الحالي

1. `/tree`
2. البحث عن رسالة المستخدم السابقة وتحديدها
3. اختيار `No summary` (أو التلخيص إذا لزم)
4. تحرير النص المُعبأ مسبقًا في المحرر
5. الإرسال

التأثير: ينمو فرع جديد من النقطة المحددة داخل نفس ملف الجلسة.

### مغادرة الفرع الحالي مع إشارة سياقية

1. تفعيل `branchSummary.enabled`
2. `/tree` وتحديد العقدة المستهدفة
3. اختيار `Summarize` (أو طلب مخصص)

التأثير: يُلحق إدخال `branch_summary` عند الموضع المستهدف قبل المتابعة.

### فحص إدخالات الحجز المخفية

1. `/tree`
2. ضغط `Alt+A` (الكل)
3. البحث عن `model` أو `thinking` أو `custom` أو التسميات

التأثير: فحص الجدول الزمني الداخلي الكامل، وليس عقد المحادثة فقط.

### وضع إشارة على نقاط التحول للرجوع إليها لاحقًا

1. `/tree`
2. الانتقال إلى الإدخال
3. `Shift+L` وضبط تسمية
4. لاحقًا استخدام `Alt+L` (`labeled-only`) للانتقال السريع

التأثير: تنقل سريع بين معالم الفرع الدائمة.
