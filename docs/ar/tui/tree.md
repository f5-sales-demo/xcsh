# مرجع أمر `/tree`

يفتح أمر `/tree` متصفح **شجرة الجلسة** التفاعلي. يتيح لك الانتقال إلى أي إدخال في ملف الجلسة الحالي والمتابعة من تلك النقطة.

هذا انتقال داخل الملف وليس تصديرًا لجلسة جديدة.

## ما يفعله أمر `/tree`

- يبني شجرة من إدخالات الجلسة الحالية (`SessionManager.getTree()`)
- يفتح `TreeSelectorComponent` مع التنقل بلوحة المفاتيح والفلاتر والبحث
- عند الاختيار، يستدعي `AgentSession.navigateTree(targetId, { summarize, customInstructions })`
- يعيد بناء المحادثة المرئية من مسار الفرع الجديد
- يملأ محرر النصوص اختياريًا عند اختيار رسالة مستخدم/مخصصة

التنفيذ الأساسي:

- `src/modes/controllers/input-controller.ts` (أمر `/tree`، ربط اختصارات لوحة المفاتيح، سلوك الضغط المزدوج على Escape)
- `src/modes/controllers/selector-controller.ts` (تشغيل واجهة الشجرة + تدفق طلب الملخص)
- `src/modes/components/tree-selector.ts` (التنقل، الفلاتر، البحث، التسميات، العرض)
- `src/session/agent-session.ts` (تبديل الفرع عبر `navigateTree` + الملخص الاختياري)
- `src/session/session-manager.ts` (`getTree`، `branch`، `branchWithSummary`، `resetLeaf`، حفظ التسميات)

## كيفية فتحه

أي مما يلي يفتح نفس المحدد:

- `/tree`
- إجراء اختصار لوحة المفاتيح المُعدّ `tree`
- الضغط المزدوج على Escape في محرر فارغ عندما يكون `doubleEscapeAction = "tree"` (الافتراضي)
- `/branch` عندما يكون `doubleEscapeAction = "tree"` (يوجّه إلى محدد الشجرة بدلاً من منتقي فروع المستخدم فقط)

## نموذج واجهة الشجرة

تُعرض الشجرة من مؤشرات إدخالات الجلسة الأصلية (`id` / `parentId`).

- يتم ترتيب العناصر الفرعية حسب الطابع الزمني تصاعديًا (الأقدم أولاً، والأحدث في الأسفل)
- يُحدد الفرع النشط (المسار من الجذر إلى الفرع الحالي) بعلامة نقطة
- تُعرض التسميات (إن وُجدت) كـ `[label]` قبل نص العقدة
- إذا وُجدت جذور متعددة (سلاسل أصلية يتيمة/معطلة)، تُعرض تحت جذر تفريع افتراضي

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

يُعيد المحدد التمركز حول التحديد الحالي ويعرض حتى:

- `max(5, floor(terminalHeight / 2))` صفوف

## اختصارات لوحة المفاتيح داخل محدد الشجرة

- `Up` / `Down`: تحريك التحديد (يلتف دائريًا)
- `Left` / `Right`: صفحة لأعلى / صفحة لأسفل
- `Enter`: اختيار العقدة
- `Esc`: مسح البحث إن كان نشطًا؛ وإلا إغلاق المحدد
- `Ctrl+C`: إغلاق المحدد
- `Type`: إلحاق بنص البحث
- `Backspace`: حذف حرف من البحث
- `Shift+L`: تعديل/مسح التسمية على الإدخال المحدد
- `Ctrl+O`: التنقل بين الفلاتر للأمام
- `Shift+Ctrl+O`: التنقل بين الفلاتر للخلف
- `Alt+D/T/U/L/A`: الانتقال مباشرة إلى وضع فلتر محدد

## دلالات الفلاتر والبحث

أوضاع الفلاتر (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

يعرض معظم عقد المحادثة، لكن يخفي أنواع إدخالات الحفظ الداخلي:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

مثل `default`، بالإضافة إلى إخفاء رسائل `toolResult`.

### `user-only`

إدخالات `message` فقط حيث يكون الدور `user`.

### `labeled-only`

الإدخالات التي تحتوي حاليًا على تسمية محلولة فقط.

### `all`

كل شيء في شجرة الجلسة، بما في ذلك إدخالات الحفظ الداخلي/المخصصة.

### سلوك عقد المساعد التي تحتوي على أدوات فقط

رسائل المساعد التي تحتوي **فقط على استدعاءات أدوات** (بدون نص) تكون مخفية افتراضيًا في جميع طرق العرض المفلترة ما لم:

- تكن الرسالة خطأ/متوقفة (`stopReason` ليس `stop`/`toolUse`)، أو
- تكن الفرع الحالي (يبقى مرئيًا دائمًا)

### سلوك البحث

- يتم تقسيم الاستعلام حسب المسافات
- المطابقة غير حساسة لحالة الأحرف
- يجب أن تتطابق جميع الرموز (دلالات AND)
- يشمل النص القابل للبحث التسمية والدور والمحتوى الخاص بالنوع (نص الرسالة، نص ملخص الفرع، النوع المخصص، مقتطفات أوامر الأدوات، إلخ.)

## نتائج الاختيار (مهم)

يحسب `navigateTree` سلوك الفرع الجديد من نوع الإدخال المحدد:

### اختيار رسالة `user`

- يصبح الفرع الجديد هو `parentId` للإدخال المحدد
- إذا كان الأصل `null` (رسالة مستخدم جذرية)، يُعاد تعيين الفرع إلى الجذر (`resetLeaf()`)
- يتم نسخ نص الرسالة المحددة إلى المحرر للتعديل/إعادة الإرسال

### اختيار `custom_message`

- نفس قاعدة الفرع كرسائل المستخدم (`parentId`)
- يتم استخراج المحتوى النصي ونسخه إلى المحرر

### اختيار عقدة غير مستخدم (مساعد/أداة/ملخص/ضغط/حفظ داخلي مخصص/إلخ.)

- يصبح الفرع الجديد هو معرف العقدة المحددة
- لا يتم ملء المحرر مسبقًا

### اختيار الفرع الحالي

- لا يحدث شيء؛ يُغلق المحدد مع رسالة "Already at this point"

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## تدفق التلخيص عند التبديل

يتم التحكم في طلب الملخص بواسطة `branchSummary.enabled` (الافتراضي: `false`).

عند التفعيل، بعد اختيار عقدة تسأل الواجهة:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

تفاصيل التدفق:

- الضغط على Escape في نافذة الملخص يعيد فتح محدد الشجرة
- إلغاء الطلب المخصص يعود إلى حلقة اختيار الملخص
- أثناء التلخيص، تعرض الواجهة مؤشر تحميل وتربط `Esc` بـ `abortBranchSummary()`
- إذا تم إلغاء التلخيص، يُعاد فتح محدد الشجرة ولا يُطبق أي انتقال

تفاصيل `navigateTree` الداخلية:

- يجمع إدخالات الفرع المهجور من الفرع القديم إلى السلف المشترك
- يُصدر حدث `session_before_tree` (يمكن للإضافات الإلغاء أو حقن ملخص)
- يستخدم الملخص الافتراضي فقط إذا طُلب وكان مطلوبًا
- يُطبق الانتقال باستخدام:
  - `branchWithSummary(...)` عند وجود ملخص
  - `branch(newLeafId)` للانتقال غير الجذري بدون ملخص
  - `resetLeaf()` للانتقال الجذري بدون ملخص
- يستبدل محادثة الوكيل بسياق الجلسة المُعاد بناؤه
- يُصدر حدث `session_tree`

ملاحظة: إذا طلب المستخدم ملخصًا ولم يكن هناك شيء لتلخيصه، يستمر التنقل بدون إنشاء إدخال ملخص.

## التسميات

تعديلات التسميات في واجهة الشجرة تستدعي `appendLabelChange(targetId, label)`.

- التسمية غير الفارغة تُعيّن/تُحدّث التسمية المحلولة
- التسمية الفارغة تمسحها
- تُخزن التسميات كإدخالات `label` إلحاقية فقط
- تعرض عقد الشجرة حالة التسمية المحلولة، وليس سجل إدخالات التسميات الخام

## أمر `/tree` مقارنة بالعمليات المجاورة

| العملية | النطاق | النتيجة |
|---|---|---|
| `/tree` | ملف الجلسة الحالي | ينقل الفرع إلى النقطة المحددة (نفس الملف) |
| `/branch` | عادةً ملف الجلسة الحالي -> ملف جلسة جديد | افتراضيًا يُفرّع من رسالة **مستخدم** محددة إلى ملف جلسة جديد؛ إذا كان `doubleEscapeAction = "tree"`، يفتح `/branch` واجهة تنقل الشجرة بدلاً من ذلك |
| `/fork` | الجلسة الحالية بأكملها | يُكرر الجلسة في ملف جلسة محفوظ جديد |
| `/resume` | قائمة الجلسات | يتحول إلى ملف جلسة آخر |

الفرق الرئيسي: `/tree` هو أداة تنقل/إعادة تموضع داخل ملف جلسة واحد. بينما `/branch` و `/fork` و `/resume` جميعها تُغيّر سياق ملف الجلسة.

## سير عمل المشغّل

### إعادة التشغيل من رسالة مستخدم سابقة دون فقدان الفرع الحالي

1. `/tree`
2. البحث/اختيار رسالة مستخدم سابقة
3. اختيار `No summary` (أو التلخيص إذا لزم الأمر)
4. تعديل النص المملوء مسبقًا في المحرر
5. الإرسال

النتيجة: ينمو فرع جديد من النقطة المحددة داخل نفس ملف الجلسة.

### مغادرة الفرع الحالي مع علامة سياق مرجعية

1. تفعيل `branchSummary.enabled`
2. `/tree` واختيار العقدة المستهدفة
3. اختيار `Summarize` (أو طلب مخصص)

النتيجة: يُلحق إدخال `branch_summary` في الموضع المستهدف قبل المتابعة.

### فحص إدخالات الحفظ الداخلي المخفية

1. `/tree`
2. الضغط على `Alt+A` (الكل)
3. البحث عن `model` أو `thinking` أو `custom` أو التسميات

النتيجة: فحص الخط الزمني الداخلي الكامل، وليس فقط عقد المحادثة.

### وضع إشارات مرجعية لنقاط محورية للانتقال لاحقًا

1. `/tree`
2. الانتقال إلى الإدخال
3. `Shift+L` وتعيين تسمية
4. لاحقًا استخدام `Alt+L` (`labeled-only`) للانتقال بسرعة

النتيجة: تنقل سريع بين المعالم الدائمة للفروع.
