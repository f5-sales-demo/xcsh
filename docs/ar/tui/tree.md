# مرجع أمر `/tree`

يفتح أمر `/tree` متصفح **شجرة الجلسة** التفاعلي. يتيح لك الانتقال إلى أي إدخال في ملف الجلسة الحالي والمتابعة من تلك النقطة.

هذا انتقال بين الأوراق داخل الملف، وليس تصدير جلسة جديدة.

## ما يفعله `/tree`

- يبني شجرة من إدخالات الجلسة الحالية (`SessionManager.getTree()`)
- يفتح `TreeSelectorComponent` مع التنقل بلوحة المفاتيح والفلاتر والبحث
- عند الاختيار، يستدعي `AgentSession.navigateTree(targetId, { summarize, customInstructions })`
- يعيد بناء المحادثة المرئية من مسار الورقة الجديد
- يملأ مسبقاً نص المحرر اختيارياً عند اختيار رسالة مستخدم/مخصصة

التنفيذ الأساسي:

- `src/modes/controllers/input-controller.ts` (`/tree`، ربط اختصارات لوحة المفاتيح، سلوك الضغط المزدوج على Escape)
- `src/modes/controllers/selector-controller.ts` (تشغيل واجهة الشجرة + تدفق طلب الملخص)
- `src/modes/components/tree-selector.ts` (التنقل، الفلاتر، البحث، التسميات، العرض)
- `src/session/agent-session.ts` (`navigateTree` تبديل الأوراق + ملخص اختياري)
- `src/session/session-manager.ts` (`getTree`، `branch`، `branchWithSummary`، `resetLeaf`، استمرار التسميات)

## كيفية فتحه

أي مما يلي يفتح نفس المُحدِّد:

- `/tree`
- إجراء اختصار لوحة المفاتيح المُعدّ `tree`
- الضغط المزدوج على Escape في محرر فارغ عندما يكون `doubleEscapeAction = "tree"` (الافتراضي)
- `/branch` عندما يكون `doubleEscapeAction = "tree"` (يوجّه إلى مُحدِّد الشجرة بدلاً من منتقي الفروع الخاص بالمستخدم فقط)

## نموذج واجهة الشجرة

تُعرض الشجرة من مؤشرات الإدخال الأصلية للجلسة (`id` / `parentId`).

- يتم ترتيب العناصر الفرعية تصاعدياً حسب الطابع الزمني (الأقدم أولاً، الأحدث أسفل)
- يُعلَّم الفرع النشط (المسار من الجذر إلى الورقة الحالية) بنقطة
- تُعرض التسميات (إن وُجدت) كـ `[label]` قبل نص العقدة
- إذا وُجدت جذور متعددة (سلاسل أصلية يتيمة/مكسورة)، تُعرض تحت جذر تفريع افتراضي

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

يعيد المُحدِّد التمركز حول الاختيار الحالي ويعرض حتى:

- `max(5, floor(terminalHeight / 2))` صف

## اختصارات لوحة المفاتيح داخل مُحدِّد الشجرة

- `Up` / `Down`: تحريك الاختيار (يلتف)
- `Left` / `Right`: صفحة للأعلى / صفحة للأسفل
- `Enter`: اختيار العقدة
- `Esc`: مسح البحث إذا كان نشطاً؛ وإلا إغلاق المُحدِّد
- `Ctrl+C`: إغلاق المُحدِّد
- `Type`: إلحاق باستعلام البحث
- `Backspace`: حذف حرف من البحث
- `Shift+L`: تحرير/مسح التسمية على الإدخال المحدد
- `Ctrl+O`: تبديل الفلتر للأمام
- `Shift+Ctrl+O`: تبديل الفلتر للخلف
- `Alt+D/T/U/L/A`: الانتقال مباشرة إلى وضع فلتر محدد

## دلالات الفلاتر والبحث

أوضاع الفلتر (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

يعرض معظم عقد المحادثة، لكنه يخفي أنواع إدخالات المسك الدفتري:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

مثل `default`، بالإضافة إلى إخفاء رسائل `toolResult`.

### `user-only`

فقط إدخالات `message` حيث الدور هو `user`.

### `labeled-only`

فقط الإدخالات التي تحلّ حالياً إلى تسمية.

### `all`

كل شيء في شجرة الجلسة، بما في ذلك إدخالات المسك الدفتري/المخصصة.

### سلوك عقد المساعد التي تحتوي على أدوات فقط

رسائل المساعد التي تحتوي **فقط على استدعاءات أدوات** (بدون نص) مخفية افتراضياً في جميع العروض المفلترة ما لم:

- تكن الرسالة خطأ/متوقفة (`stopReason` ليس `stop`/`toolUse`)، أو
- تكن الورقة الحالية (تبقى مرئية دائماً)

### سلوك البحث

- يُقسَّم الاستعلام إلى رموز بالمسافات
- المطابقة غير حساسة لحالة الأحرف
- يجب أن تتطابق جميع الرموز (دلالات AND)
- النص القابل للبحث يشمل التسمية والدور والمحتوى الخاص بالنوع (نص الرسالة، نص ملخص الفرع، النوع المخصص، مقتطفات أوامر الأدوات، إلخ.)

## نتائج الاختيار (مهم)

يحسب `navigateTree` سلوك الورقة الجديدة من نوع الإدخال المحدد:

### اختيار رسالة `user`

- تصبح الورقة الجديدة `parentId` الخاص بالإدخال المحدد
- إذا كان الأصل `null` (رسالة مستخدم جذرية)، تُعاد الورقة إلى الجذر (`resetLeaf()`)
- يُنسخ نص الرسالة المحددة إلى المحرر للتعديل/إعادة الإرسال

### اختيار `custom_message`

- نفس قاعدة الورقة كرسائل المستخدم (`parentId`)
- يُستخرج المحتوى النصي ويُنسخ إلى المحرر

### اختيار عقدة غير مستخدم (مساعد/أداة/ملخص/ضغط/مسك دفتري مخصص/إلخ.)

- تصبح الورقة الجديدة معرّف العقدة المحددة
- لا يُملأ المحرر مسبقاً

### اختيار الورقة الحالية

- لا يحدث شيء؛ يُغلق المُحدِّد مع رسالة "Already at this point"

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## تدفق الملخص عند التبديل

يُتحكَّم في طلب الملخص بواسطة `branchSummary.enabled` (الافتراضي: `false`).

عند التفعيل، بعد اختيار عقدة تسأل الواجهة:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

تفاصيل التدفق:

- الضغط على Escape في طلب الملخص يعيد فتح مُحدِّد الشجرة
- إلغاء الطلب المخصص يعود إلى حلقة اختيار الملخص
- أثناء التلخيص، تعرض الواجهة مُحمِّلاً وتربط `Esc` بـ `abortBranchSummary()`
- إذا تم إلغاء التلخيص، يُعاد فتح مُحدِّد الشجرة ولا يُطبَّق أي انتقال

تفاصيل `navigateTree` الداخلية:

- يجمع إدخالات الفرع المهجور من الورقة القديمة إلى السلف المشترك
- يُصدر `session_before_tree` (يمكن للإضافات الإلغاء أو حقن ملخص)
- يستخدم المُلخِّص الافتراضي فقط إذا طُلب وكان ضرورياً
- يُطبِّق الانتقال باستخدام:
  - `branchWithSummary(...)` عندما يوجد ملخص
  - `branch(newLeafId)` للانتقال غير الجذري بدون ملخص
  - `resetLeaf()` للانتقال الجذري بدون ملخص
- يستبدل محادثة الوكيل بسياق الجلسة المُعاد بناؤه
- يُصدر `session_tree`

ملاحظة: إذا طلب المستخدم ملخصاً ولكن لا يوجد شيء لتلخيصه، يستمر التنقل دون إنشاء إدخال ملخص.

## التسميات

تستدعي تعديلات التسميات في واجهة الشجرة `appendLabelChange(targetId, label)`.

- التسمية غير الفارغة تُعيّن/تُحدّث التسمية المحلولة
- التسمية الفارغة تمسحها
- تُخزَّن التسميات كإدخالات `label` إلحاقية فقط
- تعرض عقد الشجرة حالة التسمية المحلولة، وليس سجل إدخالات التسميات الخام

## `/tree` مقابل العمليات المجاورة

| العملية | النطاق | النتيجة |
|---|---|---|
| `/tree` | ملف الجلسة الحالي | ينقل الورقة إلى النقطة المحددة (نفس الملف) |
| `/branch` | عادة ملف الجلسة الحالي -> ملف جلسة جديد | افتراضياً يتفرع من رسالة **المستخدم** المحددة إلى ملف جلسة جديد؛ إذا كان `doubleEscapeAction = "tree"`، يفتح `/branch` واجهة تنقل الشجرة بدلاً من ذلك |
| `/fork` | الجلسة الحالية بالكامل | ينسخ الجلسة إلى ملف جلسة محفوظ جديد |
| `/resume` | قائمة الجلسات | يتبدل إلى ملف جلسة آخر |

التمييز الرئيسي: `/tree` هو أداة تنقل/إعادة تموضع داخل ملف جلسة واحد. أما `/branch` و`/fork` و`/resume` فجميعها تُغيّر سياق ملف الجلسة.

## سير عمل المشغّل

### إعادة التشغيل من طلب مستخدم سابق دون فقدان الفرع الحالي

1. `/tree`
2. البحث عن/اختيار رسالة مستخدم سابقة
3. اختيار `No summary` (أو التلخيص إذا لزم الأمر)
4. تعديل النص المُملأ مسبقاً في المحرر
5. إرسال

الأثر: ينمو فرع جديد من النقطة المحددة داخل نفس ملف الجلسة.

### مغادرة الفرع الحالي مع علامة سياقية

1. تفعيل `branchSummary.enabled`
2. `/tree` واختيار العقدة المستهدفة
3. اختيار `Summarize` (أو طلب مخصص)

الأثر: يُلحق إدخال `branch_summary` عند الموضع المستهدف قبل المتابعة.

### فحص إدخالات المسك الدفتري المخفية

1. `/tree`
2. الضغط على `Alt+A` (الكل)
3. البحث عن `model` أو `thinking` أو `custom` أو التسميات

الأثر: فحص الجدول الزمني الداخلي الكامل، وليس فقط عقد المحادثة.

### وضع إشارات مرجعية لنقاط محورية للانتقال لاحقاً

1. `/tree`
2. الانتقال إلى الإدخال
3. `Shift+L` وتعيين تسمية
4. لاحقاً استخدام `Alt+L` (`labeled-only`) للانتقال بسرعة

الأثر: تنقل سريع بين معالم الفروع الدائمة.
