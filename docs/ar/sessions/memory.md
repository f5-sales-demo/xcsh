# الذاكرة المستقلة

عند التفعيل، يستخرج الوكيل تلقائيًا المعرفة الدائمة من الجلسات السابقة ويحقن ملخصًا مختصرًا في كل جلسة جديدة. وبمرور الوقت يبني مخزن ذاكرة على مستوى المشروع — القرارات التقنية، وسير العمل المتكرر، والمشكلات المعروفة — ينتقل تلقائيًا دون جهد يدوي.

معطّل افتراضيًا. يمكن تفعيله عبر `/settings` أو `config.yml`:

```yaml
memories:
  enabled: true
```

## الاستخدام

### ما الذي يتم حقنه

عند بدء الجلسة، إذا كان ملخص ذاكرة موجودًا للمشروع الحالي، يتم حقنه في موجّه النظام ككتلة **إرشادات الذاكرة**. يتم توجيه الوكيل إلى:

- التعامل مع الذاكرة كسياق استدلالي — مفيد للعمليات والقرارات السابقة، لكنه ليس مرجعيًا بشأن حالة المستودع الحالية.
- الإشارة إلى مسار عنصر الذاكرة عندما تغيّر الذاكرة الخطة، ومقارنتها بأدلة المستودع الحالي قبل التنفيذ.
- تفضيل حالة المستودع وتعليمات المستخدم عند تعارضها مع الذاكرة؛ والتعامل مع الذاكرة المتعارضة باعتبارها قديمة.

### قراءة عناصر الذاكرة

يمكن للوكيل قراءة ملفات الذاكرة مباشرة باستخدام عناوين `memory://` مع أداة `read`:

| العنوان | المحتوى |
|---|---|
| `memory://root` | الملخص المختصر المحقون عند بدء التشغيل |
| `memory://root/MEMORY.md` | مستند الذاكرة طويلة المدى الكامل |
| `memory://root/skills/<name>/SKILL.md` | دليل إجرائي لمهارة مُولَّدة |

### أمر `/memory` المختصر

| الأمر الفرعي | التأثير |
|---|---|
| `view` | عرض حمولة حقن الذاكرة الحالية |
| `clear` / `reset` | حذف جميع بيانات الذاكرة والعناصر المُولَّدة |
| `enqueue` / `rebuild` | فرض تشغيل التوحيد عند بدء التشغيل التالي |

## كيف يعمل

تُبنى الذاكرات بواسطة خط معالجة في الخلفية يعمل عند بدء التشغيل أو يُفعَّل يدويًا عبر الأمر المختصر.

**المرحلة الأولى — الاستخراج لكل جلسة:** لكل جلسة سابقة تغيرت منذ آخر معالجة لها، يقرأ نموذج سجل الجلسة ويستخرج الإشارات الدائمة: القرارات التقنية، والقيود، والأعطال المحلولة، وسير العمل المتكرر. يتم تخطي الجلسات الحديثة جدًا أو القديمة جدًا أو النشطة حاليًا. ينتج كل استخراج كتلة ذاكرة خام وملخصًا قصيرًا لتلك الجلسة.

**المرحلة الثانية — التوحيد:** بعد الاستخراج، تقرأ جولة نموذج ثانية جميع الاستخراجات لكل جلسة وتنتج ثلاثة مخرجات تُكتب على القرص:

- `MEMORY.md` — مستند ذاكرة طويلة المدى مُنسَّق
- `memory_summary.md` — النص المختصر المحقون عند بدء الجلسة
- `skills/` — أدلة إجرائية قابلة لإعادة الاستخدام، كل منها في دليل فرعي خاص به

تستخدم المرحلة الثانية قفلًا (lease) لمنع التشغيل المزدوج عند بدء عمليات متعددة في وقت واحد. تُحذف أدلة المهارات القديمة من عمليات التشغيل السابقة تلقائيًا.

تُفحص جميع المخرجات بحثًا عن أسرار قبل كتابتها على القرص.

### سلوك الاستخراج

يُدار سلوك استخراج الذاكرة والتوحيد بالكامل بواسطة ملفات الموجّهات الثابتة في `src/prompts/memories/`.

| الملف | الغرض | المتغيرات |
|---|---|---|
| `stage_one_system.md` | موجّه النظام للاستخراج لكل جلسة | — |
| `stage_one_input.md` | قالب دور المستخدم الذي يغلف محتوى الجلسة | `{{thread_id}}`، `{{response_items_json}}` |
| `consolidation.md` | موجّه التوحيد عبر الجلسات | `{{raw_memories}}`، `{{rollout_summaries}}` |
| `read_path.md` | إرشادات الذاكرة المحقونة في الجلسات الحية | `{{memory_summary}}` |

### اختيار النموذج

تعتمد الذاكرة على نظام أدوار النماذج.

| المرحلة | الدور | الغرض |
|---|---|---|
| المرحلة الأولى (الاستخراج) | `default` | استخراج المعرفة لكل جلسة |
| المرحلة الثانية (التوحيد) | `smol` | التوليف عبر الجلسات |

إذا لم يكن `smol` مُهيّأً، تعود المرحلة الثانية إلى الدور `default`.

## الإعدادات

| الإعداد | القيمة الافتراضية | الوصف |
|---|---|---|
| `memories.enabled` | `false` | مفتاح التشغيل الرئيسي |
| `memories.maxRolloutAgeDays` | `30` | لا تُعالج الجلسات الأقدم من هذه المدة |
| `memories.minRolloutIdleHours` | `12` | تُتخطى الجلسات النشطة مؤخرًا أقل من هذه المدة |
| `memories.maxRolloutsPerStartup` | `64` | الحد الأقصى للجلسات المعالجة في بدء تشغيل واحد |
| `memories.summaryInjectionTokenLimit` | `5000` | الحد الأقصى لرموز الملخص المحقون في موجّه النظام |

تتوفر أدوات ضبط إضافية (التزامن، مدد الأقفال، ميزانيات الرموز) في الإعدادات للاستخدام المتقدم.

## الملفات الرئيسية

- `src/memories/index.ts` — تنسيق خط المعالجة، والحقن، ومعالجة الأوامر المختصرة
- `src/memories/storage.ts` — طابور المهام وسجل السلاسل المدعوم بـ SQLite
- `src/prompts/memories/` — قوالب موجّهات الذاكرة
- `src/internal-urls/memory-protocol.ts` — معالج عناوين `memory://`
