# المهارات

المهارات هي حزم قدرات مدعومة بملفات يتم اكتشافها عند بدء التشغيل وتُعرض للنموذج على شكل:

- بيانات وصفية خفيفة في موجه النظام (الاسم + الوصف)
- محتوى عند الطلب عبر `read skill://...`
- أوامر تفاعلية اختيارية `/skill:<name>`

يغطي هذا المستند سلوك وقت التشغيل الحالي في `src/extensibility/skills.ts` و `src/discovery/builtin.ts` و `src/internal-urls/skill-protocol.ts` و `src/discovery/agents-md.ts`.

## ما هي المهارة في قاعدة الشيفرة هذه

تُمثَّل المهارة المكتشفة على النحو التالي:

- `name`
- `description`
- `filePath` (مسار `SKILL.md`)
- `baseDir` (دليل المهارة)
- البيانات الوصفية للمصدر (`provider`، `level`، المسار)

يتطلب وقت التشغيل فقط `name` و `path` للصلاحية. عملياً، تعتمد جودة المطابقة على أن يكون `description` ذا معنى.

## التخطيط المطلوب وتوقعات SKILL.md

### تخطيط الدليل

بالنسبة للاكتشاف المبني على المزودين (مزودو native/Claude/Codex/Agents/plugin)، يتم اكتشاف المهارات **بمستوى واحد تحت `skills/`**:

- `<skills-root>/<skill-name>/SKILL.md`

الأنماط المتداخلة مثل `<skills-root>/group/<skill>/SKILL.md` لا يتم اكتشافها بواسطة محمّلات المزودين.

بالنسبة لـ `skills.customDirectories`، يستخدم المسح نفس التخطيط غير التعاودي (`*/SKILL.md`).

```text
Provider-discovered layout (non-recursive under skills/):

<root>/skills/
  ├─ postgres/
  │   └─ SKILL.md      ✅ discovered
  ├─ pdf/
  │   └─ SKILL.md      ✅ discovered
  └─ team/
      └─ internal/
          └─ SKILL.md  ❌ not discovered by provider loaders

Custom-directory scanning is also non-recursive, so nested paths are ignored unless you point `customDirectories` at that nested parent.
```

### البيانات الأمامية لـ `SKILL.md`

حقول البيانات الأمامية المدعومة في نوع المهارة:

- `name?: string`
- `description?: string`
- `globs?: string[]`
- `alwaysApply?: boolean`
- المفاتيح الإضافية تُحفظ كبيانات وصفية غير معروفة

سلوك وقت التشغيل الحالي:

- `name` يُعيَّن افتراضياً إلى اسم دليل المهارة
- `description` مطلوب من أجل:
  - اكتشاف مهارات مزود `.xcsh` الأصلي (`requireDescription: true`)
  - عمليات مسح `skills.customDirectories` عبر `scanSkillsFromDir` في `src/discovery/helpers.ts` (غير تعاودي)
- المزودون غير الأصليين يمكنهم تحميل المهارات بدون وصف

## خط أنابيب الاكتشاف

تقوم `discoverSkills()` في `src/extensibility/skills.ts` بتمريرتين:

1. **مزودو القدرات** عبر `loadCapability("skills")`
2. **الأدلة المخصصة** عبر `scanSkillsFromDir(..., { requireDescription: true })` (تعداد أدلة بمستوى واحد)

إذا كانت `skills.enabled` تساوي `false`، يُرجع الاكتشاف مهارات فارغة.

### مزودو المهارات المدمجون والأولوية

ترتيب المزودين يعتمد على الأولوية أولاً (الأعلى يفوز)، ثم ترتيب التسجيل في حالة التساوي.

مزودو المهارات المسجلون حالياً:

1. `native` (الأولوية 100) — مهارات المستخدم/المشروع `.xcsh` عبر `src/discovery/builtin.ts`
2. `claude` (الأولوية 80)
3. مجموعة الأولوية 70 (بترتيب التسجيل):
   - `claude-plugins`
   - `agents`
   - `codex`

مفتاح إزالة التكرار هو اسم المهارة. العنصر الأول بالاسم المعطى يفوز.

### مفاتيح التبديل للمصدر والتصفية

تطبق `discoverSkills()` هذه الضوابط:

- مفاتيح تبديل المصدر: `enableCodexUser`، `enableClaudeUser`، `enableClaudeProject`، `enablePiUser`، `enablePiProject`
- مرشحات glob على اسم المهارة:
  - `ignoredSkills` (استبعاد)
  - `includeSkills` (قائمة السماح بالتضمين؛ فارغة تعني تضمين الكل)

ترتيب التصفية هو:

1. المصدر مُفعَّل
2. غير مُتجاهَل
3. مُضمَّن (إذا كانت قائمة التضمين موجودة)

بالنسبة للمزودين بخلاف codex/claude/native (مثل `agents`، `claude-plugins`)، يعود التفعيل حالياً إلى: مُفعَّل إذا كان **أي** مفتاح تبديل مصدر مدمج مُفعَّلاً.

### التعامل مع التصادمات والتكرارات

- إزالة تكرار القدرات تحتفظ بالفعل بأول مهارة لكل اسم (المزود ذو الأولوية الأعلى)
- `extensibility/skills.ts` بالإضافة إلى ذلك:
  - يزيل تكرار الملفات المتطابقة عبر `realpath` (آمن مع الروابط الرمزية)
  - يصدر تحذيرات تصادم عندما يتعارض اسم مهارة لاحق
  - يحتفظ بواجهة `discoverSkillsFromDir({ dir, source })` المريحة كمحول رقيق فوق `scanSkillsFromDir`
- مهارات الأدلة المخصصة تُدمج بعد مهارات المزودين وتتبع نفس سلوك التصادم

## سلوك الاستخدام في وقت التشغيل

### العرض في موجه النظام

يستخدم بناء موجه النظام (`src/system-prompt.ts`) المهارات المكتشفة كالتالي:

- إذا كانت أداة `read` متاحة:
  - تضمين قائمة المهارات المكتشفة في الموجه
- وإلا:
  - حذف القائمة المكتشفة

الوكلاء الفرعيون لأداة المهام يتلقون قائمة المهارات المكتشفة/المقدمة للجلسة عبر إنشاء الجلسة العادي؛ لا يوجد تجاوز لتثبيت المهارات لكل مهمة.

### أوامر `/skill:<name>` التفاعلية

إذا كانت `skills.enableSkillCommands` تساوي true، يسجل الوضع التفاعلي أمر شرطة مائلة واحداً لكل مهارة مكتشفة.

سلوك `/skill:<name> [args]`:

- يقرأ ملف المهارة مباشرة من `filePath`
- يزيل البيانات الأمامية
- يحقن جسم المهارة كرسالة مخصصة لاحقة
- يُلحق البيانات الوصفية (`Skill: <path>`، اختيارياً `User: <args>`)

## سلوك عنوان URL `skill://`

يدعم `src/internal-urls/skill-protocol.ts`:

- `skill://<name>` → يُحلَّل إلى `SKILL.md` الخاص بتلك المهارة
- `skill://<name>/<relative-path>` → يُحلَّل داخل دليل تلك المهارة

```text
skill:// URL resolution

skill://pdf
  -> <pdf-base>/SKILL.md

skill://pdf/references/tables.md
  -> <pdf-base>/references/tables.md

Guards:
- reject absolute paths
- reject `..` traversal
- reject any resolved path escaping <pdf-base>
```

تفاصيل التحليل:

- يجب أن يتطابق اسم المهارة تماماً
- المسارات النسبية تُفك ترميزها من URL
- المسارات المطلقة مرفوضة
- اجتياز المسار (`..`) مرفوض
- المسار المُحلَّل يجب أن يبقى ضمن `baseDir`
- الملفات المفقودة تُرجع خطأ صريح `File not found`

نوع المحتوى:

- `.md` => `text/markdown`
- كل شيء آخر => `text/plain`

لا يتم إجراء بحث احتياطي للأصول المفقودة.

## المهارات مقابل AGENTS.md والأوامر والأدوات والخطافات

### المهارات مقابل AGENTS.md

- **المهارات**: حزم قدرات مسماة واختيارية يتم اختيارها حسب سياق المهمة أو تُطلب صراحة
- **AGENTS.md/ملفات السياق**: ملفات تعليمات دائمة تُحمَّل كقدرة ملف سياق وتُدمج وفقاً لقواعد المستوى/العمق

يمسح `src/discovery/agents-md.ts` تحديداً الأدلة الأم من `cwd` لاكتشاف ملفات `AGENTS.md` المستقلة (حتى عمق 20)، مع استبعاد أجزاء الأدلة المخفية.

### المهارات مقابل أوامر الشرطة المائلة

- **المهارات**: محتوى معرفة/سير عمل قابل للقراءة بواسطة النموذج
- **أوامر الشرطة المائلة**: نقاط دخول أوامر يستدعيها المستخدم
- `/skill:<name>` هو غلاف مريح يحقن نص المهارة؛ ولا يغير دلالات اكتشاف المهارات

### المهارات مقابل الأدوات المخصصة

- **المهارات**: محتوى توثيق/سير عمل يُحمَّل عبر سياق الموجه و `read`
- **الأدوات المخصصة**: واجهات أدوات قابلة للتنفيذ يمكن للنموذج استدعاؤها مع مخططات وتأثيرات جانبية في وقت التشغيل

### المهارات مقابل الخطافات

- **المهارات**: محتوى سلبي
- **الخطافات**: معترضات مدفوعة بالأحداث في وقت التشغيل يمكنها حظر/تعديل السلوك أثناء التنفيذ

## إرشادات التأليف العملية المرتبطة بمنطق الاكتشاف

- ضع كل مهارة في دليلها الخاص: `<skills-root>/<skill-name>/SKILL.md`
- أضف دائماً بيانات أمامية صريحة لـ `name` و `description`
- احتفظ بالأصول المرجعية تحت نفس دليل المهارة واستخدم `skill://<name>/...` للوصول إليها
- للتصنيف المتداخل (`team/domain/skill`)، وجّه `skills.customDirectories` إلى الدليل الأم المتداخل؛ المسح نفسه يبقى غير تعاودي
- تجنب تكرار أسماء المهارات عبر المصادر؛ أول تطابق يفوز حسب أولوية المزود
