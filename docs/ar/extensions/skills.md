# المهارات

المهارات هي حزم قدرات مدعومة بملفات يتم اكتشافها عند بدء التشغيل وتُعرض للنموذج على شكل:

- بيانات وصفية خفيفة في موجه النظام (الاسم + الوصف)
- محتوى عند الطلب عبر `read skill://...`
- أوامر تفاعلية اختيارية `/skill:<name>`

يغطي هذا المستند سلوك وقت التشغيل الحالي في `src/extensibility/skills.ts`، و`src/discovery/builtin.ts`، و`src/internal-urls/skill-protocol.ts`، و`src/discovery/agents-md.ts`.

## ما هي المهارة في قاعدة الشيفرة هذه

تُمثَّل المهارة المكتشفة على النحو التالي:

- `name`
- `description`
- `filePath` (مسار `SKILL.md`)
- `baseDir` (دليل المهارة)
- بيانات وصفية للمصدر (`provider`، `level`، المسار)

يتطلب وقت التشغيل فقط `name` و`path` للصلاحية. عملياً، تعتمد جودة المطابقة على أن يكون `description` ذا معنى.

## التخطيط المطلوب وتوقعات SKILL.md

### تخطيط الدليل

بالنسبة للاكتشاف القائم على الموفر (موفرات native/Claude/Codex/Agents/plugin)، يتم اكتشاف المهارات **بمستوى واحد تحت `skills/`**:

- `<skills-root>/<skill-name>/SKILL.md`

الأنماط المتداخلة مثل `<skills-root>/group/<skill>/SKILL.md` لا يتم اكتشافها بواسطة محمّلات الموفر.

بالنسبة لـ `skills.customDirectories`، يستخدم المسح نفس التخطيط غير التكراري (`*/SKILL.md`).

```text
Provider-discovered layout (non-recursive under skills/):

<root>/skills/
  ├─ postgres/
  │   └─ SKILL.md      ✅ discovered
  ├─ pdf/
  │   └─ SKILL.md      ✅ discovered
  └─ team/
      └─ internal/
          └─ SKILL.md  ❌ not discovered by provider loaders

Custom-directory scanning is also non-recursive, so nested paths are ignored unless you point `customDirectories` at that nested parent.
```

### البيانات الأولية لـ `SKILL.md`

حقول البيانات الأولية المدعومة في نوع المهارة:

- `name?: string`
- `description?: string`
- `globs?: string[]`
- `alwaysApply?: boolean`
- المفاتيح الإضافية تُحفظ كبيانات وصفية غير معروفة

سلوك وقت التشغيل الحالي:

- `name` يتم تعيينه افتراضياً إلى اسم دليل المهارة
- `description` مطلوب لـ:
  - اكتشاف مهارات موفر `.xcsh` الأصلي (`requireDescription: true`)
  - مسح `skills.customDirectories` عبر `scanSkillsFromDir` في `src/discovery/helpers.ts` (غير تكراري)
- الموفرات غير الأصلية يمكنها تحميل المهارات بدون وصف

## خط أنابيب الاكتشاف

`discoverSkills()` في `src/extensibility/skills.ts` يقوم بتمريرتين:

1. **موفرات القدرات** عبر `loadCapability("skills")`
2. **الأدلة المخصصة** عبر `scanSkillsFromDir(..., { requireDescription: true })` (تعداد الدليل بمستوى واحد)

إذا كان `skills.enabled` يساوي `false`، فإن الاكتشاف لا يُرجع أي مهارات.

### موفرات المهارات المدمجة والأسبقية

ترتيب الموفرات يكون بالأولوية أولاً (الأعلى يفوز)، ثم ترتيب التسجيل في حالة التساوي.

موفرات المهارات المسجلون حالياً:

1. `native` (الأولوية 100) — مهارات المستخدم/المشروع `.xcsh` عبر `src/discovery/builtin.ts`
2. `claude` (الأولوية 80)
3. مجموعة الأولوية 70 (بترتيب التسجيل):
   - `claude-plugins`
   - `agents`
   - `codex`

مفتاح إزالة التكرار هو اسم المهارة. العنصر الأول بالاسم المحدد يفوز.

### مفاتيح التبديل للمصدر والتصفية

`discoverSkills()` يطبق عناصر التحكم التالية:

- مفاتيح تبديل المصدر: `enableCodexUser`، `enableClaudeUser`، `enableClaudeProject`، `enablePiUser`، `enablePiProject`
- مرشحات glob على اسم المهارة:
  - `ignoredSkills` (استبعاد)
  - `includeSkills` (قائمة السماح بالتضمين؛ فارغة تعني تضمين الكل)

ترتيب التصفية هو:

1. المصدر مُفعّل
2. غير مُتجاهَل
3. مُضمَّن (إذا كانت قائمة التضمين موجودة)

بالنسبة للموفرات بخلاف codex/claude/native (مثل `agents`، `claude-plugins`)، يعود التمكين حالياً إلى: مُفعّل إذا كان **أي** مفتاح تبديل مصدر مدمج مُفعّلاً.

### التعامل مع التصادمات والتكرارات

- إزالة تكرار القدرات تحتفظ بالفعل بأول مهارة لكل اسم (الموفر الأعلى أسبقية)
- `extensibility/skills.ts` يقوم بالإضافة:
  - إزالة تكرار الملفات المتطابقة بواسطة `realpath` (آمن مع الروابط الرمزية)
  - إصدار تحذيرات التصادم عندما يتعارض اسم مهارة لاحق
  - الاحتفاظ بواجهة `discoverSkillsFromDir({ dir, source })` المريحة كمحول خفيف فوق `scanSkillsFromDir`
- مهارات الأدلة المخصصة تُدمج بعد مهارات الموفر وتتبع نفس سلوك التصادم

## سلوك الاستخدام في وقت التشغيل

### العرض في موجه النظام

بناء موجه النظام (`src/system-prompt.ts`) يستخدم المهارات المكتشفة كالتالي:

- إذا كانت أداة `read` متاحة:
  - تضمين قائمة المهارات المكتشفة في الموجه
- وإلا:
  - حذف القائمة المكتشفة

الوكلاء الفرعيون لأداة المهام يتلقون قائمة المهارات المكتشفة/المقدمة للجلسة عبر إنشاء الجلسة العادي؛ لا يوجد تجاوز لتثبيت المهارات لكل مهمة.

### أوامر `/skill:<name>` التفاعلية

إذا كان `skills.enableSkillCommands` يساوي true، يسجل الوضع التفاعلي أمر شرطة مائلة واحد لكل مهارة مكتشفة.

سلوك `/skill:<name> [args]`:

- يقرأ ملف المهارة مباشرة من `filePath`
- يزيل البيانات الأولية
- يحقن محتوى المهارة كرسالة مخصصة لاحقة
- يُلحق البيانات الوصفية (`Skill: <path>`، واختيارياً `User: <args>`)

## سلوك عنوان URL `skill://`

`src/internal-urls/skill-protocol.ts` يدعم:

- `skill://<name>` → يُحل إلى `SKILL.md` الخاص بتلك المهارة
- `skill://<name>/<relative-path>` → يُحل داخل دليل تلك المهارة

```text
skill:// URL resolution

skill://pdf
  -> <pdf-base>/SKILL.md

skill://pdf/references/tables.md
  -> <pdf-base>/references/tables.md

Guards:
- reject absolute paths
- reject `..` traversal
- reject any resolved path escaping <pdf-base>
```

تفاصيل الحل:

- اسم المهارة يجب أن يتطابق تماماً
- المسارات النسبية يتم فك ترميز URL لها
- المسارات المطلقة مرفوضة
- اجتياز المسار (`..`) مرفوض
- المسار المُحلّ يجب أن يبقى ضمن `baseDir`
- الملفات المفقودة تُرجع خطأ `File not found` صريح

نوع المحتوى:

- `.md` => `text/markdown`
- كل شيء آخر => `text/plain`

لا يتم إجراء بحث احتياطي عن الأصول المفقودة.

## المهارات مقابل AGENTS.md والأوامر والأدوات والخطافات

### المهارات مقابل AGENTS.md

- **المهارات**: حزم قدرات مسماة واختيارية يتم اختيارها حسب سياق المهمة أو يُطلب صراحةً
- **AGENTS.md/ملفات السياق**: ملفات تعليمات دائمة يتم تحميلها كقدرة ملف سياق ودمجها بقواعد المستوى/العمق

`src/discovery/agents-md.ts` يتنقل تحديداً في الأدلة الأصلية من `cwd` لاكتشاف ملفات `AGENTS.md` المستقلة (حتى عمق 20)، مع استبعاد أجزاء الأدلة المخفية.

### المهارات مقابل أوامر الشرطة المائلة

- **المهارات**: محتوى معرفي/سير عمل قابل للقراءة بواسطة النموذج
- **أوامر الشرطة المائلة**: نقاط دخول أوامر يستدعيها المستخدم
- `/skill:<name>` هو غلاف مريح يحقن نص المهارة؛ لا يغير دلالات اكتشاف المهارات

### المهارات مقابل الأدوات المخصصة

- **المهارات**: محتوى توثيق/سير عمل يُحمّل عبر سياق الموجه و`read`
- **الأدوات المخصصة**: واجهات أدوات قابلة للتنفيذ يستدعيها النموذج بمخططات وتأثيرات جانبية في وقت التشغيل

### المهارات مقابل الخطافات

- **المهارات**: محتوى سلبي
- **الخطافات**: معترضات وقت تشغيل مدفوعة بالأحداث يمكنها حظر/تعديل السلوك أثناء التنفيذ

## إرشادات التأليف العملية المرتبطة بمنطق الاكتشاف

- ضع كل مهارة في دليلها الخاص: `<skills-root>/<skill-name>/SKILL.md`
- قم دائماً بتضمين بيانات أولية صريحة لـ `name` و`description`
- احتفظ بالأصول المُشار إليها تحت نفس دليل المهارة وقم بالوصول إليها عبر `skill://<name>/...`
- للتصنيف المتداخل (`team/domain/skill`)، وجّه `skills.customDirectories` إلى الدليل الأصل المتداخل؛ المسح نفسه يبقى غير تكراري
- تجنب تكرار أسماء المهارات عبر المصادر؛ أول تطابق يفوز بأسبقية الموفر
