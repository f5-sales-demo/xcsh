# المهارات

المهارات هي حزم قدرات مدعومة بملفات يتم اكتشافها عند بدء التشغيل وتُعرض للنموذج على شكل:

- بيانات وصفية خفيفة في موجه النظام (الاسم + الوصف)
- محتوى عند الطلب عبر `read skill://...`
- أوامر تفاعلية اختيارية `/skill:<name>`

يغطي هذا المستند سلوك وقت التشغيل الحالي في `src/extensibility/skills.ts`، و`src/discovery/builtin.ts`، و`src/internal-urls/skill-protocol.ts`، و`src/discovery/agents-md.ts`.

## ما هي المهارة في قاعدة الكود هذه

تُمثَّل المهارة المكتشفة بالشكل التالي:

- `name`
- `description`
- `filePath` (مسار `SKILL.md`)
- `baseDir` (مجلد المهارة)
- البيانات الوصفية للمصدر (`provider`، `level`، المسار)

يتطلب وقت التشغيل فقط `name` و`path` للصلاحية. في الممارسة العملية، تعتمد جودة المطابقة على أن يكون `description` ذا معنى.

## التخطيط المطلوب وتوقعات SKILL.md

### تخطيط المجلدات

بالنسبة للاكتشاف المبني على المزودين (المزودون الأصليون/Claude/Codex/Agents/الإضافات)، يتم اكتشاف المهارات **بمستوى واحد تحت `skills/`**:

- `<skills-root>/<skill-name>/SKILL.md`

الأنماط المتداخلة مثل `<skills-root>/group/<skill>/SKILL.md` لا يتم اكتشافها بواسطة محمّلات المزودين.

بالنسبة لـ `skills.customDirectories`، يستخدم الفحص نفس التخطيط غير التكراري (`*/SKILL.md`).

```text
Provider-discovered layout (non-recursive under skills/):

<root>/skills/
  ├─ postgres/
  │   └─ SKILL.md      ✅ discovered
  ├─ pdf/
  │   └─ SKILL.md      ✅ discovered
  └─ team/
      └─ internal/
          └─ SKILL.md  ❌ not discovered by provider loaders

Custom-directory scanning is also non-recursive, so nested paths are ignored unless you point `customDirectories` at that nested parent.
```

### البيانات الأمامية لـ `SKILL.md`

حقول البيانات الأمامية المدعومة في نوع المهارة:

- `name?: string`
- `description?: string`
- `globs?: string[]`
- `alwaysApply?: boolean`
- المفاتيح الإضافية تُحفظ كبيانات وصفية غير معروفة

سلوك وقت التشغيل الحالي:

- `name` يأخذ القيمة الافتراضية من اسم مجلد المهارة
- `description` مطلوب لـ:
  - اكتشاف مهارات مزود `.xcsh` الأصلي (`requireDescription: true`)
  - فحص `skills.customDirectories` عبر `scanSkillsFromDir` في `src/discovery/helpers.ts` (غير تكراري)
- المزودون غير الأصليين يمكنهم تحميل المهارات بدون وصف

## خط أنابيب الاكتشاف

`discoverSkills()` في `src/extensibility/skills.ts` يقوم بمرحلتين:

1. **مزودو القدرات** عبر `loadCapability("skills")`
2. **المجلدات المخصصة** عبر `scanSkillsFromDir(..., { requireDescription: true })` (تعداد مجلدات بمستوى واحد)

إذا كان `skills.enabled` يساوي `false`، فإن الاكتشاف لا يُرجع أي مهارات.

### مزودو المهارات المدمجون والأسبقية

ترتيب المزودين يعتمد على الأولوية أولاً (الأعلى يفوز)، ثم ترتيب التسجيل في حالة التساوي.

مزودو المهارات المسجلون حالياً:

1. `native` (أولوية 100) — مهارات المستخدم/المشروع `.xcsh` عبر `src/discovery/builtin.ts`
2. `claude` (أولوية 80)
3. مجموعة الأولوية 70 (بترتيب التسجيل):
   - `claude-plugins`
   - `agents`
   - `codex`

مفتاح إزالة التكرار هو اسم المهارة. العنصر الأول بالاسم المعطى يفوز.

### مفاتيح تبديل المصدر والتصفية

`discoverSkills()` يطبق عناصر التحكم التالية:

- مفاتيح تبديل المصدر: `enableCodexUser`، `enableClaudeUser`، `enableClaudeProject`، `enablePiUser`، `enablePiProject`
- مرشحات glob على اسم المهارة:
  - `ignoredSkills` (استبعاد)
  - `includeSkills` (قائمة السماح بالتضمين؛ فارغة تعني تضمين الكل)

ترتيب التصفية هو:

1. المصدر مُفعّل
2. غير مُتجاهل
3. مُضمّن (إذا كانت قائمة التضمين موجودة)

بالنسبة للمزودين غير codex/claude/native (على سبيل المثال `agents`، `claude-plugins`)، يعود التفعيل حالياً إلى: مُفعّل إذا كان **أي** مفتاح تبديل مصدر مدمج مُفعّلاً.

### التعامل مع التصادمات والتكرارات

- إزالة تكرار القدرات تحتفظ بالفعل بأول مهارة لكل اسم (المزود الأعلى أسبقية)
- `extensibility/skills.ts` بالإضافة إلى ذلك:
  - يزيل التكرارات للملفات المتطابقة بواسطة `realpath` (آمن مع الروابط الرمزية)
  - يصدر تحذيرات تصادم عندما يتعارض اسم مهارة لاحق
  - يحتفظ بواجهة API المريحة `discoverSkillsFromDir({ dir, source })` كمحول رقيق فوق `scanSkillsFromDir`
- مهارات المجلدات المخصصة تُدمج بعد مهارات المزودين وتتبع نفس سلوك التصادم

## سلوك الاستخدام في وقت التشغيل

### العرض في موجه النظام

بناء موجه النظام (`src/system-prompt.ts`) يستخدم المهارات المكتشفة كما يلي:

- إذا كانت أداة `read` متاحة:
  - تضمين قائمة المهارات المكتشفة في الموجه
- وإلا:
  - حذف القائمة المكتشفة

الوكلاء الفرعيون لأداة المهام يتلقون قائمة المهارات المكتشفة/المقدمة للجلسة عبر إنشاء الجلسة العادي؛ لا يوجد تجاوز لتثبيت المهارات لكل مهمة.

### أوامر `/skill:<name>` التفاعلية

إذا كان `skills.enableSkillCommands` يساوي true، يسجل الوضع التفاعلي أمر شرطة مائلة واحد لكل مهارة مكتشفة.

سلوك `/skill:<name> [args]`:

- يقرأ ملف المهارة مباشرة من `filePath`
- يزيل البيانات الأمامية
- يحقن نص المهارة كرسالة مخصصة لاحقة
- يُلحق البيانات الوصفية (`Skill: <path>`، اختيارياً `User: <args>`)

## سلوك عنوان URL `skill://`

`src/internal-urls/skill-protocol.ts` يدعم:

- `skill://<name>` ← يُحلّ إلى `SKILL.md` الخاص بتلك المهارة
- `skill://<name>/<relative-path>` ← يُحلّ داخل مجلد تلك المهارة

```text
skill:// URL resolution

skill://pdf
  -> <pdf-base>/SKILL.md

skill://pdf/references/tables.md
  -> <pdf-base>/references/tables.md

Guards:
- reject absolute paths
- reject `..` traversal
- reject any resolved path escaping <pdf-base>
```

تفاصيل الحل:

- يجب أن يتطابق اسم المهارة تماماً
- المسارات النسبية يتم فك ترميز URL الخاص بها
- المسارات المطلقة مرفوضة
- اجتياز المسار (`..`) مرفوض
- المسار المُحلّ يجب أن يبقى ضمن `baseDir`
- الملفات المفقودة تُرجع خطأ `File not found` صريح

نوع المحتوى:

- `.md` => `text/markdown`
- كل شيء آخر => `text/plain`

لا يتم إجراء بحث احتياطي للأصول المفقودة.

## المهارات مقابل AGENTS.md والأوامر والأدوات والخطافات

### المهارات مقابل AGENTS.md

- **المهارات**: حزم قدرات مسماة واختيارية يتم اختيارها حسب سياق المهمة أو يُطلب صراحة
- **AGENTS.md/ملفات السياق**: ملفات تعليمات دائمة تُحمّل كقدرة ملف سياق وتُدمج حسب قواعد المستوى/العمق

`src/discovery/agents-md.ts` يمشي تحديداً في المجلدات الأصل من `cwd` لاكتشاف ملفات `AGENTS.md` المستقلة (حتى عمق 20)، مع استبعاد أجزاء المجلدات المخفية.

### المهارات مقابل أوامر الشرطة المائلة

- **المهارات**: محتوى معرفي/سير عمل قابل للقراءة بواسطة النموذج
- **أوامر الشرطة المائلة**: نقاط دخول أوامر يستدعيها المستخدم
- `/skill:<name>` هو غلاف مريح يحقن نص المهارة؛ لا يغير دلالات اكتشاف المهارات

### المهارات مقابل الأدوات المخصصة

- **المهارات**: محتوى توثيقي/سير عمل يُحمّل عبر سياق الموجه و`read`
- **الأدوات المخصصة**: واجهات API أدوات قابلة للتنفيذ يستدعيها النموذج مع مخططات وآثار جانبية في وقت التشغيل

### المهارات مقابل الخطافات

- **المهارات**: محتوى سلبي
- **الخطافات**: معترضات وقت التشغيل المدفوعة بالأحداث التي يمكنها حظر/تعديل السلوك أثناء التنفيذ

## إرشادات التأليف العملية المرتبطة بمنطق الاكتشاف

- ضع كل مهارة في مجلدها الخاص: `<skills-root>/<skill-name>/SKILL.md`
- أضف دائماً بيانات أمامية صريحة لـ `name` و`description`
- احتفظ بالأصول المرجعية تحت نفس مجلد المهارة وقم بالوصول إليها عبر `skill://<name>/...`
- للتصنيف المتداخل (`team/domain/skill`)، وجّه `skills.customDirectories` إلى المجلد الأصل المتداخل؛ الفحص نفسه يبقى غير تكراري
- تجنب تكرار أسماء المهارات عبر المصادر؛ أول تطابق يفوز حسب أسبقية المزود
