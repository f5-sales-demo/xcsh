# النقل من pi-mono: دليل عملي للدمج

هذا الدليل عبارة عن قائمة تحقق قابلة للتكرار لنقل التغييرات من pi-mono إلى هذا المستودع.
استخدمه لأي عملية دمج: ملف واحد، أو فرع ميزة، أو مزامنة إصدار كاملة.

**الإيداع:** `b21b42d032919de2f2e6920a76fa9a37c3920c0a`
**التاريخ:** 2026-03-22

قم بتحديث هذا القسم بعد كل مزامنة؛ لا تعد استخدام النطاق السابق.

## 0) تحديد النطاق

- حدد المرجع المصدري (إيداع، وسم، أو طلب سحب).
- أدرج الحزم أو المجلدات التي تخطط للتعديل عليها.
- قرر أي الميزات ضمن النطاق وأيها يتم تخطيها عمداً.

## 1) نقل الكود بأمان

- فضّل فرقاً نظيفاً ومركزاً بدلاً من النسخ الشامل.
- تجنب نسخ الملفات المبنية أو المولّدة.
- إذا أضاف المصدر ملفات جديدة، أضفها صراحةً وراجع محتوياتها.

## 2) مطابقة اصطلاحات امتدادات الاستيراد

تحذف معظم ملفات TypeScript التنفيذية `.js` في الاستيرادات الداخلية، لكن بعض نقاط دخول الاختبارات/القياس تحتفظ بـ `.js` للتوافق مع ESM
في وقت التشغيل. اتبع النمط الحالي للحزمة المحلية؛ لا تزل الامتدادات بشكل شامل.

- في ملفات التشغيل ضمن `packages/coding-agent`، اجعل الاستيرادات الداخلية بدون امتداد ما لم تكن تستورد أصولاً غير TypeScript.
- في `packages/tui/test` و `packages/natives/bench`، احتفظ بـ `.js` حيث تستخدمه الملفات المحيطة بالفعل.
- احتفظ بامتدادات الملفات الحقيقية عندما تتطلبها الأدوات (مثل `.json`، `.css`، تضمينات نصوص `.md`).
- مثال: `import { x } from "./foo.js";` → `import { x } from "./foo";` (فقط عندما يكون اصطلاح الحزمة بدون امتداد).

## 3) استبدال نطاقات الاستيراد

يستخدم المصدر نطاقات حزم مختلفة. استبدلها بشكل متسق.

- استبدل النطاقات القديمة بالنطاق المحلي المستخدم هنا.
- أمثلة (عدّل لتتطابق مع الحزم الفعلية التي تنقلها):
  - `@mariozechner/pi-coding-agent` → `@f5xc-salesdemos/xcsh`
  - `@mariozechner/pi-agent-core` → `@f5xc-salesdemos/pi-agent-core`
  - `@mariozechner/pi-tui` → `@f5xc-salesdemos/pi-tui`
  - `@mariozechner/pi-ai` → `@f5xc-salesdemos/pi-ai`

## 4) استخدام واجهات Bun البرمجية حيثما تتفوق على Node

نعمل على Bun. استبدل واجهات Node البرمجية فقط عندما يوفر Bun بديلاً أفضل.

**يجب الاستبدال:**

- تشغيل العمليات: `child_process.spawn` → صدفة Bun `$` للأوامر البسيطة، `Bun.spawn`/`Bun.spawnSync` للبث أو العمل طويل الأمد
- إدخال/إخراج الملفات: `fs.readFileSync` → `Bun.file().text()` / `Bun.write()`
- عملاء HTTP: `node-fetch`، `axios` → `fetch` الأصلي
- تجزئة التشفير: `node:crypto` → Web Crypto أو `Bun.hash`
- SQLite: `better-sqlite3` → `bun:sqlite`
- تحميل المتغيرات البيئية: `dotenv` → Bun يحمّل `.env` تلقائياً

**لا تستبدل (هذه تعمل بشكل جيد في Bun):**

- `os.homedir()` — لا تستبدلها بـ `Bun.env.HOME` أو القيمة الحرفية `"~"`
- `os.tmpdir()` — لا تستبدلها بـ `Bun.env.TMPDIR || "/tmp"` أو مسارات مكتوبة يدوياً
- `fs.mkdtempSync()` — لا تستبدلها ببناء المسار اليدوي
- `path.join()`، `path.resolve()`، إلخ — هذه لا مشكلة فيها

**نمط الاستيراد:** استخدم بادئة `node:` مع استيرادات مساحة الأسماء فقط (بدون استيرادات مسماة من `node:fs` أو `node:path`).

**اصطلاحات Bun إضافية:**

- فضّل صدفة Bun `$` للأوامر القصيرة غير المتدفقة؛ استخدم `Bun.spawn` فقط عندما تحتاج إدخال/إخراج متدفق أو التحكم بالعملية.
- استخدم `Bun.file()`/`Bun.write()` للملفات و `node:fs/promises` للمجلدات.
- تجنب فحوصات `Bun.file().exists()`؛ استخدم معالجة `isEnoent` في try/catch.
- فضّل `Bun.sleep(ms)` على أغلفة `setTimeout`.

**خاطئ:**

```typescript
// BROKEN: env vars may be undefined, "~" is not expanded
const home = Bun.env.HOME || "~";
const tmp = Bun.env.TMPDIR || "/tmp";
```

**صحيح:**

```typescript
import * as os from "node:os";
import * as fs from "node:fs";
import * as path from "node:path";

const configDir = path.join(os.homedir(), ".config", "myapp");
const tempDir = fs.mkdtempSync(path.join(os.tmpdir(), "myapp-"));
```

## 5) تفضيل تضمينات Bun (بدون نسخ)

لا تنسخ الأصول التشغيلية أو ملفات المورّدين في وقت البناء.

- إذا كان المصدر ينسخ الأصول إلى مجلد dist، استبدل ذلك بتضمينات متوافقة مع Bun.
- الموجّهات (Prompts) هي ملفات `.md` ثابتة؛ استخدم استيرادات النصوص في Bun (`with { type: "text" }`) وHandlebars بدلاً من سلاسل الموجّهات المضمّنة.
- استخدم `import.meta.dir` + `Bun.file` لتحميل الموارد غير النصية المجاورة.
- احتفظ بالأصول في المستودع ودع المجمّع يضمّنها.
- أزل سكربتات النسخ ما لم يطلبها المستخدم صراحةً.
- إذا كان المصدر يقرأ ملفاً احتياطياً مجمّعاً في وقت التشغيل، استبدل قراءات نظام الملفات باستيراد نص مضمّن من Bun.
  - مثال (نص تعليمات Codex الاحتياطي):
    - `const FALLBACK_PROMPT_PATH = join(import.meta.dir, "codex-instructions.md");` -> تمت إزالته
    - `import FALLBACK_INSTRUCTIONS from "./codex-instructions.md" with { type: "text" };`
    - استخدم `return FALLBACK_INSTRUCTIONS;` بدلاً من `readFileSync(FALLBACK_PROMPT_PATH, "utf8")`

## 6) نقل `package.json` بعناية

عامل `package.json` كعقد. ادمج بشكل مقصود.

- احتفظ بـ `name`، `version`، `type`، `exports`، و `bin` الحاليين ما لم يتطلب النقل تغييرات.
- استبدل سكربتات npm/node ببدائل Bun (مثل `bun check`، `bun test`).
- تأكد من أن التبعيات تستخدم النطاق الصحيح.
- لا تخفّض إصدارات التبعيات لإصلاح أخطاء الأنواع؛ قم بالترقية بدلاً من ذلك.
- تحقق من روابط حزم مساحة العمل و `peerDependencies`.

## 7) محاذاة نمط الكود والأدوات

- حافظ على اصطلاحات التنسيق الحالية.
- لا تُدخل `any` ما لم يكن مطلوباً.
- تجنب الاستيرادات الديناميكية واستيرادات الأنواع المضمّنة؛ استخدم استيرادات المستوى الأعلى فقط.
- لا تبنِ الموجّهات في الكود أبداً؛ الموجّهات هي ملفات `.md` ثابتة تُعرض باستخدام Handlebars.
- في coding-agent، لا تستخدم `console.log`/`console.warn`/`console.error` أبداً؛ استخدم `logger` من `@f5xc-salesdemos/pi-utils`.
- استخدم `Promise.withResolvers()` بدلاً من `new Promise((resolve, reject) => ...)`.
- **لا تستخدم كلمات `private`/`protected`/`public` على حقول أو طرق الفئات.** استخدم حقول ES الخاصة `#` للتغليف؛ واترك الأعضاء القابلة للوصول بدون كلمة مفتاحية. الاستثناء الوحيد هو خصائص معاملات المُنشئ (`constructor(private readonly x: T)`)، حيث تكون الكلمة المفتاحية مطلوبة من TypeScript. عند نقل كود المصدر الذي يستخدم `private foo` أو `protected bar`، حوّل إلى `#foo` (خاص) أو `bar` بدون كلمة مفتاحية (قابل للوصول).
- فضّل المساعدات والأدوات الموجودة على الكود المخصص الجديد.
- حافظ على تغييرات البنية التحتية الموجهة لـ Bun التي أُجريت بالفعل في هذا المستودع:
  - بيئة التشغيل هي Bun (بدون نقاط دخول Node).
  - مدير الحزم هو Bun (بدون ملفات قفل npm).
  - واجهات Node الثقيلة (`child_process`، `readline`) مستبدلة ببدائل Bun.
  - واجهات Node الخفيفة (`os.homedir`، `os.tmpdir`، `fs.mkdtempSync`، `path.*`) محفوظة.
  - سطور shebang في CLI تستخدم `bun` (ليس `node`، ليس `tsx`).
  - الحزم تستخدم الملفات المصدرية مباشرة (بدون خطوة بناء TypeScript).
  - سير عمل CI يشغّل Bun للتثبيت/الفحص/الاختبار.

## 11) حماية الميزات المحسّنة (قائمة مصائد الانحدار)

إذا حسّنت سلوكاً محلياً بالفعل، عاملها على أنها **غير قابلة للتفاوض**. قبل النقل، دوّن
التحسينات وأضف فحوصات صريحة حتى لا تضيع في الدمج.

- **جمّد السلوك المتوقع**: أضف ملاحظة "قبل/بعد" مختصرة لكل تحسين (المدخلات، المخرجات،
  القيم الافتراضية، الحالات الحدية). هذا يمنع التراجع الصامت.
- **اربط الواجهات القديمة بالجديدة**: إذا أعاد المصدر تسمية مفاهيم (hooks → extensions، custom tools → tools، إلخ)،
  تأكد من أن كل نقطة دخول قديمة لا تزال تمر عبر المسار الصحيح. علم أو تصدير واحد مفقود يعني فقدان الوظيفة.
- **تحقق من التصديرات**: افحص `exports` في `package.json`، والأنواع العامة، وملفات البرميل. غالباً ما تنسى عمليات النقل من المصدر
  إعادة تصدير الإضافات المحلية.
- **غطِّ المسارات غير السعيدة**: إذا أصلحت معالجة الأخطاء، أو المهل الزمنية، أو المنطق الاحتياطي، أضف اختباراً أو
  على الأقل قائمة تحقق يدوية تمارس تلك المسارات.
- **تحقق من القيم الافتراضية وترتيب دمج التكوين**: غالباً ما تعيش التحسينات في القيم الافتراضية. تأكد من أن القيم الافتراضية الجديدة
  لم تتراجع (مثل أسبقية التكوين الجديدة، الميزات المعطّلة، قوائم الأدوات).
- **دقق سلوك البيئة/الصدفة**: إذا أصلحت التنفيذ أو العزل، تحقق من أن المسار الجديد لا يزال يستخدم بيئتك
  المنقّحة ولا يعيد تقديم تجاوزات الأسماء المستعارة/الدوال.
- **أعد تشغيل عينات مستهدفة**: احتفظ بمجموعة صغيرة من الأمثلة "المعروفة بأنها جيدة" وشغّلها بعد النقل
  (أعلام CLI، تسجيل الإضافات، تنفيذ الأدوات).

## 12) اكتشاف ومعالجة الكود المُعاد تصميمه

قبل نقل ملف، تحقق مما إذا كان المصدر قد أعاد تصميمه بشكل كبير:

```bash
# Compare the file you're about to port against what you have locally
git diff HEAD upstream/main -- path/to/file.ts
```

إذا أظهر الفرق أن الملف تمت **إعادة تصميمه** (ليس مجرد ترقيع):

- تجريدات جديدة، مفاهيم مُعاد تسميتها، وحدات مدمجة، تدفق بيانات متغير

فيجب عليك **قراءة التنفيذ الجديد بدقة** قبل النقل. الدمج الأعمى للكود المُعاد تصميمه يفقد الوظائف لأن:

ملاحظة: تم مؤخراً تقسيم الوضع التفاعلي إلى controllers/utils/types. عند نقل التغييرات ذات الصلة إلى الخلف، انقل التحديثات إلى الملفات الفردية التي أنشأناها وتأكد من بقاء ربط `interactive-mode.ts` متزامناً.

1. **تتغير القيم الافتراضية بصمت** - متغير جديد `defaultFoo = [a, b]` قد يحل محل `getAllFoo()` القديم الذي كان يُرجع `[a, b, c, d, e]`.

2. **تُسقط خيارات الواجهة البرمجية** - عندما تُدمج الأنظمة (مثل `hooks` + `customTools` → `extensions`)، قد لا تمر الخيارات القديمة عبر التنفيذ الجديد.

3. **تصبح مسارات الكود متقادمة** - مفهوم مُعاد تسميته (مثل `hookMessage` → `custom`) يحتاج تحديثات في كل عبارة switch، وحارس نوع، ومعالج — وليس فقط في التعريف.

4. **يتقلص السياق/القدرات** - الواجهات القديمة ربما كشفت `{ logger, typebox, pi }` التي نسيت الواجهات الجديدة تضمينها.

### عملية النقل الدلالي

عندما يُعيد المصدر تصميم وحدة:

1. **اقرأ التنفيذ القديم** - افهم ما كان يفعله، ما الخيارات التي كان يقبلها، وما الذي كان يكشفه.

2. **اقرأ التنفيذ الجديد** - افهم التجريدات الجديدة وكيف تُطابق السلوك القديم.

3. **تحقق من تكافؤ الميزات** - لكل قدرة في الكود القديم، تأكد من أن الكود الجديد يحافظ عليها أو يزيلها صراحةً.

4. **ابحث عن البقايا** - ابحث عن الأسماء/المفاهيم القديمة التي قد فُوتت في عبارات switch، والمعالجات، ومكونات واجهة المستخدم.

5. **اختبر الحدود** - أعلام CLI، وخيارات SDK، ومعالجات الأحداث، والقيم الافتراضية — هذه هي حيث تختبئ الانحدارات.

### فحوصات سريعة

```bash
# Find all uses of an old concept that may need updating
rg "oldConceptName" --type ts

# Compare default values between versions
git show upstream/main:path/to/file.ts | rg "default|DEFAULT"

# Check if all enum/union values have handlers
rg "case \"" path/to/file.ts
```

## 13) قائمة تحقق سريعة للتدقيق

استخدم هذه كمراجعة نهائية قبل الانتهاء:

- [ ] امتدادات الاستيراد تتبع اصطلاح الحزمة المحلية (بدون إزالة شاملة لـ `.js`)
- [ ] لا توجد واجهات برمجية خاصة بـ Node فقط في الكود الجديد/المنقول
- [ ] جميع نطاقات الحزم محدّثة
- [ ] سكربتات `package.json` تستخدم Bun
- [ ] الموجّهات هي استيرادات نصية `.md` (بدون سلاسل موجّهات مضمّنة)
- [ ] لا يوجد `console.*` في coding-agent (استخدم `logger`)
- [ ] الأصول تُحمّل عبر أنماط تضمين Bun (بدون سكربتات نسخ)
- [ ] الاختبارات أو الفحوصات تعمل (أو مُشار إليها صراحةً على أنها محظورة)
- [ ] لا توجد انحدارات في الوظائف (انظر الأقسام 11-12)

## 14) تنسيق رسالة الإيداع

عند إيداع عملية نقل خلفي، اتبع تنسيق المستودع `<type>(scope): <past-tense description>` واحتفظ بنطاق
الإيداعات في العنوان.

```
fix(coding-agent): backported pi-mono changes (<from>..<to>)

packages/<package>:
- <type>: <description>
- <type>: <description> (#<issue> by @<contributor>)

packages/<other-package>:
- <type>: <description>
```

**مثال:**

```
fix(coding-agent): backported pi-mono changes (9f3eef65f..52532c7c0)

packages/ai:
- fix: handle "sensitive" stop reason from Anthropic API
- fix: normalize tool call IDs with special characters for Responses API
- fix: add overflow detection for Bedrock, MiniMax, Kimi providers
- fix: 429 status is rate limiting, not context overflow

packages/tui:
- fix: refactored autocomplete state tracking
- fix: file autocomplete should not trigger on empty text
- fix: configurable autocomplete max visible items
- fix: improved table column width calculation with word-aware wrapping

packages/coding-agent:
- fix: preserve external config.yml edits on save (#1046 by @nicobailonMD)
- fix: resolve macOS NFD and curly quote variants in file paths
```

**القواعد:**

- جمّع التغييرات حسب الحزمة
- استخدم أنواع الإيداعات التقليدية (`fix`، `feat`، `refactor`، `perf`، `docs`)
- ضمّن أرقام المشكلات/طلبات السحب من المصدر وإسناد المساهم للمساهمات الخارجية
- نطاق الإيداعات في العنوان يساعد في تتبع نقاط المزامنة

## 15) الاختلافات المقصودة

يحتوي فرعنا على قرارات معمارية تختلف عن المصدر. **لا تنقل هذه الأنماط من المصدر:**

### بنية واجهة المستخدم

| المصدر                                    | فرعنا                                                  | السبب                                                                |
