# إخفاء الأسرار

يمنع إرسال القيم الحساسة (مفاتيح API، والرموز المميزة، وكلمات المرور) إلى موفري LLM. عند التفعيل، تُستبدل الأسرار بعناصر نائبة حتمية قبل مغادرتها للعملية، ويُعاد استرجاعها في وسائط استدعاء الأدوات التي يُرجعها النموذج.

## التفعيل

مفعّل افتراضيًا. يمكن التبديل عبر واجهة المستخدم `/settings` أو مباشرةً في `config.yml`:

```yaml
secrets:
  enabled: false
```

## آلية العمل

1. عند بدء الجلسة، تُجمع الأسرار من مصدرين:
   - **متغيرات البيئة** المطابقة لأنماط الأسرار الشائعة (`*_KEY`، و`*_SECRET`، و`*_TOKEN`، و`*_PASSWORD`، إلخ) بقيم لا تقل عن 8 أحرف
   - **ملفات `secrets.yml`** (انظر أدناه)

2. تُستبدل جميع قيم الأسرار في الرسائل الصادرة إلى LLM بعناصر نائبة من قبيل `<<$env:S0>>` و`<<$env:S1>>` وما إلى ذلك.

3. تُجرى عملية مسح معمّق على وسائط استدعاء الأدوات التي يُرجعها النموذج، ويُعاد استرجاع العناصر النائبة إلى قيمها الأصلية قبل التنفيذ.

يتحكم وضعان في ما يحدث لكل سر:

| الوضع | السلوك | قابلية الاسترداد |
|---|---|---|
| `obfuscate` (الافتراضي) | يُستبدل بعنصر نائب مفهرس `<<$env:SN>>` | نعم (يُرفع الإخفاء في وسائط الأدوات) |
| `replace` | يُستبدل بسلسلة حتمية بنفس الطول | لا (أحادي الاتجاه) |

## secrets.yml

عرّف إدخالات الأسرار المخصصة في YAML. يتم التحقق من موقعين:

| المستوى | المسار | الغرض |
|---|---|---|
| عام | `~/.xcsh/agent/secrets.yml` | الأسرار عبر جميع المشاريع |
| المشروع | `<cwd>/.xcsh/secrets.yml` | الأسرار الخاصة بالمشروع |

تُقدَّم إدخالات المشروع على الإدخالات العامة ذات `content` المطابق.

### المخطط

يحتوي كل إدخال في المصفوفة على هذه الحقول:

| الحقل | النوع | مطلوب | الوصف |
|---|---|---|---|
| `type` | `"plain"` أو `"regex"` | نعم | استراتيجية المطابقة |
| `content` | سلسلة نصية | نعم | قيمة السر (plain) أو نمط التعبير النمطي (regex) |
| `mode` | `"obfuscate"` أو `"replace"` | لا | الافتراضي: `"obfuscate"` |
| `replacement` | سلسلة نصية | لا | الاستبدال المخصص (وضع replace فقط) |
| `flags` | سلسلة نصية | لا | أعلام التعبير النمطي (النوع regex فقط) |

### أمثلة

#### الأسرار العادية

```yaml
# إخفاء مفتاح API محدد (الوضع الافتراضي)
- type: plain
  content: sk-proj-abc123def456

# استبدال كلمة مرور قاعدة البيانات بسلسلة ثابتة
- type: plain
  content: hunter2
  mode: replace
  replacement: "********"
```

#### أسرار التعبير النمطي

```yaml
# إخفاء أي مفتاح بأسلوب AWS
- type: regex
  content: "AKIA[0-9A-Z]{16}"

# مطابقة غير حساسة لحالة الأحرف مع أعلام صريحة
- type: regex
  content: "api[_-]?key\\s*=\\s*\\w+"
  flags: "i"

# صيغة التعبير النمطي الحرفي (النمط والأعلام في سلسلة واحدة)
- type: regex
  content: "/bearer\\s+[a-zA-Z0-9._~+\\/=-]+/i"
```

تُجري إدخالات التعبير النمطي دائمًا مسحًا شاملًا (يُطبَّق العلم `g` تلقائيًا). تُدعم صيغة التعبير النمطي الحرفي `/pattern/flags` بديلًا عن حقلَي `content` و`flags` المنفصلَين. تُعالَج الشُّرَط المائلة المُهربة داخل النمط (`\\/`) بشكل صحيح.

#### وضع الاستبدال مع التعبير النمطي

```yaml
# استبدال أحادي الاتجاه لسلاسل الاتصال (غير قابل للاسترداد)
- type: regex
  content: "postgres://[^\\s]+"
  mode: replace
  replacement: "postgres://***"
```

## التفاعل مع اكتشاف متغيرات البيئة

تُجمع متغيرات البيئة دائمًا أولًا. تُضاف الإدخالات المعرَّفة في الملفات بعدها، مما يتيح لإدخالات الملفات تغطية الأسرار التي لا توجد في متغيرات البيئة (ملفات الإعداد، والقيم المضمَّنة في الكود، إلخ). إذا ظهرت القيمة ذاتها في المصدرين، فإن وضع إدخال الملف يحظى بالأولوية.

## الملفات الرئيسية

- `src/secrets/index.ts` -- التحميل والدمج وجمع متغيرات البيئة
- `src/secrets/obfuscator.ts` -- فئة `SecretObfuscator`، وتوليد العناصر النائبة، وإخفاء الرسائل
- `src/secrets/regex.ts` -- تحليل التعبير النمطي الحرفي وتصريفه
- `src/config/settings-schema.ts` -- تعريف إعداد `secrets.enabled`
