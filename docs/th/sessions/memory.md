# หน่วยความจำอัตโนมัติ

เมื่อเปิดใช้งาน เอเจนต์จะดึงความรู้ที่คงทนจากเซสชันก่อนหน้าโดยอัตโนมัติ และฉีดสรุปแบบกะทัดรัดเข้าไปในแต่ละเซสชันใหม่ เมื่อเวลาผ่านไป ระบบจะสร้างที่จัดเก็บหน่วยความจำในระดับโปรเจกต์ — การตัดสินใจทางเทคนิค เวิร์กโฟลว์ที่เกิดขึ้นซ้ำ ข้อผิดพลาดที่พบบ่อย — ที่ส่งต่อไปโดยไม่ต้องดำเนินการด้วยตนเอง

ปิดใช้งานโดยค่าเริ่มต้น เปิดใช้งานผ่าน `/settings` หรือ `config.yml`:

```yaml
memories:
  enabled: true
```

## การใช้งาน

### สิ่งที่ถูกฉีดเข้าไป

เมื่อเริ่มเซสชัน หากมีสรุปหน่วยความจำสำหรับโปรเจกต์ปัจจุบัน สรุปนั้นจะถูกฉีดเข้าไปในพรอมต์ระบบในรูปแบบบล็อก **Memory Guidance** เอเจนต์จะได้รับคำสั่งให้:

- ถือว่าหน่วยความจำเป็นบริบทแบบฮิวริสติก — มีประโยชน์สำหรับกระบวนการและการตัดสินใจก่อนหน้า แต่ไม่ใช่แหล่งอ้างอิงที่เชื่อถือได้สำหรับสถานะรีโปปัจจุบัน
- อ้างอิงเส้นทางอาร์ติแฟกต์หน่วยความจำเมื่อหน่วยความจำเปลี่ยนแปลงแผน และจับคู่กับหลักฐานจากรีโปปัจจุบันก่อนดำเนินการ
- ให้ความสำคัญกับสถานะรีโปและคำสั่งของผู้ใช้เมื่อขัดแย้งกับหน่วยความจำ ถือว่าหน่วยความจำที่ขัดแย้งเป็นข้อมูลเก่า

### การอ่านอาร์ติแฟกต์หน่วยความจำ

เอเจนต์สามารถอ่านไฟล์หน่วยความจำโดยตรงโดยใช้ URL `memory://` กับเครื่องมือ `read`:

| URL | เนื้อหา |
|---|---|
| `memory://root` | สรุปแบบกะทัดรัดที่ฉีดเข้าไปเมื่อเริ่มต้น |
| `memory://root/MEMORY.md` | เอกสารหน่วยความจำระยะยาวฉบับเต็ม |
| `memory://root/skills/<name>/SKILL.md` | คู่มือปฏิบัติงานทักษะที่สร้างขึ้น |

### คำสั่ง slash `/memory`

| คำสั่งย่อย | ผลลัพธ์ |
|---|---|
| `view` | แสดงข้อมูลหน่วยความจำที่ฉีดเข้าไปในปัจจุบัน |
| `clear` / `reset` | ลบข้อมูลหน่วยความจำทั้งหมดและอาร์ติแฟกต์ที่สร้างขึ้น |
| `enqueue` / `rebuild` | บังคับให้การรวบรวมทำงานเมื่อเริ่มต้นครั้งถัดไป |

## วิธีการทำงาน

หน่วยความจำถูกสร้างโดยไปป์ไลน์ที่ทำงานในเบื้องหลังเมื่อเริ่มต้นหรือทริกเกอร์ด้วยตนเองผ่านคำสั่ง slash

**เฟส 1 — การดึงข้อมูลต่อเซสชัน:** สำหรับแต่ละเซสชันก่อนหน้าที่มีการเปลี่ยนแปลงตั้งแต่ครั้งสุดท้ายที่ประมวลผล โมเดลจะอ่านประวัติเซสชันและดึงสัญญาณที่คงทน: การตัดสินใจทางเทคนิค ข้อจำกัด ความล้มเหลวที่แก้ไขแล้ว เวิร์กโฟลว์ที่เกิดขึ้นซ้ำ เซสชันที่ใหม่เกินไป เก่าเกินไป หรือกำลังใช้งานอยู่จะถูกข้าม แต่ละการดึงข้อมูลจะสร้างบล็อกหน่วยความจำดิบและสรุปสั้นสำหรับเซสชันนั้น

**เฟส 2 — การรวบรวม:** หลังจากการดึงข้อมูล โมเดลจะผ่านรอบที่สองเพื่ออ่านการดึงข้อมูลต่อเซสชันทั้งหมดและสร้างเอาต์พุตสามรายการที่เขียนลงดิสก์:

- `MEMORY.md` — เอกสารหน่วยความจำระยะยาวที่คัดสรรแล้ว
- `memory_summary.md` — ข้อความกะทัดรัดที่ฉีดเข้าไปเมื่อเริ่มเซสชัน
- `skills/` — คู่มือปฏิบัติงานเชิงขั้นตอนที่สามารถนำกลับมาใช้ได้ แต่ละรายการอยู่ในไดเรกทอรีย่อยของตัวเอง

เฟส 2 ใช้การล็อกเพื่อป้องกันการรันซ้ำเมื่อหลายกระบวนการเริ่มต้นพร้อมกัน ไดเรกทอรีทักษะที่เก่าจากการรันก่อนหน้าจะถูกตัดออกโดยอัตโนมัติ

เอาต์พุตทั้งหมดจะถูกสแกนหาข้อมูลลับก่อนเขียนลงดิสก์

### พฤติกรรมการดึงข้อมูล

พฤติกรรมการดึงข้อมูลหน่วยความจำและการรวบรวมถูกขับเคลื่อนทั้งหมดโดยไฟล์พรอมต์แบบคงที่ใน `src/prompts/memories/`

| ไฟล์ | วัตถุประสงค์ | ตัวแปร |
|---|---|---|
| `stage_one_system.md` | พรอมต์ระบบสำหรับการดึงข้อมูลต่อเซสชัน | — |
| `stage_one_input.md` | เทมเพลตเทิร์นของผู้ใช้ที่ห่อหุ้มเนื้อหาเซสชัน | `{{thread_id}}`, `{{response_items_json}}` |
| `consolidation.md` | พรอมต์สำหรับการรวบรวมข้ามเซสชัน | `{{raw_memories}}`, `{{rollout_summaries}}` |
| `read_path.md` | คำแนะนำหน่วยความจำที่ฉีดเข้าไปในเซสชันจริง | `{{memory_summary}}` |

### การเลือกโมเดล

หน่วยความจำใช้ระบบบทบาทของโมเดล

| เฟส | บทบาท | วัตถุประสงค์ |
|---|---|---|
| เฟส 1 (การดึงข้อมูล) | `default` | การดึงข้อมูลความรู้ต่อเซสชัน |
| เฟส 2 (การรวบรวม) | `smol` | การสังเคราะห์ข้ามเซสชัน |

หากไม่ได้กำหนดค่า `smol` เฟส 2 จะใช้บทบาท `default` แทน

## การกำหนดค่า

| การตั้งค่า | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| `memories.enabled` | `false` | สวิตช์หลัก |
| `memories.maxRolloutAgeDays` | `30` | เซสชันที่เก่ากว่านี้จะไม่ถูกประมวลผล |
| `memories.minRolloutIdleHours` | `12` | เซสชันที่ใช้งานล่าสุดกว่านี้จะถูกข้าม |
| `memories.maxRolloutsPerStartup` | `64` | จำนวนสูงสุดของเซสชันที่ประมวลผลในการเริ่มต้นครั้งเดียว |
| `memories.summaryInjectionTokenLimit` | `5000` | จำนวนโทเค็นสูงสุดของสรุปที่ฉีดเข้าไปในพรอมต์ระบบ |

ตัวปรับแต่งเพิ่มเติม (การทำงานพร้อมกัน ระยะเวลาการล็อก งบประมาณโทเค็น) มีอยู่ในการกำหนดค่าสำหรับการใช้งานขั้นสูง

## ไฟล์สำคัญ

- `src/memories/index.ts` — การจัดการไปป์ไลน์ การฉีดข้อมูล การจัดการคำสั่ง slash
- `src/memories/storage.ts` — คิวงานที่ใช้ SQLite และทะเบียนเธรด
- `src/prompts/memories/` — เทมเพลตพรอมต์หน่วยความจำ
- `src/internal-urls/memory-protocol.ts` — ตัวจัดการ URL `memory://`
