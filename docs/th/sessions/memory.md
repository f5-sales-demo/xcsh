# หน่วยความจำอัตโนมัติ

เมื่อเปิดใช้งาน เอเจนต์จะดึงความรู้ที่คงทนจากเซสชันก่อนหน้าโดยอัตโนมัติ และแทรกสรุปแบบกระชับเข้าไปในแต่ละเซสชันใหม่ เมื่อเวลาผ่านไประบบจะสร้างคลังหน่วยความจำระดับโปรเจกต์ ไม่ว่าจะเป็นการตัดสินใจทางเทคนิค เวิร์กโฟลว์ที่เกิดซ้ำ หรือข้อผิดพลาดที่ควรระวัง ซึ่งจะถูกส่งต่อไปโดยไม่ต้องดำเนินการด้วยตนเอง

ปิดใช้งานเป็นค่าเริ่มต้น เปิดใช้งานผ่าน `/settings` หรือ `config.yml`:

```yaml
memories:
  enabled: true
```

## การใช้งาน

### สิ่งที่ถูกแทรกเข้าไป

เมื่อเริ่มเซสชัน หากมีสรุปหน่วยความจำสำหรับโปรเจกต์ปัจจุบัน ระบบจะแทรกเข้าไปในระบบพรอมต์เป็นบล็อก **Memory Guidance** โดยเอเจนต์จะได้รับคำสั่งให้:

- ปฏิบัติต่อหน่วยความจำในฐานะบริบทแบบฮิวริสติก — มีประโยชน์สำหรับกระบวนการและการตัดสินใจก่อนหน้า แต่ไม่ถือเป็นแหล่งข้อมูลที่เชื่อถือได้สำหรับสถานะ repo ปัจจุบัน
- อ้างอิงเส้นทางของอาร์ติแฟกต์หน่วยความจำเมื่อหน่วยความจำเปลี่ยนแผน และจับคู่กับหลักฐานจาก repo ปัจจุบันก่อนดำเนินการ
- ให้ความสำคัญกับสถานะ repo และคำสั่งของผู้ใช้เมื่อขัดแย้งกับหน่วยความจำ โดยถือว่าหน่วยความจำที่ขัดแย้งนั้นล้าสมัย

### การอ่านอาร์ติแฟกต์หน่วยความจำ

เอเจนต์สามารถอ่านไฟล์หน่วยความจำโดยตรงโดยใช้ URL `memory://` กับเครื่องมือ `read`:

| URL | เนื้อหา |
|---|---|
| `memory://root` | สรุปแบบกระชับที่แทรกเข้าตอนเริ่มต้น |
| `memory://root/MEMORY.md` | เอกสารหน่วยความจำระยะยาวฉบับเต็ม |
| `memory://root/skills/<name>/SKILL.md` | คู่มือทักษะที่สร้างขึ้น |

### คำสั่งสแลช `/memory`

| คำสั่งย่อย | ผลลัพธ์ |
|---|---|
| `view` | แสดงเพย์โหลดการแทรกหน่วยความจำปัจจุบัน |
| `clear` / `reset` | ลบข้อมูลหน่วยความจำทั้งหมดและอาร์ติแฟกต์ที่สร้างขึ้น |
| `enqueue` / `rebuild` | บังคับให้การรวบรวมทำงานในการเริ่มต้นครั้งถัดไป |

## วิธีการทำงาน

หน่วยความจำถูกสร้างขึ้นโดยไปป์ไลน์พื้นหลังที่ทำงานตอนเริ่มต้นหรือเรียกใช้ด้วยตนเองผ่านคำสั่งสแลช

**เฟส 1 — การดึงข้อมูลต่อเซสชัน:** สำหรับแต่ละเซสชันก่อนหน้าที่มีการเปลี่ยนแปลงตั้งแต่ครั้งสุดท้ายที่ถูกประมวลผล โมเดลจะอ่านประวัติเซสชันและดึงสัญญาณที่คงทนออกมา ได้แก่ การตัดสินใจทางเทคนิค ข้อจำกัด ข้อผิดพลาดที่แก้ไขแล้ว และเวิร์กโฟลว์ที่เกิดซ้ำ เซสชันที่ใหม่เกินไป เก่าเกินไป หรือกำลังใช้งานอยู่จะถูกข้าม การดึงข้อมูลแต่ละครั้งจะสร้างบล็อกหน่วยความจำดิบและสรุปสั้น ๆ สำหรับเซสชันนั้น

**เฟส 2 — การรวบรวม:** หลังจากการดึงข้อมูล โมเดลจะทำการประมวลผลรอบที่สองโดยอ่านข้อมูลที่ดึงจากทุกเซสชันและสร้างผลลัพธ์สามรายการที่เขียนลงดิสก์:

- `MEMORY.md` — เอกสารหน่วยความจำระยะยาวที่คัดสรรแล้ว
- `memory_summary.md` — ข้อความกระชับที่แทรกเข้าตอนเริ่มเซสชัน
- `skills/` — คู่มือขั้นตอนที่นำกลับมาใช้ซ้ำได้ แต่ละรายการอยู่ในไดเรกทอรีย่อยของตัวเอง

เฟส 2 ใช้กลไกล็อค (lease) เพื่อป้องกันการทำงานซ้ำซ้อนเมื่อหลายกระบวนการเริ่มต้นพร้อมกัน ไดเรกทอรีทักษะที่ล้าสมัยจากการทำงานครั้งก่อนจะถูกลบออกโดยอัตโนมัติ

ผลลัพธ์ทั้งหมดจะถูกสแกนหาความลับก่อนเขียนลงดิสก์

### พฤติกรรมการดึงข้อมูล

พฤติกรรมการดึงข้อมูลหน่วยความจำและการรวบรวมถูกขับเคลื่อนทั้งหมดโดยไฟล์พรอมต์แบบคงที่ใน `src/prompts/memories/`

| ไฟล์ | วัตถุประสงค์ | ตัวแปร |
|---|---|---|
| `stage_one_system.md` | ระบบพรอมต์สำหรับการดึงข้อมูลต่อเซสชัน | — |
| `stage_one_input.md` | เทมเพลตเทิร์นผู้ใช้ที่ห่อหุ้มเนื้อหาเซสชัน | `{{thread_id}}`, `{{response_items_json}}` |
| `consolidation.md` | พรอมต์สำหรับการรวบรวมข้ามเซสชัน | `{{raw_memories}}`, `{{rollout_summaries}}` |
| `read_path.md` | แนวทางหน่วยความจำที่แทรกเข้าในเซสชันสด | `{{memory_summary}}` |

### การเลือกโมเดล

หน่วยความจำใช้ระบบบทบาทโมเดลที่มีอยู่

| เฟส | บทบาท | วัตถุประสงค์ |
|---|---|---|
| เฟส 1 (การดึงข้อมูล) | `default` | การดึงความรู้ต่อเซสชัน |
| เฟส 2 (การรวบรวม) | `smol` | การสังเคราะห์ข้ามเซสชัน |

หากไม่ได้กำหนดค่า `smol` เฟส 2 จะใช้บทบาท `default` แทน

## การกำหนดค่า

| การตั้งค่า | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| `memories.enabled` | `false` | สวิตช์หลัก |
| `memories.maxRolloutAgeDays` | `30` | เซสชันที่เก่ากว่านี้จะไม่ถูกประมวลผล |
| `memories.minRolloutIdleHours` | `12` | เซสชันที่มีกิจกรรมล่าสุดกว่านี้จะถูกข้าม |
| `memories.maxRolloutsPerStartup` | `64` | จำนวนเซสชันสูงสุดที่ประมวลผลในการเริ่มต้นครั้งเดียว |
| `memories.summaryInjectionTokenLimit` | `5000` | โทเค็นสูงสุดของสรุปที่แทรกเข้าในระบบพรอมต์ |

ตัวปรับแต่งเพิ่มเติม (ความเกิดพร้อมกัน ระยะเวลาล็อค งบประมาณโทเค็น) มีให้ในการกำหนดค่าสำหรับการใช้งานขั้นสูง

## ไฟล์สำคัญ

- `src/memories/index.ts` — การจัดการไปป์ไลน์ การแทรก การจัดการคำสั่งสแลช
- `src/memories/storage.ts` — คิวงานและรีจิสทรีเธรดที่ใช้ SQLite
- `src/prompts/memories/` — เทมเพลตพรอมต์หน่วยความจำ
- `src/internal-urls/memory-protocol.ts` — ตัวจัดการ URL `memory://`
