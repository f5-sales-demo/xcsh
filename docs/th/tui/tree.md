# คู่มืออ้างอิงคำสั่ง `/tree`

`/tree` เปิดตัวนำทาง **Session Tree** แบบโต้ตอบ ช่วยให้คุณข้ามไปยังรายการใดก็ได้ในไฟล์เซสชันปัจจุบันและดำเนินการต่อจากจุดนั้น

นี่คือการย้าย leaf ภายในไฟล์ ไม่ใช่การส่งออกเซสชันใหม่

## สิ่งที่ `/tree` ทำ

- สร้างโครงสร้างต้นไม้จากรายการเซสชันปัจจุบัน (`SessionManager.getTree()`)
- เปิด `TreeSelectorComponent` พร้อมการนำทางด้วยแป้นพิมพ์ ตัวกรอง และการค้นหา
- เมื่อเลือก จะเรียก `AgentSession.navigateTree(targetId, { summarize, customInstructions })`
- สร้างแชทที่มองเห็นได้ใหม่จากเส้นทาง leaf ใหม่
- เลือกที่จะเติมข้อความในเอดิเตอร์ล่วงหน้าเมื่อเลือกข้อความของผู้ใช้/ข้อความกำหนดเอง

การ implement หลัก:

- `src/modes/controllers/input-controller.ts` (`/tree`, การเชื่อมต่อ keybinding, พฤติกรรม double-escape)
- `src/modes/controllers/selector-controller.ts` (เปิด UI ต้นไม้ + ขั้นตอนพรอมต์สรุป)
- `src/modes/components/tree-selector.ts` (การนำทาง, ตัวกรอง, การค้นหา, ป้ายกำกับ, การแสดงผล)
- `src/session/agent-session.ts` (`navigateTree` การสลับ leaf + สรุปเสริม)
- `src/session/session-manager.ts` (`getTree`, `branch`, `branchWithSummary`, `resetLeaf`, การคงอยู่ของป้ายกำกับ)

## วิธีเปิดใช้งาน

วิธีใดก็ได้ต่อไปนี้จะเปิดตัวเลือกเดียวกัน:

- `/tree`
- การกระทำ keybinding ที่กำหนดค่า `tree`
- double-escape บนเอดิเตอร์ว่างเมื่อ `doubleEscapeAction = "tree"` (ค่าเริ่มต้น)
- `/branch` เมื่อ `doubleEscapeAction = "tree"` (ไปที่ตัวเลือกต้นไม้แทนตัวเลือกสาขาเฉพาะผู้ใช้)

## โมเดล UI ต้นไม้

ต้นไม้แสดงผลจากตัวชี้แม่ของรายการเซสชัน (`id` / `parentId`)

- รายการลูกเรียงตามเวลาจากเก่าไปใหม่ (เก่ากว่าก่อน ใหม่กว่าอยู่ด้านล่าง)
- สาขาที่ใช้งานอยู่ (เส้นทางจากรากไปยัง leaf ปัจจุบัน) ถูกทำเครื่องหมายด้วยจุด
- ป้ายกำกับ (ถ้ามี) แสดงเป็น `[label]` ก่อนข้อความของโหนด
- ถ้ามีรากหลายตัว (สายแม่ที่กำพร้า/ขาด) จะแสดงภายใต้รากแยกสาขาเสมือน

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

ตัวเลือกจะเลื่อนกลับมาตรงกลางรอบๆ การเลือกปัจจุบันและแสดงได้สูงสุด:

- `max(5, floor(terminalHeight / 2))` แถว

## การผูกแป้นพิมพ์ภายในตัวเลือกต้นไม้

- `Up` / `Down`: เลื่อนการเลือก (วนรอบ)
- `Left` / `Right`: เลื่อนหน้าขึ้น / เลื่อนหน้าลง
- `Enter`: เลือกโหนด
- `Esc`: ล้างการค้นหาถ้ากำลังใช้งาน มิฉะนั้นปิดตัวเลือก
- `Ctrl+C`: ปิดตัวเลือก
- `Type`: เพิ่มต่อท้ายคำค้นหา
- `Backspace`: ลบอักขระค้นหา
- `Shift+L`: แก้ไข/ล้างป้ายกำกับบนรายการที่เลือก
- `Ctrl+O`: วนตัวกรองไปข้างหน้า
- `Shift+Ctrl+O`: วนตัวกรองไปข้างหลัง
- `Alt+D/T/U/L/A`: ข้ามไปยังโหมดตัวกรองเฉพาะโดยตรง

## ตัวกรองและความหมายของการค้นหา

โหมดตัวกรอง (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

แสดงโหนดการสนทนาส่วนใหญ่ แต่ซ่อนประเภทรายการบันทึกระบบ:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

เหมือนกับ `default` แต่ซ่อนข้อความ `toolResult` เพิ่มเติม

### `user-only`

เฉพาะรายการ `message` ที่มีบทบาทเป็น `user`

### `labeled-only`

เฉพาะรายการที่ปัจจุบันมีป้ายกำกับ

### `all`

ทุกอย่างในโครงสร้างต้นไม้เซสชัน รวมถึงรายการบันทึกระบบ/กำหนดเอง

### พฤติกรรมโหนด assistant ที่มีเฉพาะ tool

ข้อความ assistant ที่มี**เฉพาะการเรียก tool** (ไม่มีข้อความ) จะถูกซ่อนตามค่าเริ่มต้นในมุมมองที่กรองทั้งหมด เว้นแต่:

- ข้อความเป็นข้อผิดพลาด/ถูกยกเลิก (`stopReason` ไม่ใช่ `stop`/`toolUse`) หรือ
- เป็น leaf ปัจจุบัน (แสดงเสมอ)

### พฤติกรรมการค้นหา

- คำค้นหาถูกแบ่งด้วยช่องว่าง
- การจับคู่ไม่สนใจตัวพิมพ์ใหญ่-เล็ก
- ทุก token ต้องตรงกัน (ความหมายแบบ AND)
- ข้อความที่ค้นหาได้รวมถึงป้ายกำกับ บทบาท และเนื้อหาเฉพาะประเภท (ข้อความ ข้อความสรุปสาขา ประเภทกำหนดเอง ข้อมูลย่อคำสั่ง tool เป็นต้น)

## ผลลัพธ์ของการเลือก (สำคัญ)

`navigateTree` คำนวณพฤติกรรม leaf ใหม่จากประเภทรายการที่เลือก:

### การเลือกข้อความ `user`

- leaf ใหม่จะกลายเป็น `parentId` ของรายการที่เลือก
- ถ้า parent เป็น `null` (ข้อความผู้ใช้ราก) leaf จะรีเซ็ตไปที่ราก (`resetLeaf()`)
- ข้อความของข้อความที่เลือกจะถูกคัดลอกไปยังเอดิเตอร์เพื่อแก้ไข/ส่งใหม่

### การเลือก `custom_message`

- กฎ leaf เดียวกับข้อความผู้ใช้ (`parentId`)
- เนื้อหาข้อความจะถูกแยกออกและคัดลอกไปยังเอดิเตอร์

### การเลือกโหนดที่ไม่ใช่ผู้ใช้ (assistant/tool/สรุป/การบีบอัด/บันทึกระบบกำหนดเอง/อื่นๆ)

- leaf ใหม่จะกลายเป็น id ของโหนดที่เลือก
- เอดิเตอร์จะไม่ถูกเติมล่วงหน้า

### การเลือก leaf ปัจจุบัน

- ไม่มีการดำเนินการ; ตัวเลือกปิดพร้อมข้อความ "Already at this point"

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## ขั้นตอนการสรุปเมื่อสลับ

พรอมต์สรุปถูกควบคุมโดย `branchSummary.enabled` (ค่าเริ่มต้น: `false`)

เมื่อเปิดใช้งาน หลังจากเลือกโหนด UI จะถาม:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

รายละเอียดขั้นตอน:

- Escape ในพรอมต์สรุปจะเปิดตัวเลือกต้นไม้อีกครั้ง
- การยกเลิกพรอมต์กำหนดเองจะกลับไปที่ลูปตัวเลือกสรุป
- ระหว่างการสรุป UI จะแสดงตัวโหลดและผูก `Esc` กับ `abortBranchSummary()`
- ถ้าการสรุปถูกยกเลิก ตัวเลือกต้นไม้จะเปิดใหม่และไม่มีการย้ายที่ถูกนำไปใช้

ภายในของ `navigateTree`:

- รวบรวมรายการสาขาที่ถูกทิ้งจาก leaf เก่าไปยังบรรพบุรุษร่วม
- ส่ง `session_before_tree` (ส่วนขยายสามารถยกเลิกหรือแทรกสรุปได้)
- ใช้ตัวสรุปเริ่มต้นเฉพาะเมื่อถูกร้องขอและจำเป็น
- ดำเนินการย้ายด้วย:
  - `branchWithSummary(...)` เมื่อมีสรุป
  - `branch(newLeafId)` สำหรับการย้ายที่ไม่ใช่รากโดยไม่มีสรุป
  - `resetLeaf()` สำหรับการย้ายไปรากโดยไม่มีสรุป
- แทนที่การสนทนาของ agent ด้วยบริบทเซสชันที่สร้างใหม่
- ส่ง `session_tree`

หมายเหตุ: ถ้าผู้ใช้ร้องขอสรุปแต่ไม่มีอะไรให้สรุป การนำทางจะดำเนินต่อโดยไม่สร้างรายการสรุป

## ป้ายกำกับ

การแก้ไขป้ายกำกับใน UI ต้นไม้จะเรียก `appendLabelChange(targetId, label)`

- ป้ายกำกับที่ไม่ว่างจะตั้ง/อัปเดตป้ายกำกับที่แก้ไขแล้ว
- ป้ายกำกับว่างจะล้างมัน
- ป้ายกำกับถูกเก็บเป็นรายการ `label` แบบ append-only
- โหนดต้นไม้แสดงสถานะป้ายกำกับที่แก้ไขแล้ว ไม่ใช่ประวัติรายการป้ายกำกับดิบ

## `/tree` เทียบกับการดำเนินการที่เกี่ยวข้อง

| การดำเนินการ | ขอบเขต | ผลลัพธ์ |
|---|---|---|
| `/tree` | ไฟล์เซสชันปัจจุบัน | ย้าย leaf ไปยังจุดที่เลือก (ไฟล์เดียวกัน) |
| `/branch` | โดยปกติไฟล์เซสชันปัจจุบัน -> ไฟล์เซสชันใหม่ | ตามค่าเริ่มต้นจะแยกสาขาจากข้อความ **ผู้ใช้** ที่เลือกเป็นไฟล์เซสชันใหม่; ถ้า `doubleEscapeAction = "tree"`, `/branch` จะเปิด UI นำทางต้นไม้แทน |
| `/fork` | ทั้งเซสชันปัจจุบัน | ทำสำเนาเซสชันเป็นไฟล์เซสชันที่คงอยู่ใหม่ |
| `/resume` | รายการเซสชัน | สลับไปยังไฟล์เซสชันอื่น |

ความแตกต่างหลัก: `/tree` เป็นเครื่องมือนำทาง/จัดตำแหน่งใหม่ภายในไฟล์เซสชันเดียว `/branch`, `/fork` และ `/resume` ทั้งหมดเปลี่ยนบริบทไฟล์เซสชัน

## เวิร์กโฟลว์สำหรับผู้ปฏิบัติงาน

### เรียกใช้ซ้ำจากพรอมต์ผู้ใช้ก่อนหน้าโดยไม่สูญเสียสาขาปัจจุบัน

1. `/tree`
2. ค้นหา/เลือกข้อความผู้ใช้ก่อนหน้า
3. เลือก `No summary` (หรือสรุปถ้าจำเป็น)
4. แก้ไขข้อความที่เติมล่วงหน้าในเอดิเตอร์
5. ส่ง

ผลลัพธ์: สาขาใหม่เติบโตจากจุดที่เลือกภายในไฟล์เซสชันเดียวกัน

### ออกจากสาขาปัจจุบันพร้อมเครื่องหมายบริบท

1. เปิดใช้งาน `branchSummary.enabled`
2. `/tree` และเลือกโหนดเป้าหมาย
3. เลือก `Summarize` (หรือพรอมต์กำหนดเอง)

ผลลัพธ์: รายการ `branch_summary` จะถูกเพิ่มที่ตำแหน่งเป้าหมายก่อนดำเนินการต่อ

### ตรวจสอบรายการบันทึกระบบที่ซ่อนอยู่

1. `/tree`
2. กด `Alt+A` (ทั้งหมด)
3. ค้นหา `model`, `thinking`, `custom` หรือป้ายกำกับ

ผลลัพธ์: ตรวจสอบไทม์ไลน์ภายในทั้งหมด ไม่ใช่แค่โหนดการสนทนา

### บุ๊กมาร์กจุดหมุนสำหรับการข้ามในภายหลัง

1. `/tree`
2. เลื่อนไปที่รายการ
3. `Shift+L` และตั้งป้ายกำกับ
4. ภายหลังใช้ `Alt+L` (`labeled-only`) เพื่อข้ามอย่างรวดเร็ว

ผลลัพธ์: การนำทางที่รวดเร็วระหว่างจุดสังเกตสาขาที่คงทน
