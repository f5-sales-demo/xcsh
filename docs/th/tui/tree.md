# คู่มืออ้างอิงคำสั่ง `/tree`

`/tree` เปิดตัวนำทาง **Session Tree** แบบโต้ตอบ ช่วยให้คุณข้ามไปยังรายการใดก็ได้ในไฟล์เซสชันปัจจุบันและดำเนินการต่อจากจุดนั้น

นี่คือการย้าย leaf ภายในไฟล์ ไม่ใช่การส่งออกเซสชันใหม่

## สิ่งที่ `/tree` ทำ

- สร้างโครงสร้างต้นไม้จากรายการเซสชันปัจจุบัน (`SessionManager.getTree()`)
- เปิด `TreeSelectorComponent` พร้อมการนำทางด้วยแป้นพิมพ์ ตัวกรอง และการค้นหา
- เมื่อเลือก จะเรียก `AgentSession.navigateTree(targetId, { summarize, customInstructions })`
- สร้างแชทที่มองเห็นใหม่จากเส้นทาง leaf ใหม่
- สามารถเติมข้อความในเอดิเตอร์ล่วงหน้าเมื่อเลือกข้อความ user/custom

การ implement หลัก:

- `src/modes/controllers/input-controller.ts` (`/tree`, การผูกคีย์, พฤติกรรม double-escape)
- `src/modes/controllers/selector-controller.ts` (เปิด UI ต้นไม้ + ขั้นตอนสรุป)
- `src/modes/components/tree-selector.ts` (การนำทาง, ตัวกรอง, การค้นหา, ป้ายกำกับ, การเรนเดอร์)
- `src/session/agent-session.ts` (`navigateTree` การสลับ leaf + สรุปตามต้องการ)
- `src/session/session-manager.ts` (`getTree`, `branch`, `branchWithSummary`, `resetLeaf`, การเก็บป้ายกำกับ)

## วิธีเปิด

วิธีใดต่อไปนี้จะเปิดตัวเลือกเดียวกัน:

- `/tree`
- action คีย์ที่กำหนดค่าไว้ `tree`
- double-escape บนเอดิเตอร์ที่ว่างเมื่อ `doubleEscapeAction = "tree"` (ค่าเริ่มต้น)
- `/branch` เมื่อ `doubleEscapeAction = "tree"` (เปลี่ยนเส้นทางไปยังตัวเลือกต้นไม้แทนตัวเลือกแยกสาขาเฉพาะผู้ใช้)

## โมเดล UI ของต้นไม้

ต้นไม้ถูกเรนเดอร์จากตัวชี้ parent ของรายการเซสชัน (`id` / `parentId`)

- โหนดลูกเรียงลำดับตามเวลาจากน้อยไปมาก (เก่ากว่าก่อน ใหม่กว่าอยู่ด้านล่าง)
- สาขาที่ใช้งานอยู่ (เส้นทางจาก root ถึง leaf ปัจจุบัน) ถูกทำเครื่องหมายด้วยจุดกลม
- ป้ายกำกับ (ถ้ามี) แสดงเป็น `[label]` ก่อนข้อความโหนด
- หากมีหลาย root (สาย parent ที่ขาดหรือกำพร้า) จะแสดงภายใต้ root แยกสาขาเสมือน

```text
Example tree view (active path marked with •):

├─ user: "Start task"
│  └─ assistant: "Plan"
│     ├─ • user: "Try approach A"
│     │  └─ • assistant: "A result"
│     │     └─ • [milestone] user: "Continue A"
│     └─ user: "Try approach B"
│        └─ assistant: "B result"
```

ตัวเลือกจะจัดศูนย์ใหม่รอบการเลือกปัจจุบันและแสดงได้สูงสุด:

- `max(5, floor(terminalHeight / 2))` แถว

## คีย์ลัดภายในตัวเลือกต้นไม้

- `Up` / `Down`: เลื่อนการเลือก (วนรอบ)
- `Left` / `Right`: เลื่อนหน้าขึ้น / เลื่อนหน้าลง
- `Enter`: เลือกโหนด
- `Esc`: ล้างการค้นหาถ้าใช้งานอยู่ มิฉะนั้นปิดตัวเลือก
- `Ctrl+C`: ปิดตัวเลือก
- `Type`: เพิ่มข้อความในคำค้นหา
- `Backspace`: ลบอักขระในคำค้นหา
- `Shift+L`: แก้ไข/ล้างป้ายกำกับบนรายการที่เลือก
- `Ctrl+O`: สลับตัวกรองไปข้างหน้า
- `Shift+Ctrl+O`: สลับตัวกรองย้อนกลับ
- `Alt+D/T/U/L/A`: ข้ามไปยังโหมดตัวกรองเฉพาะโดยตรง

## ตัวกรองและหลักการทำงานของการค้นหา

โหมดตัวกรอง (`TreeList`):

1. `default`
2. `no-tools`
3. `user-only`
4. `labeled-only`
5. `all`

### `default`

แสดงโหนดการสนทนาส่วนใหญ่ แต่ซ่อนประเภทรายการที่เป็นการจัดการภายใน:

- `label`
- `custom`
- `model_change`
- `thinking_level_change`

### `no-tools`

เหมือนกับ `default` แต่ซ่อนข้อความ `toolResult` เพิ่มเติม

### `user-only`

เฉพาะรายการ `message` ที่ role เป็น `user`

### `labeled-only`

เฉพาะรายการที่แก้ไขเป็นป้ายกำกับได้ในปัจจุบัน

### `all`

ทุกอย่างในต้นไม้เซสชัน รวมถึงรายการจัดการภายใน/custom

### พฤติกรรมโหนด assistant ที่มีเฉพาะเครื่องมือ

ข้อความ assistant ที่มี **เฉพาะการเรียกเครื่องมือ** (ไม่มีข้อความ) จะถูกซ่อนตามค่าเริ่มต้นในมุมมองที่กรองทั้งหมด ยกเว้น:

- ข้อความเป็น error/ถูกยกเลิก (`stopReason` ไม่ใช่ `stop`/`toolUse`) หรือ
- เป็น leaf ปัจจุบัน (จะแสดงเสมอ)

### พฤติกรรมการค้นหา

- คำค้นหาจะถูกแยกคำด้วยช่องว่าง
- การจับคู่ไม่สนใจตัวพิมพ์เล็ก-ใหญ่
- ทุกคำต้องตรงกัน (หลักการ AND)
- ข้อความที่ค้นหาได้รวมถึงป้ายกำกับ, role และเนื้อหาเฉพาะประเภท (ข้อความ, ข้อความสรุปสาขา, ประเภท custom, ส่วนย่อของคำสั่งเครื่องมือ เป็นต้น)

## ผลลัพธ์การเลือก (สำคัญ)

`navigateTree` คำนวณพฤติกรรม leaf ใหม่จากประเภทรายการที่เลือก:

### การเลือกข้อความ `user`

- Leaf ใหม่จะเป็น `parentId` ของรายการที่เลือก
- หาก parent เป็น `null` (ข้อความ user ที่เป็น root) leaf จะรีเซ็ตไปยัง root (`resetLeaf()`)
- ข้อความที่เลือกจะถูกคัดลอกไปยังเอดิเตอร์เพื่อแก้ไข/ส่งใหม่

### การเลือก `custom_message`

- กฎ leaf เดียวกับข้อความ user (`parentId`)
- เนื้อหาข้อความจะถูกดึงออกและคัดลอกไปยังเอดิเตอร์

### การเลือกโหนดที่ไม่ใช่ user (assistant/tool/summary/compaction/custom bookkeeping/อื่นๆ)

- Leaf ใหม่จะเป็น id ของโหนดที่เลือก
- เอดิเตอร์จะไม่ถูกเติมล่วงหน้า

### การเลือก leaf ปัจจุบัน

- ไม่มีการดำเนินการ ตัวเลือกปิดพร้อมข้อความ "Already at this point"

```text
Selection decision (simplified):

selected node
   │
   ├─ is current leaf? ── yes ──> close selector (no-op)
   │
   ├─ is user/custom_message? ── yes ──> leaf := parentId (or resetLeaf for root)
   │                                     + prefill editor text
   │
   └─ otherwise ──> leaf := selected node id
                    + no editor prefill
```

## ขั้นตอนการสรุปเมื่อสลับ

การแสดงข้อความสรุปควบคุมโดย `branchSummary.enabled` (ค่าเริ่มต้น: `false`)

เมื่อเปิดใช้งาน หลังจากเลือกโหนด UI จะถาม:

- `No summary`
- `Summarize`
- `Summarize with custom prompt`

รายละเอียดขั้นตอน:

- กด Escape ในข้อความสรุปจะเปิดตัวเลือกต้นไม้ใหม่
- การยกเลิก custom prompt จะกลับไปที่วงลูปตัวเลือกสรุป
- ระหว่างการสรุป UI จะแสดงตัวโหลดและผูก `Esc` กับ `abortBranchSummary()`
- หากการสรุปถูกยกเลิก ตัวเลือกต้นไม้จะเปิดใหม่และไม่มีการย้าย

รายละเอียดภายในของ `navigateTree`:

- รวบรวมรายการสาขาที่ถูกทิ้งจาก leaf เก่าถึงบรรพบุรุษร่วม
- ปล่อยเหตุการณ์ `session_before_tree` (ส่วนขยายสามารถยกเลิกหรือแทรกสรุปได้)
- ใช้ตัวสรุปเริ่มต้นเฉพาะเมื่อมีการร้องขอและจำเป็น
- ดำเนินการย้ายด้วย:
  - `branchWithSummary(...)` เมื่อมีสรุป
  - `branch(newLeafId)` สำหรับการย้ายที่ไม่ใช่ root โดยไม่มีสรุป
  - `resetLeaf()` สำหรับการย้ายไปยัง root โดยไม่มีสรุป
- แทนที่บทสนทนาของ agent ด้วยบริบทเซสชันที่สร้างใหม่
- ปล่อยเหตุการณ์ `session_tree`

หมายเหตุ: หากผู้ใช้ร้องขอสรุปแต่ไม่มีอะไรให้สรุป การนำทางจะดำเนินต่อโดยไม่สร้างรายการสรุป

## ป้ายกำกับ

การแก้ไขป้ายกำกับใน UI ต้นไม้เรียก `appendLabelChange(targetId, label)`

- ป้ายกำกับที่ไม่ว่างจะตั้งค่า/อัปเดตป้ายกำกับที่แก้ไขแล้ว
- ป้ายกำกับว่างจะล้างป้ายกำกับ
- ป้ายกำกับถูกจัดเก็บเป็นรายการ `label` แบบเพิ่มเท่านั้น (append-only)
- โหนดต้นไม้แสดงสถานะป้ายกำกับที่แก้ไขแล้ว ไม่ใช่ประวัติรายการป้ายกำกับดิบ

## `/tree` เทียบกับการดำเนินการที่เกี่ยวข้อง

| การดำเนินการ | ขอบเขต | ผลลัพธ์ |
|---|---|---|
| `/tree` | ไฟล์เซสชันปัจจุบัน | ย้าย leaf ไปยังจุดที่เลือก (ไฟล์เดิม) |
| `/branch` | โดยปกติไฟล์เซสชันปัจจุบัน -> ไฟล์เซสชันใหม่ | ตามค่าเริ่มต้นจะแยกสาขาจากข้อความ **user** ที่เลือกไปยังไฟล์เซสชันใหม่ หาก `doubleEscapeAction = "tree"`, `/branch` จะเปิด UI นำทางต้นไม้แทน |
| `/fork` | เซสชันปัจจุบันทั้งหมด | คัดลอกเซสชันไปยังไฟล์เซสชันใหม่ที่บันทึกถาวร |
| `/resume` | รายการเซสชัน | สลับไปยังไฟล์เซสชันอื่น |

ข้อแตกต่างสำคัญ: `/tree` เป็นเครื่องมือนำทาง/จัดตำแหน่งใหม่ภายในไฟล์เซสชันเดียว `/branch`, `/fork` และ `/resume` ทั้งหมดเปลี่ยนบริบทไฟล์เซสชัน

## เวิร์กโฟลว์สำหรับผู้ใช้งาน

### รันซ้ำจากข้อความ user ก่อนหน้าโดยไม่สูญเสียสาขาปัจจุบัน

1. `/tree`
2. ค้นหา/เลือกข้อความ user ก่อนหน้า
3. เลือก `No summary` (หรือสรุปถ้าจำเป็น)
4. แก้ไขข้อความที่เติมล่วงหน้าในเอดิเตอร์
5. ส่ง

ผลลัพธ์: สาขาใหม่เติบโตจากจุดที่เลือกภายในไฟล์เซสชันเดียวกัน

### ออกจากสาขาปัจจุบันพร้อมเครื่องหมายบริบท

1. เปิดใช้งาน `branchSummary.enabled`
2. `/tree` และเลือกโหนดเป้าหมาย
3. เลือก `Summarize` (หรือ custom prompt)

ผลลัพธ์: รายการ `branch_summary` จะถูกเพิ่มที่ตำแหน่งเป้าหมายก่อนดำเนินการต่อ

### ตรวจสอบรายการจัดการภายในที่ซ่อนอยู่

1. `/tree`
2. กด `Alt+A` (all)
3. ค้นหา `model`, `thinking`, `custom` หรือป้ายกำกับ

ผลลัพธ์: ตรวจสอบไทม์ไลน์ภายในทั้งหมด ไม่ใช่แค่โหนดการสนทนา

### บุ๊กมาร์กจุดสำคัญเพื่อข้ามกลับไปทีหลัง

1. `/tree`
2. เลื่อนไปที่รายการ
3. `Shift+L` และตั้งป้ายกำกับ
4. ภายหลังใช้ `Alt+L` (`labeled-only`) เพื่อข้ามไปอย่างรวดเร็ว

ผลลัพธ์: นำทางอย่างรวดเร็วระหว่างจุดสังเกตสาขาที่คงทน
